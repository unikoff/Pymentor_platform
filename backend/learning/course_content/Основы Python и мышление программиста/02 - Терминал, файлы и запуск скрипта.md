# Терминал, файлы и запуск скрипта

Установим необходимые инструменты, создадим папку проекта и запустим один и тот же Python-файл через терминал, VS Code или PyCharm.

**Темы:** установка и запуск · терминал или IDE

## 01. Что понадобится для работы

Для первых проектов нужны три вещи: Python, папка с файлами и удобное место для редактирования кода. Представьте рабочий стол: Python — исполнитель, папка — ящик с проектом, редактор — место, где мы меняем инструкции.

1. **Установить Python:** он нужен, чтобы компьютер мог выполнять файлы `.py`.
2. **Выбрать редактор:** можно использовать VS Code или PyCharm.
3. **Создать папку проекта:** все файлы StudyHub будут лежать в одном понятном месте.

Затем мы запустим файл двумя способами и создадим виртуальное окружение, которое понадобится проекту позже.

### Исполнитель кода — Python

Скачивается с официального сайта Python. В Windows при установке важно разрешить добавление Python в PATH, чтобы команда работала в терминале.

```powershell
python --version
```

### Лёгкий редактор — VS Code

После установки добавьте расширение Python. Внутри есть редактор, дерево файлов, терминал и кнопка запуска.

### Готовая Python-IDE — PyCharm

В PyCharm многие инструменты уже собраны вместе. Нужно открыть папку проекта и выбрать установленный интерпретатор.

> VS Code и PyCharm — не разные версии Python. Это разные рабочие столы. В обоих случаях код выполняет тот же интерпретатор.

## 02. Редактор, Проводник и терминал — разные инструменты

Новичок часто видит несколько окон и думает, что это одна программа. Проще представить мастерскую: в одном месте лежат детали, в другом мы редактируем чертёж, в третьем запускаем работу.

### Показывает папки и файлы — Проводник

Здесь можно увидеть папку `studyhub`, файл `main.py` и проверить точное имя.

### Изменяет код — редактор

Здесь пишется и сохраняется содержимое файла. Несохранённые изменения ещё не попали на диск.

```python
print("StudyHub")
```

### Выполняет команды — терминал

Здесь мы сообщаем компьютеру, какой файл нужно запустить, и видим результат.

```powershell
python main.py
```

**код в редакторе**

```python
print("StudyHub Planner")
```

**команда и результат**

```text
> python main.py
StudyHub Planner
```

> Команда `python main.py` пишется в терминале, а строка `print("StudyHub")` — внутри файла.

## 03. Папка проекта и путь как адрес

Путь к файлу похож на почтовый адрес. Чтобы найти квартиру, нужно знать дом и номер. Чтобы найти `main.py`, терминалу нужно знать папку, в которой он лежит.

**простая структура**

```text
backend-course/
└── studyhub/
    ├── main.py
    └── .venv/
```

### `pwd` — где я нахожусь?

Команда показывает текущую папку. Это как посмотреть на табличку с адресом комнаты, в которой вы сейчас стоите.

### `dir` — что лежит рядом?

Команда показывает файлы и папки внутри текущего места. Если `main.py` не виден, команда `python main.py` не сможет найти его по короткому имени.

### `cd` — перейти в другую папку

`cd studyhub` открывает дочернюю папку, а `cd ..` возвращает на один уровень выше.

```powershell
pwd
dir
cd studyhub
dir
```

**Пошаговый разбор**

1. Сначала узнаём текущий адрес. **Состояние:** папка = backend-course
2. Проверяем, что рядом есть папка studyhub. **Состояние:** найдено = studyhub
3. Переходим внутрь проекта. **Состояние:** папка = backend-course/studyhub
4. Проверяем, что теперь рядом находится main.py. **Состояние:** найдено = main.py

> Если в Windows файл называется `main.py.txt`, Python не считает его обычным Python-файлом. Проверяйте расширение через Проводник или `dir`.

## 04. Два способа запуска: терминал и IDE

Один файл можно запустить разными кнопками, но внутри происходит одно и то же: выбранный интерпретатор получает путь до `main.py` и выполняет код.

### Прямой запуск — терминал

Самый прозрачный способ. Видно текущую папку, команду, путь и результат.

```powershell
python main.py
```

### Кнопка Run или встроенный терминал — VS Code

Можно нажать запуск файла или открыть терминал внутри VS Code и ввести ту же команду.

### Run main — PyCharm

PyCharm сам формирует команду, используя выбранный интерпретатор и текущую конфигурацию проекта.

### Какой способ основной в курсе

Мы чаще используем терминал, потому что он показывает все шаги без скрытой магии. Кнопка IDE остаётся удобным дополнительным способом.

### Почему результат должен совпадать

Если IDE и терминал запускают один сохранённый файл тем же Python, вывод будет одинаковым. Если он отличается, обычно выбран другой файл, другой интерпретатор или изменения не сохранены.

### Сравнение решений

**Вопрос:** Как объяснить разницу между способами запуска?

### Терминал

```powershell
python main.py
```

Мы вручную указываем интерпретатору файл.

### Кнопка IDE

```python
Run main.py
```

IDE собирает похожую команду за нас.

**Вывод:** предпочтителен правый вариант. Оба способа нормальны. Для обучения полезно уметь хотя бы один раз выполнить прямую команду самостоятельно.

## 05. Сохранение файла и первые проблемы запуска

Python выполняет сохранённую версию файла. Это похоже на отправку документа: пока вы не нажали «Сохранить», на диске остаётся предыдущая версия.

**studyhub/main.py**

```python
print("Новая версия StudyHub")
```

**проверяем запуск**

```text
> python main.py
Новая версия StudyHub
```

### Команда python не найдена

Проверьте `python --version`. В Windows также может работать `py --version`. Если не работает ничего, нужно вернуться к установке Python.

### Сообщение can't open file

Python запустился, но не нашёл файл по указанному адресу. Проверьте `pwd`, `dir` и имя файла.

### Показывается старый текст

Сначала сохраните файл. Затем убедитесь, что запускается именно тот `main.py`, который открыт в редакторе.

### Разбор ошибки

```python
print("Новый текст")
```

**Вопрос:** Код изменён, но после запуска виден старый результат. Что проверить первым?

A. Файл сохранён и запускается нужный main.py
B. Удалить Python
C. Добавить вторую команду print

**Ответ:** A. Файл сохранён и запускается нужный main.py

Интерпретатор читает сохранённый файл по конкретному пути.

## 06. Виртуальное окружение: отдельный набор инструментов

Виртуальное окружение можно представить как отдельный ящик с инструментами для одного проекта. Сейчас он почти пустой, поэтому внешне ничего не изменится. Позже в него будут устанавливаться библиотеки StudyHub.

### Зачем оно обязательно проекту

У разных проектов могут быть разные библиотеки и разные версии. Отдельный ящик не позволяет инструментам одного проекта случайно смешаться с другим.

### Нужно ли скачивать .venv отдельно

Нет. После установки Python модуль `venv` уже доступен. Папка создаётся командой `python -m venv .venv`.

### Почему сейчас почти ничего не поменяется

В первых файлах используются только возможности самого Python. Настоящую пользу окружения мы увидим в следующих блоках, когда начнём устанавливать дополнительные библиотеки.

**создание и активация в PowerShell**

```text
> python -m venv .venv
> .\.venv\Scripts\Activate.ps1
(.venv) PS C:\backend-course\studyhub>
> python main.py
StudyHub Planner
> deactivate
```

> Создать окружение нужно один раз. После открытия нового терминала существующее окружение просто активируют снова. Если PowerShell блокирует активацию, не вводите случайные команды из интернета — разберите настройку с наставником.

## 07. Практикум: один файл, три запуска

Цель практики — увидеть, что терминал, VS Code и PyCharm могут запускать один и тот же сохранённый файл. Разница находится в интерфейсе, а не в самом Python-коде.

**studyhub/main.py**

```python
print("StudyHub Planner")
print("Файл сохранён и найден")
```

**прямой запуск**

```text
> pwd
C:\backend-course\studyhub
> dir
.venv   main.py
> python main.py
StudyHub Planner
Файл сохранён и найден
```

### Шаг 1. Запуск из терминала

Проверьте папку через `pwd` и `dir`, затем выполните `python main.py`.

### Шаг 2. Запуск из редактора

Откройте тот же файл в VS Code или PyCharm и используйте кнопку Run.

### Шаг 3. Изменение и повтор

Замените вторую строку на `Запуск проверен двумя способами`, сохраните файл и повторите оба запуска.

### Соберите безопасный запуск

Расположите действия в понятном порядке.

**Фрагменты**

- `открыть папку studyhub`
- `сохранить main.py`
- `pwd и dir`
- `python main.py`

**Правильный порядок**

```powershell
открыть папку studyhub
сохранить main.py
pwd и dir
python main.py
```

Сначала выбираем проект и сохраняем файл, затем подтверждаем путь и запускаем.

## 08. Что должно получиться после занятия

После урока не требуется помнить все команды без подсказки. Важно понимать маршрут и уметь повторить его по короткой инструкции.

### Вопрос 1

Что выполняет Python-код при нажатии Run в IDE?

A. Выбранный Python-интерпретатор
B. сама кнопка
C. браузер

**Ответ:** A. Выбранный Python-интерпретатор

IDE только помогает сформировать запуск. Код выполняет интерпретатор.

### Вопрос 2

Зачем перед запуском использовать pwd и dir?

A. Проверить папку и наличие файла
B. Изменить код
C. Создать GitHub

**Ответ:** A. Проверить папку и наличие файла

Эти команды помогают понять, где находится терминал и что лежит рядом.

### Вопрос 3

Нужно ли скачивать .venv отдельной программой?

A. Нет, её создаёт установленный Python
B. Да, с GitHub
C. Да, из VS Code

**Ответ:** A. Нет, её создаёт установленный Python

Команда python -m venv .venv создаёт окружение внутри проекта.

### Главное после занятия

- Python скачивается отдельно и выполняет файлы `.py`.
- VS Code и PyCharm являются рабочими средами, а не заменой интерпретатора.
- `pwd` показывает текущую папку, `dir` — содержимое, `cd` меняет папку.
- Один файл можно запускать командой терминала или кнопкой IDE.
- Перед запуском файл нужно сохранить.
- Виртуальное окружение создаётся для проекта сейчас, а его польза станет заметна с появлением библиотек.

### Практика

Установите Python и один редактор, создайте папку studyhub, запустите main.py через терминал и через кнопку IDE, затем создайте .venv.
