# Цикл while и проверка ввода

Научимся повторять действие, пока условие остаётся истинным: проверять ввод, завершать цикл через break и не попадать в бесконечное повторение.

**Темы:** while и состояние · break · continue

## 01. Маршрут занятия

`while` похож на фразу «повторяй, пока дверь закрыта». Мы заранее не знаем, сколько попыток потребуется, но понимаем условие продолжения. У цикла обязательно должен быть путь к завершению.

1. **Задать состояние:** определить значение до первой проверки.
2. **Записать условие:** объяснить, пока какое правило цикл должен продолжаться.
3. **Гарантировать выход:** изменить состояние или выполнить достижимый `break`.

В конце вы найдёте бесконечный цикл, повторите ввод до допустимой команды и реализуете алгоритм, который вручную двигается по значениям.

> Перед запуском ответьте на два вопроса: «что изменится после одного круга?» и «какое значение остановит цикл?». Если ответа нет, цикл пока запускать рано.

## 02. Три части while

Управляемый `while` похож на счётчик попыток: есть стартовое значение, правило продолжения и действие, которое приближает нас к остановке. Условие проверяется перед каждым новым кругом.

```python
attempt = 1
while attempt <= 3:
    print(attempt)
    attempt = attempt + 1
```

**Пошаговый разбор**

1. Начальное состояние attempt = 1. **Состояние:** attempt = 1
2. 1 <= 3 даёт True, поэтому начинается первая итерация. **Состояние:** attempt = 1
3. Печатается 1. **Состояние:** вывод = 1; attempt = 1
4. attempt становится 2. **Состояние:** attempt = 2
5. После нескольких шагов attempt станет 4. **Состояние:** attempt = 4
6. 4 <= 3 даёт False, цикл завершён. **Состояние:** attempt = 4; вывод = 1 ⏎ 2 ⏎ 3

### Верно или неверно

Если условие `while` ложно до первой итерации, тело всё равно выполнится один раз.

**Ответ:** неверно. while проверяет условие до входа в тело, поэтому возможны ноль итераций.

### Вспомните модель

**Вопрос:** Назовите три части управляемого цикла while.

**Ответ:** Начальное состояние, условие продолжения и изменение состояния либо явный выход.

## 03. Как появляется бесконечный цикл

Бесконечный цикл похож на будильник без кнопки выключения. Если условие зависит от значения, которое никогда не меняется, программа будет повторять один и тот же круг.

### Разбор ошибки

```python
number = 1
while number <= 5:
    print(number)
```

**Вопрос:** Почему цикл не завершится?

A. print перезапускает while
B. number остаётся равным 1
C. Оператор <= всегда возвращает True

**Ответ:** B. number остаётся равным 1

Состояние не обновляется, поэтому условие 1 <= 5 остаётся истинным.

**Исправленный вариант**

```python
number = 1
while number <= 5:
    print(number)
    number = number + 1
```

Бесконечный цикл не всегда является ошибкой. Консольное приложение может использовать `while True`, если внутри есть видимый и достижимый выход.

Если невозможно словами доказать завершение, не запускайте цикл наугад. Сначала уменьшите пример и проследите несколько значений вручную.

### Сравнение решений

**Вопрос:** Какой вариант явно движется к завершению?

### Состояние не меняется

```python
number = 1
while number <= 5:
    print(number)
```

Условие всегда проверяет одно и то же значение.

### Состояние обновляется

```python
number = 1
while number <= 5:
    print(number)
    number += 1
```

После каждой итерации число приближается к границе выхода.

**Вывод:** предпочтителен правый вариант. Изменение number делает условие ложным после конечного числа шагов.

## 04. Повторный ввод до допустимой команды

Количество неверных попыток заранее неизвестно. Поэтому ввод обновляет состояние, а условие проверяет, можно ли завершить повтор.

**повторяем до допустимой команды**

```python
command = ""

while command not in ("add", "list", "exit"):
    command = input("Команда: ").strip().lower()

print("Команда принята")
```

После каждой попытки переменная `command` получает новое значение. Как только оно входит в набор допустимых вариантов, условие становится ложным.

`strip()` и `lower()` должны применяться к каждой новой строке пользователя, поэтому находятся внутри цикла.

### Соберите цикл проверки команды

Расположите шаги так, чтобы программа повторяла ввод и затем сообщала об успехе.

**Фрагменты**

- `command = ""`
- `while command not in ("add", "list", "exit"):`
- `command = input("Команда: ").strip().lower()`
- `print("Команда принята")`
- `command = "exit"` — завершит цикл до первого ввода

**Правильный порядок**

```python
command = ""
while command not in ("add", "list", "exit"):
    command = input("Команда: ").strip().lower()
print("Команда принята")
```

Начальное значение недопустимо, ввод обновляет состояние, а сообщение находится после цикла.

## 05. break завершает ближайший цикл

`break` немедленно прекращает ближайший цикл. Строки ниже в текущей итерации уже не выполняются.

```python
while True:
    command = input("Команда: ").strip().lower()

    if command == "exit":
        break

    print(f"Выполняем: {command}")

print("Работа завершена")
```

### Предскажите результат

```python
number = 1
while True:
    print(number)
    if number == 3:
        break
    number += 1
```

*Подсказка: После break цикл прекращается немедленно.*

**Ответ**

```text
1
2
3
```

> `while True` допустим, когда условие выхода короткое, заметное и гарантированно может стать истинным.

## 06. continue пропускает остаток итерации

`continue` не завершает цикл. Оно сразу переходит к следующей проверке условия.

```python
number = 0
while number < 5:
    number += 1
    if number == 3:
        continue
    print(number)
```

**Пошаговый разбор**

1. Начальное значение number = 0. **Состояние:** number = 0
2. На первой итерации number становится 1. **Состояние:** number = 1
3. 1 печатается. **Состояние:** вывод = 1
4. На третьей итерации number становится 3. **Состояние:** number = 3
5. continue пропускает print для значения 3. **Состояние:** number = 3; вывод = 1 ⏎ 2
6. Позже печатаются 4 и 5. **Состояние:** вывод = 1 ⏎ 2 ⏎ 4 ⏎ 5

### Разбор ошибки

```python
number = 0
while number < 5:
    if number == 3:
        continue
    number += 1
```

**Вопрос:** Почему цикл может застрять на значении 3?

A. continue запрещён внутри while
B. Обновление number находится после continue и перестаёт выполняться
C. Условие number < 5 неверно

**Ответ:** B. Обновление number находится после continue и перестаёт выполняться

Когда number равен 3, выполнение каждый раз переходит к новой итерации до увеличения.

**Исправленный вариант**

```python
number = 0
while number < 5:
    number += 1
    if number == 3:
        continue
```

## 07. Выбор цикла и практика StudyHub

Используйте `for` для готовой последовательности, а `while` — когда повтор зависит от состояния или неизвестного количества попыток.

### Готовый набор — for

Обрабатывается каждый существующий элемент.

```python
for task in tasks:
    print(task)
```

### Состояние — while

Повтор продолжается, пока состояние удовлетворяет условию.

```python
while command != "exit":
    ...
```

**подсчёт цифр без преобразования в строку**

```python
number = 1250
digits = 0

while number > 0:
    number //= 10
    digits += 1

print(digits)  # 4
```

Деление на `10` последовательно превращает `1250 → 125 → 12 → 1 → 0`.

Для исходного `0` нужен отдельный граничный результат: тело цикла не выполнится ни разу, но число всё равно содержит одну цифру.

> В задаче на поиск первого допустимого значения вручную увеличивайте индекс и сначала проверяйте, не вышел ли он за границу последовательности.

## 08. Проверь себя

### Вопрос 1

Когда выполняется while?

A. пока условие истинно
B. ровно один раз
C. только для строк

**Ответ:** A. пока условие истинно

Условие проверяется перед каждой итерацией.

### Вопрос 2

Что делает цикл конечным?

A. print
B. изменение состояния или break
C. комментарий

**Ответ:** B. изменение состояния или break

Условие должно стать ложным либо должен выполниться выход.

### Вопрос 3

Что делает break?

A. пропускает итерацию
B. завершает ближайший цикл
C. возвращает значение

**Ответ:** B. завершает ближайший цикл

break немедленно прекращает цикл.

### Вопрос 4

Что делает continue?

A. завершает цикл
B. пропускает остаток итерации
C. удаляет условие

**Ответ:** B. пропускает остаток итерации

Следующее выполнение начинается с новой проверки условия.

### Вопрос 5

Какой цикл подходит для меню до команды exit?

A. while
B. for по символам
C. цикл не нужен

**Ответ:** A. while

Количество команд заранее неизвестно и зависит от состояния.

### Главное после занятия

- `while` проверяет условие перед каждой итерацией.
- У цикла есть состояние, условие и изменение.
- Без обновления состояния возможен бесконечный цикл.
- `break` завершает цикл, `continue` пропускает остаток шага.
- `for` удобен для набора, `while` — для состояния.
