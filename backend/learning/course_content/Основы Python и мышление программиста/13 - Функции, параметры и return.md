# Функции, параметры и return

Научимся превращать повторяемые правила в функции: разделим определение и вызов, передадим аргументы в параметры, вернём результат через return и разберём локальные переменные.

**Темы:** def · вызов · return · параметры и контракт

## 01. Маршрут занятия

Функция — не просто способ сократить код. Она даёт действию имя, задаёт вход и выход и позволяет проверить правило отдельно от остального проекта.

1. **Определить инструмент:** создать функцию через `def`, не выполняя её тело сразу.
2. **Передать данные:** сопоставить аргументы вызова с параметрами функции.
3. **Получить результат:** вернуть значение через `return` и использовать его дальше.

Дополнительно вы разберёте вызов без параметров, порядок аргументов, ранний `return`, значение `None`, локальные переменные и одну ответственность функции.

> Рабочий вопрос: **что функция получает, что делает и что возвращает?**

## 02. Определение и вызов — разные моменты

`def` создаёт функцию, но тело начинает выполняться только при вызове со скобками.

```python
def greet(name):
    return f"Привет, {name}!"

message = greet("Анна")
print(message)
```

**Пошаговый разбор**

1. Python создаёт функцию greet и связывает её с именем. Тело пока не выполняется. **Состояние:** greet = функция
2. Вызов greet("Анна") передаёт значение "Анна" в параметр name. **Состояние:** name = "Анна"
3. Функция создаёт строку и возвращает её месту вызова. **Состояние:** результат = "Привет, Анна!"
4. Возвращённое значение сохраняется в message. **Состояние:** message = "Привет, Анна!"
5. print показывает сохранённый результат. **Состояние:** вывод = Привет, Анна!

### Сравнение решений

**Вопрос:** Что окажется в переменной result?

### Без вызова

```python
result = greet
```

Сохраняется сама функция.

### С вызовом

```python
result = greet("Анна")
```

Сохраняется возвращённая строка.

**Вывод:** предпочтителен правый вариант. Скобки означают выполнить функцию сейчас.

### Верно или неверно

Функция без параметров вызывается без скобок: `app_name`.

**Ответ:** неверно. Даже без параметров вызов записывается как app_name().

## 03. Параметры и аргументы

Параметры — имена входных значений в определении. Аргументы — конкретные значения каждого вызова.

**параметры и аргументы**

```python
def format_task(title, priority):
    return f"[{priority}] {title}"

message = format_task("SQL", 5)
print(message)
```

### Предскажите результат

```python
def subtract(total, completed):
    return total - completed

print(subtract(10, 3))
print(subtract(3, 10))
```

*Подсказка: Позиционные аргументы сопоставляются слева направо.*

**Ответ**

```text
7
-7
```

### Сопоставление

Соедините элемент записи с его ролью.

| Фрагмент | Роль |
| --- | --- |
| title в def format_task(title, priority) | параметр |
| "SQL" в format_task("SQL", 5) | аргумент |
| format_task(...) | вызов функции |
| message = | сохранение результата |

Параметр существует внутри вызова, аргумент предоставляет ему конкретное значение.

### Разбор ошибки

```python
def format_task(title, priority):
    return f"[{priority}] {title}"

print(format_task("SQL"))
```

**Вопрос:** Почему возникнет TypeError?

A. Строка SQL слишком короткая
B. Параметр priority не получил аргумент
C. Функции нельзя вызывать внутри print

**Ответ:** B. Параметр priority не получил аргумент

Определение требует два входных значения, а вызов передал только одно.

**Исправленный вариант**

```python
print(format_task("SQL", 5))
```

## 04. return передаёт значение и завершает вызов

`return` возвращает значение вызывающему коду. После первого сработавшего `return` остальные строки текущего вызова не выполняются.

```python
def normalize_title(title):
    cleaned = title.strip()
    if cleaned == "":
        return "Без названия"
    return cleaned

print(normalize_title("   "))
print(normalize_title("  SQL  "))
```

**Пошаговый разбор**

1. Первый вызов получает строку из пробелов. **Состояние:** title = " "
2. strip создаёт пустую строку. **Состояние:** cleaned = ""
3. Условие истинно, возвращается запасной текст. Нижний return пропускается. **Состояние:** результат = "Без названия"
4. Второй вызов начинается с новыми локальными значениями. **Состояние:** title = " SQL "
5. cleaned становится SQL. **Состояние:** cleaned = "SQL"
6. Условие ложно, выполняется итоговый return. **Состояние:** результат = "SQL"

### Соберите функцию проверки заголовка

Расположите строки так, чтобы функция возвращала bool.

**Фрагменты**

- `def is_valid_title(title):`
- `cleaned = title.strip()`
- `return cleaned!= ""`
- `print(cleaned)` — показывает значение, но не возвращает bool

**Правильный порядок**

```python
def is_valid_title(title):
    cleaned = title.strip()
    return cleaned != ""
```

Функция получает строку, очищает её и возвращает результат сравнения.

### Вспомните модель

**Вопрос:** Что происходит с кодом после сработавшего return?

**Ответ:** Текущий вызов функции завершается, поэтому следующие строки этого вызова не выполняются.

## 05. print и return решают разные задачи

`print` показывает данные человеку, а `return` передаёт результат программе.

### Сравнение решений

**Вопрос:** Какой вариант подходит для функции расчёта?

### Только печать

```python
def add(a, b):
    print(a + b)

result = add(2, 3)
```

Человек увидит 5, но result станет None.

### Возврат результата

```text
def add(a, b):
    return a + b

result = add(2, 3)
```

result получает число 5 и может использовать его дальше.

**Вывод:** предпочтителен правый вариант. Платформа сравнивает возвращённое значение solve, а не текст в терминале.

### Предскажите результат

```python
def wrong_add(a, b):
    print(a + b)

result = wrong_add(2, 3)
print(result)
```

*Подсказка: Функция без сработавшего return возвращает None.*

**Ответ**

```text
5
None
```

### Разбор ошибки

```python
def calculate_left(total, completed):
    left = total - completed
    print(left)

result = calculate_left(10, 4)
print(result * 2)
```

**Вопрос:** Почему последняя строка завершится TypeError?

A. left слишком маленькое
B. result равен None, потому что функция ничего не вернула
C. Функция не может содержать переменные

**Ответ:** B. result равен None, потому что функция ничего не вернула

Печать внутри функции не становится её возвращаемым значением.

**Исправленный вариант**

```python
def calculate_left(total, completed):
    return total - completed

result = calculate_left(10, 4)
print(result * 2)
```

> В заданиях платформы не заменяйте `return` выводом. Даже визуально правильное число не будет результатом функции.

## 06. Локальные переменные и явные зависимости

Параметры и переменные, созданные внутри функции, относятся к конкретному вызову и недоступны снаружи.

**локальное значение**

```python
def calculate_left(total, completed):
    left = total - completed
    return left

result = calculate_left(10, 4)
print(result)
# print(left)  # NameError
```

### Сравнение решений

**Вопрос:** Как сделать зависимость функции видимой?

### Скрытая внешняя переменная

```python
priority = 5

def format_task(title):
    return f"[{priority}] {title}"
```

По параметрам не видно, откуда берётся priority.

### Явный параметр

```python
def format_task(title, priority):
    return f"[{priority}] {title}"
```

Все необходимые данные передаются в вызов.

**Вывод:** предпочтителен правый вариант. Явные зависимости легче проверять и переиспользовать.

### Верно или неверно

Две разные функции могут иметь локальную переменную с именем `result`, не мешая друг другу.

**Ответ:** верно. Локальные области разных вызовов не являются одной общей переменной.

## 07. Контракт и одна ответственность

Хорошая функция отвечает на один основной вопрос. Её контракт описывает вход, возвращаемое значение, изменение аргументов и граничные случаи.

### Предметное имя — ясно

По имени понятно, что функция возвращает boolean-ответ.

```python
def is_valid_title(title):
    return title.strip() != ""
```

### Несколько ответственностей — слишком широко

Название перечисляет независимые действия и сигнализирует о перегруженной функции.

```python
def add_validate_print_and_save_task(...):
    ...
```

**StudyHub: три небольших правила**

```python
def normalize_title(title):
    return title.strip()

def is_valid_title(title):
    return normalize_title(title) != ""

def format_task(title, priority):
    clean_title = normalize_title(title)
    return f"[{priority}] {clean_title}"
```

### Вспомните модель

**Вопрос:** Какие четыре вопроса описывают контракт функции?

*Подсказка: Вход, выход, изменение и границы.*

**Ответ:** Что функция принимает, что возвращает, меняет ли переданные объекты и как обрабатывает пустые или граничные значения.

> Не превращайте каждую строку в функцию. Выделение оправдано, если правило имеет имя, повторяется, отдельно проверяется или делает основной сценарий понятнее.

## 08. Проверь себя

### Вопрос 1

Когда выполняется тело функции?

A. при чтении def
B. при вызове со скобками
C. только внутри print

**Ответ:** B. при вызове со скобками

def создаёт функцию, а вызов запускает её тело.

### Вопрос 2

Что является параметром?

A. имя во входной части def
B. конкретное значение вызова
C. результат print

**Ответ:** A. имя во входной части def

Параметр получает аргумент на время вызова.

### Вопрос 3

Что делает return?

A. показывает текст
B. возвращает значение и завершает вызов
C. создаёт глобальное имя

**Ответ:** B. возвращает значение и завершает вызов

После сработавшего return код ниже в этом вызове не выполняется.

### Вопрос 4

Что вернёт функция без return?

A. 0
B. None
C. пустой список

**Ответ:** B. None

Неявное возвращаемое значение — None.

### Вопрос 5

Почему print не заменяет return в solve?

A. платформа сравнивает возвращённое значение
B. print медленнее
C. print запрещён в Python

**Ответ:** A. платформа сравнивает возвращённое значение

Вывод предназначен человеку, return — вызывающему коду.

### Главное после занятия

- `def` определяет функцию, но не вызывает её.
- Вызов требует скобок.
- Параметр — имя в определении, аргумент — значение вызова.
- Позиционные аргументы сопоставляются слева направо.
- `return` отдаёт значение и завершает вызов.
- Без return функция возвращает `None`.
- Локальные переменные существуют внутри вызова.
- Хорошая функция имеет ясный контракт и одну основную ответственность.
