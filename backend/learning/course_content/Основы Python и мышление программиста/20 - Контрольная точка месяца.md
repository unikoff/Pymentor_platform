# Контрольная точка месяца

Проведём практическую защиту Console StudyHub Planner: запустим проект, объясним путь данных, проверим границы и внесём небольшое изменение без переписывания всей программы.

**Темы:** защита проекта · граничные случаи

## 01. Как пройдёт защита проекта

Контрольная точка похожа не на тест по определениям, а на практический экзамен по вождению. Недостаточно назвать правила: нужно запустить проект, объяснить свои действия, пройти неудобный сценарий и безопасно внести небольшую правку.

1. **Запустить и продемонстрировать:** пройти пользовательский сценарий по README.
2. **Объяснить устройство:** проследить данные от `input()` до словаря в списке.
3. **Изменить и проверить:** найти ответственную функцию, внести минимальную правку и повторить старые сценарии.

В конце вы покажете чистый `git status`, историю коммитов и честное ограничение хранения данных только в памяти.

> На защите важнее самостоятельно объяснить и изменить знакомый проект, чем добавить технологию, которую ученик пока не понимает.

## 02. Минимальный готовый результат

Для защиты не нужны дополнительные технологии ради впечатления. Достаточно законченного консольного проекта, который запускается по README, хранит задачи в памяти, устойчиво обрабатывает ввод и имеет понятную Git-историю.

**минимальная структура**

```text
studyhub-planner/
├── .gitignore
├── README.md
└── main.py
```

Показывает команды, проверяет ввод и завершает приложение.

```python

```

### Задачи — данные

Список словарей хранится в памяти процесса.

```python

```

### Git и README — история

Репозиторий объясняет запуск и развитие проекта.

```powershell

```

Соедините требование с доказательством.

| Левая часть | Правая часть |

## 05. Пройдите условия и циклы вручную

Трассировка похожа на просмотр записи с камер по кадрам. На каждом шаге называются текущие значения, результат условия, выбранная ветка и изменение накопителя. Итог не угадывается заранее.

```python
priority = 4
if priority == 5:
    label = "critical"
elif priority >= 3:
    label = "normal"
else:
    label = "low"
```

**Пошаговый разбор**

1. priority равен 4. **Состояние:** priority = 4
2. 4 == 5 ложно. **Состояние:** условие = False
3. 4 >= 3 истинно. **Состояние:** условие = True
4. Выбирается label = "normal". **Состояние:** label = "normal"
5. Остальные ветки цепочки не проверяются. **Состояние:** label = "normal"

### Предскажите результат

```python
tasks = [
    {"is_done": True},
    {"is_done": False},
    {"is_done": True},
]
completed = 0
for task in tasks:
    if task.get("is_done", False):
        completed += 1
print(completed)
```

*Подсказка: Накопитель увеличивается только для True.*

**Ответ**

```text
2
```

### Сопоставление

Соедините элемент цикла меню с его ролью.

| Левая часть | Правая часть |
| --- | --- |
| while True | начинает повтор интерфейса |
| continue | возвращает к меню после неверного ввода |
| break | завершает приложение по команде выхода |
| tasks | сохраняет состояние между итерациями |

У каждого элемента управляющего цикла есть отдельная роль.

## 06. Проверьте неудобные сценарии

Рабочий пример показывает только удобную дорогу. Надёжность проверяют на неудобных случаях: пустой список, пробелы вместо названия, значения рядом с границей, неизвестная команда и отсутствующий id.

### Нет задач — пусто

Выводится понятное сообщение, статистика не делит на ноль.

```python
show_tasks([])
```

### Приоритет — границы

Значения 1 и 5 допустимы, соседние — нет.

```python
0, 1, 5, 6
```

### Отсутствующий id — не найдено

Возвращается `None`, случайная задача не меняется.

```python
find_task(tasks, 999)
```

### Разбор ошибки

```python
def completion_percent(tasks):
    completed = count_done(tasks)
    return completed / len(tasks) * 100
```

**Вопрос:** Какой обязательный сценарий не обработан?

A. Пустой список
B. Приоритет 5
C. Название в нижнем регистре

**Ответ:** A. Пустой список

Для пустого списка len(tasks) равен нулю.

**Исправленный вариант**

```python
def completion_percent(tasks):
    if not tasks:
        return 0.0
    completed = count_done(tasks)
    return round(completed / len(tasks) * 100, 1)
```

**минимальная матрица ручной проверки**

```text
пустой список
название "" и "   "
приоритеты 0, 1, 5, 6
неизвестная команда
отсутствующий id
0%, частичный прогресс, 100%
```
