# Проектное меню и валидация

Соберём главное окно консольного приложения: пользователь увидит понятные команды, ошибочный ввод не сломает программу, а каждый выбор будет направлен к своему действию.

**Темы:** цикл меню · валидация команды

## 01. Как будет работать меню

Представьте администратора в учебном центре. Он показывает список доступных действий, слушает выбор посетителя и направляет его в нужный кабинет. Меню StudyHub выполняет ту же роль: показывает команды, проверяет ввод и запускает только подходящий обработчик.

1. **Показать варианты:** отдельная функция `show_menu()` объясняет доступные действия.
2. **Получить и проверить ввод:** команда очищается через `strip()` до любых сравнений.
3. **Выбрать обработчик:** валидная команда направляется к одной функции, а `5` завершает цикл.

В конце занятия в `main.py` появятся `show_menu`, `is_valid_command` и `run`. Действия с задачами пока могут оставаться заглушками.

> Читайте `run()` как короткий сценарий администратора: показать варианты, получить выбор, проверить его, направить к действию или завершить приложение.

## 02. Меню как администратор приложения

Хорошее меню похоже на понятную табличку у входа. Пользователь не должен угадывать, что вводить. `show_menu()` только показывает варианты, `input()` получает выбор, а отдельная логика решает, что выполнить.

### Показать варианты — 1

Функция печатает список команд и не спрашивает ввод.

```python
show_menu()
```

### Получить команду — 2

`input()` возвращает строку, даже если введена цифра.

```python
input("Выберите действие: ")
```

### Выбрать действие — 3

Команда направляется к отдельному обработчику.

```python
if command == "1":
    handle_add(tasks)
```

**отдельная функция отображения**

```python
def show_menu():
    print()
    print("StudyHub Planner")
    print("1. Добавить задачу")
    print("2. Показать задачи")
    print("3. Изменить статус")
    print("4. Удалить задачу")
    print("5. Выйти")
```

### Вспомните модель

**Вопрос:** Почему команды меню удобно сравнивать со строками, а не сразу преобразовывать в int?

*Подсказка: Вспомните тип результата input() и ввод вроде add или пустой строки.*

**Ответ:** `input()` уже возвращает строку. Сравнение с `"1"` – `"5"` не требует преобразования и не создаёт дополнительный `ValueError` .

## 03. Повтор меню и понятная кнопка выхода

Консольное приложение похоже на стойку обслуживания, которая снова предлагает варианты после каждого действия. Цикл повторяет меню, а команда выхода работает как понятная кнопка «закрыть»: после неё ближайший цикл завершается через `break`.

```python
def run():
    while True:
        show_menu()
        command = input("Команда: ").strip()
        if command == "5":
            print("До свидания")
            break
    print("Приложение завершено")
```

**Пошаговый разбор**

1. Начинается очередная итерация меню. **Состояние:** цикл = активен
2. Пользователь видит доступные действия. **Состояние:** меню = показано
3. После ввода "5" переменная command содержит очищенную строку. **Состояние:** command = "5"
4. Проверка command == "5" истинна. **Состояние:** условие = True
5. break немедленно завершает ближайший цикл. **Состояние:** цикл = завершён
6. Строка после while выполняется один раз. **Состояние:** вывод = Приложение завершено

### Верно или неверно

Любой `while True` является ошибкой, даже если внутри есть понятный `break`.

**Ответ:** неверно. Бесконечное условие допустимо для меню, если выход видим и достижим.

> Не прячьте единственный `break` глубоко внутри нескольких вложенных условий: маршрут завершения должен читаться без долгой трассировки.

## 04. Сначала привести ввод в порядок

Ввод пользователя похож на заполненный от руки номер талона: вокруг цифры могут оказаться случайные пробелы. Сначала строку приводят к согласованному виду через `strip()`, затем проверяют, входит ли команда в разрешённый набор.

**чистая проверка команды**

```python
VALID_COMMANDS = {"1", "2", "3", "4", "5"}

def is_valid_command(command):
    cleaned = command.strip()
    return cleaned in VALID_COMMANDS
```

### Предскажите результат

```python
print(is_valid_command(" 3 "))
print(is_valid_command("add"))
print(is_valid_command(""))
print(is_valid_command("5"))
```

*Подсказка: strip() удаляет края, после чего in проверяет членство в множестве.*

**Ответ**

```text
True
False
False
True
```

### Сравнение решений

**Вопрос:** Где лучше один раз очистить пользовательскую команду?

### Повторять в каждой ветке

```python
if command.strip() == "1":
    ...
elif command.strip() == "2":
    ...
```

Одна операция повторяется и легко забывается в новой ветке.

### Нормализовать до проверок

```python
command = input("Команда: ").strip()
if command == "1":
    ...
```

Все дальнейшие ветки работают с одним согласованным значением.

**Вывод:** предпочтителен правый вариант. Нормализация относится к границе ввода и выполняется до маршрутизации.

## 05. Ошибочная команда не должна запускать действие

Если посетитель назвал неизвестную команду, администратор не должен отправлять его в случайный кабинет. Программа сообщает об ошибке и через `continue` возвращается к следующему показу меню, не запуская обработчик.

### Разбор ошибки

```python
command = input("Команда: ").strip()
if not is_valid_command(command):
    print("Неизвестная команда")

handle_command(command)
```

**Вопрос:** Почему структура опасна?

A. После сообщения программа всё равно вызывает handle_command
B. strip нельзя использовать с input
C. В условии запрещён not

**Ответ:** A. После сообщения программа всё равно вызывает handle_command

После невалидного ввода нужно прекратить текущую итерацию.

**Исправленный вариант**

```python
command = input("Команда: ").strip()
if not is_valid_command(command):
    print("Неизвестная команда")
    continue

handle_command(command)
```

### Соберите одну итерацию меню

Расположите действия в безопасном порядке.

**Фрагменты**

- `show_menu()`
- `command = input("Команда: ").strip()`
- `if not is_valid_command(command):`
- `print("Неизвестная команда") continue`
- `handle_command(command)`

**Правильный порядок**

```python
show_menu()
command = input("Команда: ").strip()
if not is_valid_command(command):
    print("Неизвестная команда")
    continue
handle_command(command)
```

Сначала пользователь видит варианты, затем ввод проверяется и только после этого выполняется действие.

> Валидация после обработчика слишком поздняя: неверные данные уже могли изменить состояние приложения.

## 06. Куда направлять каждую команду

Главный цикл похож на диспетчера. Он не выполняет всю работу самостоятельно, а выбирает специалиста: добавление, вывод, изменение статуса или удаление. Такие отдельные функции называются обработчиками.

### Разбор ветвей

```python
if command == "1":
    handle_add(tasks)
elif command == "2":
    handle_list(tasks)
elif command == "3":
    handle_done(tasks)
elif command == "4":
    handle_delete(tasks)
else:
    break
```

| Сценарий | Выполняется | Результат |
| --- | --- | --- |
| command = "1" | строка 1 | handle_add(tasks) |
| command = "3" | строка 5 | handle_done(tasks) |
| command = "5" | строка 9 | выход из цикла |

### Управляет сценарием — run

Показывает меню, получает команду и выбирает обработчик.

```python
run()
```

### Работает с интерфейсом — handler

Может запросить данные и показать сообщение.

```python
handle_add(tasks)
```

### Реализует правило — rule

Создаёт или проверяет данные без знания о меню.

```python
create_task(...)
```

> Передача `tasks` аргументом делает зависимость обработчика явной. Глобальный список скрывает эту связь и усложняет тестирование.

## 07. Один принцип для проекта и платформы

В локальном проекте команда приходит от `input()`, а на платформе — через аргумент `solve(command)`. Это два разных способа доставки одной строки. Само правило проверки команды остаётся одинаковым.

### Сравнение решений

**Вопрос:** Как использовать одно правило в двух средах?

### Локальный проект

```python
command = input("Команда: ")
if is_valid_command(command):
    ...
```

Интерфейс получает строку от пользователя.

### Автопроверка

```python
def solve(command):
    return is_valid_command(command)
```

Платформа передаёт строку как аргумент.

**Вывод:** оба варианта допустимы. Правило одно, различается только источник входного значения.

**каркас main.py**

```python
VALID_COMMANDS = {"1", "2", "3", "4", "5"}

def show_menu():
    ...

def is_valid_command(command):
    return command.strip() in VALID_COMMANDS

def run():
    tasks = []
    while True:
        show_menu()
        command = input("Команда: ").strip()
        if not is_valid_command(command):
            print("Неизвестная команда")
            continue
        if command == "5":
            break
```

### Вспомните модель

**Вопрос:** Какие пять ручных сценариев нужно проверить до коммита?

**Ответ:** Валидную команду, команду с пробелами, неизвестное число, пустую строку и команду выхода. После каждой команды нужно проверить, повторилось ли меню или завершилось приложение.

## 08. Проверьте основную модель

### Вопрос 1

Какова главная задача меню?

A. хранить данные
B. связать выбор пользователя с действием
C. заменить функции

**Ответ:** B. связать выбор пользователя с действием

Меню является интерфейсом и маршрутизатором команд.

### Вопрос 2

Почему команда " 1 " может быть допустимой?

A. strip удаляет пробелы по краям
B. Python игнорирует пробелы в строке
C. множество меняет строку

**Ответ:** A. strip удаляет пробелы по краям

После strip значение становится строкой 1.

### Вопрос 3

Что делает continue после неверной команды?

A. завершает программу
B. начинает следующую итерацию
C. возвращает список

**Ответ:** B. начинает следующую итерацию

Оставшаяся часть текущей итерации пропускается.

### Вопрос 4

Почему tasks передаётся обработчику?

A. зависимость становится явной
B. иначе список неизменяем
C. так требует print

**Ответ:** A. зависимость становится явной

Функция явно получает объект, с которым работает.

### Главное после занятия

- Меню состоит из отображения, ввода, проверки и маршрутизации.
- Ввод нормализуется до сравнений.
- `continue` защищает обработчики от неверной команды.
- `break` создаёт видимую точку выхода.
- Состояние передаётся обработчикам аргументом.

### Практика

Соберите main.py с устойчивым меню и реализуйте solve(command) для проверки команды.
