# Переменные и базовые типы

Научимся давать данным понятные имена, менять значения и различать четыре основных типа: текст, целые числа, дробные числа и ответы да или нет.

**Темы:** имя и значение · str · int · float · bool

## 01. Зачем данным нужны имена

Представьте несколько одинаковых коробок без подписей. Пока их две, можно помнить содержимое. Когда коробок становится двадцать, начинаются ошибки. Переменная — понятная подпись к значению.

1. **Создать переменную:** дать значению имя с помощью знака `=`.
2. **Прочитать значение:** передать имя в `print()` без кавычек.
3. **Выбрать подходящий тип:** хранить текст, число или логический ответ по смыслу.

В конце в StudyHub появятся название проекта, номер версии, процент прогресса и признак готовности.

### Сравнение решений

**Вопрос:** Какой вариант проще изменить?

### Повторять текст

```python
print("StudyHub")
print("StudyHub")
print("StudyHub")
```

Название записано в трёх местах.

### Дать тексту имя

```python
project_name = "StudyHub"
print(project_name)
print(project_name)
```

Название задаётся один раз.

**Вывод:** предпочтителен правый вариант. Переменная уменьшает повторение и сразу показывает смысл данных.

## 02. Переменная как коробка с наклейкой

Для первого знакомства удобно думать так: справа находится предмет, слева — наклейка на коробке. Строка `task_count = 3` кладёт число 3 в коробку с названием task_count.

```python
project_name = "StudyHub"
print(project_name)
```

**Пошаговый разбор**

1. Создаётся имя project_name и связывается с текстом "StudyHub". **Состояние:** project_name = "StudyHub"
2. print получает имя без кавычек, находит его значение и показывает StudyHub. **Состояние:** вывод = StudyHub

### Что означает знак =

Здесь это не школьное «левая часть навсегда равна правой». Знак `=` означает: «сохрани значение справа под именем слева».

### Почему присваивание ничего не печатает

Положить предмет в коробку — не то же самое, что показать его человеку. Для вывода нужна отдельная команда `print(project_name)`.

### Имя и текст имени — разные вещи

`print(project_name)` показывает значение. `print("project_name")` показывает буквальный текст project_name.

### Разбор ошибки

```python
project_name = "StudyHub"
print("project_name")
```

**Вопрос:** Почему выводится project_name, а не StudyHub?

A. Имя записано внутри кавычек и стало обычным текстом
B. Переменные нельзя выводить
C. Нужна точка с запятой

**Ответ:** A. Имя записано внутри кавычек и стало обычным текстом

Чтобы получить значение переменной, имя передают в print без кавычек.

**Исправленный вариант**

```python
project_name = "StudyHub"
print(project_name)
```

## 03. Сначала создать, затем использовать

Python выполняет файл сверху вниз. Нельзя попросить взять предмет из коробки, которую ещё не поставили на стол.

### Сравнение решений

**Вопрос:** Какой порядок правильный?

### Сначала использование

```python
print(course_name)
course_name = "Python"
```

На первой строке имя ещё не создано.

### Сначала значение

```python
course_name = "Python"
print(course_name)
```

Имя существует до первого использования.

**Вывод:** предпочтителен правый вариант. Python должен встретить присваивание раньше, чем имя понадобится.

### Соберите правильный порядок

Сначала создайте две переменные, затем выведите их.

**Фрагменты**

- `project_name = "StudyHub"`
- `task_count = 3`
- `print(project_name)`
- `print(task_count)`

**Правильный порядок**

```python
project_name = "StudyHub"
task_count = 3
print(project_name)
print(task_count)
```

Оба имени появляются до использования.

Регистр важен: `project_name` и `Project_Name` — разные подписи.

Опечатка в имени похожа на попытку открыть соседний шкаф: Python не находит нужное значение и сообщает `NameError`.

## 04. Значение можно заменить

Переменная хранит текущее значение. Если в коробку положить новый предмет, при следующем обращении мы получим уже его.

```python
status = "new"
print(status)
status = "done"
print(status)
```

**Пошаговый разбор**

1. Сначала status хранит "new". **Состояние:** status = "new"
2. Первый print показывает new. **Состояние:** вывод = new
3. Новое присваивание заменяет текущее значение на "done". **Состояние:** status = "done"
4. Второй print показывает done. **Состояние:** вывод = new ⏎ done

### Старое значение не печатается автоматически

Python не создаёт журнал каждого присваивания. Обычная переменная сообщает только своё текущее значение.

### Справа используется текущее значение

В строке `task_count = task_count + 1` Python сначала берёт старое число, прибавляет единицу и только затем сохраняет новый результат.

### Предскажите результат

```python
task_count = 2
task_count = task_count + 1
print(task_count)
```

*Подсказка: Сначала справа вычисляется 2 + 1, затем task_count получает новое значение.*

**Ответ**

```text
3
```

## 05. Четыре базовых типа данных

Тип можно сравнить с формой контейнера. В бутылке удобно хранить воду, в коробке — обувь. Python тоже должен понимать, что перед ним: текст, целое число, дробь или ответ да/нет.

### Текст — str

Названия, сообщения, адреса и любые данные, которые нужно хранить как текст.

```python
project_name = "StudyHub"
```

### Целое число — int

Количество задач, номер версии или приоритет без дробной части.

```python
task_count = 5
```

### Дробное число — float

Проценты, вес, время или другая величина с дробной частью.

```python
progress = 62.5
```

### Да или нет — bool

Логический ответ: `True` или `False`.

```python
is_ready = True
```

`5` — число, а `"5"` — текст. Кавычки меняют смысл значения.

Дробное число записывается с точкой: `62.5`. Запятая в Python не является десятичным разделителем одного числа.

`True` и `False` пишутся без кавычек и с заглавной буквы. В кавычках это будет обычный текст.

> Номер телефона и почтовый индекс часто хранят как текст: их не складывают, а ведущий ноль может быть важен.

## 06. Как выбирать понятные имена

Хорошее имя похоже на подпись на банке. Надпись «соль» помогает сразу понять содержимое, а надпись «x» заставляет открывать банку и проверять.

### Сравнение решений

**Вопрос:** Какое имя понятнее?

### Смысл скрыт

```python
x = 5
```

Непонятно, что означает число.

### Смысл виден

```python
task_count = 5
```

Имя сообщает, что хранится количество задач.

**Вывод:** предпочтителен правый вариант. Понятное имя экономит время при чтении и отладке.

### Стиль snake_case

В Python слова в обычных именах соединяют подчёркиванием: `project_name`, `task_count`, `is_ready`.

### Формальные правила

Имя не начинается с цифры, не содержит пробел и дефис. Подходят буквы, цифры после первого символа и подчёркивание.

### Логическое имя читается как вопрос

`is_ready` читается как «готово ли?», поэтому значения True и False выглядят естественно.

### Заполните пропуск

Соедините два слова в стиле snake_case.

```python
completed___tasks = 4
```

**Варианты:** `_`, `-`, ``

**Ответ:** `_`

В обычном имени слова соединяются подчёркиванием.

## 08. Проверьте основную модель

Вопросы проверяют только базовые действия: создать имя, отличить имя от текста и выбрать тип по смыслу.

### Вопрос 1

Что означает знак = в task_count = 3?

A. Сохранить значение 3 под именем task_count
B. Сравнить два числа
C. Напечатать 3

**Ответ:** A. Сохранить значение 3 под именем task_count

Один знак = выполняет присваивание.

### Вопрос 2

Что выведет print("project_name")?

A. Текст project_name
B. Значение переменной
C. Ошибка

**Ответ:** A. Текст project_name

Кавычки превращают запись в обычный текст.

### Вопрос 3

Какой тип лучше подходит для количества задач?

A. int
B. str
C. bool

**Ответ:** A. int

Количество без дробной части обычно хранится как целое число.

### Вопрос 4

Какое имя подходит для ответа «готов ли проект»?

A. is_ready
B. value
C. x

**Ответ:** A. is_ready

Имя is_ready читается как вопрос с ответом True или False.

### Главное после занятия

- Переменная — понятное имя для текущего значения.
- Знак `=` сохраняет значение справа под именем слева.
- Имя нужно создать до первого использования.
- Имя без кавычек означает переменную, а имя в кавычках — обычный текст.
- `str` хранит текст, `int` — целые, `float` — дробные, `bool` — True или False.
- Понятные имена в snake_case снижают количество догадок.
