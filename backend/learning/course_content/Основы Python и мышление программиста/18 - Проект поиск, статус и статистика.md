# Проект: поиск, статус и статистика

Научим StudyHub находить задачу, безопасно менять её статус и считать прогресс, не смешивая поиск, изменение данных и вывод сообщений.

**Темы:** поиск и None · статистика

## 01. Поиск, изменение и подсчёт — разные шаги

Работа с задачей напоминает обращение в архив. Сначала нужно найти нужную карточку, затем отдельно изменить её, а сводный отчёт строится ещё одной функцией. Если смешать эти действия, код станет труднее проверять.

1. **Найти задачу:** пройти список и вернуть словарь либо `None`.
2. **Изменить безопасно:** проверить результат поиска и только затем обновить `is_done`.
3. **Посчитать прогресс:** отдельно получить количество выполненных задач и процент.

Дополнительно вы разберёте тип id, ранний `return`, пустой список и поиск по фрагменту названия.

> Для каждой функции заранее проговорите два результата: что она вернёт при успехе и что вернёт, если подходящей задачи нет.

## 02. Ищем карточку по номеру

Список похож на папку без отдельного алфавитного указателя. Чтобы найти карточку с нужным id, функция просматривает задачи по одной и завершает поиск сразу после совпадения.

```python
tasks = [
    {"id": 1, "title": "Python"},
    {"id": 2, "title": "SQL"},
    {"id": 3, "title": "Git"},
]
task_id = 2
for task in tasks:
    if task["id"] == task_id:
        result = task
        break
```

**Пошаговый разбор**

1. Искомый id равен 2. **Состояние:** task_id = 2
2. Первая итерация получает задачу с id 1. **Состояние:** task['id'] = 1
3. 1 == 2 ложно, цикл идёт дальше. **Состояние:** условие = False
4. Вторая итерация получает задачу с id 2. **Состояние:** task['id'] = 2
5. 2 == 2 истинно. **Состояние:** условие = True
6. Найденный словарь сохраняется как результат. **Состояние:** result['title'] = "SQL"

**рабочий контракт поиска**

```python
def find_task(tasks, task_id):
    for task in tasks:
        if task["id"] == task_id:
            return task

    return None
```

### Разбор ошибки

```python
def find_task(tasks, task_id):
    for task in tasks:
        if task["id"] == task_id:
            return task
        return None
```

**Вопрос:** Почему функция проверяет только первый элемент?

A. return None находится внутри цикла
B. Словари нельзя сравнивать по id
C. Нужен while вместо for

**Ответ:** A. return None находится внутри цикла

После первой неудачной проверки функция немедленно завершается.

**Исправленный вариант**

```python
def find_task(tasks, task_id):
    for task in tasks:
        if task["id"] == task_id:
            return task
    return None
```

## 03. Что вернуть, если карточка не найдена

Архивариус может вернуть найденную карточку или прийти с пустыми руками. В Python явное отсутствие объекта обозначается значением `None`. Оно не притворяется повреждённой задачей и требует отдельной проверки.

### Сравнение решений

**Вопрос:** Что лучше вернуть при отсутствии задачи?

### Пустой словарь

```python
return {}
```

Может быть принят за настоящую, но повреждённую задачу.

### Явное отсутствие

```python
return None
```

Контракт однозначно сообщает, что результата нет.

**Вывод:** предпочтителен правый вариант. None не маскируется под объект предметной области.

```python
task = find_task(tasks, task_id)

if task is None:
    print("Задача не найдена")
else:
    print(task["title"])
```

### Сопоставление

Соедините исход поиска с результатом.

| Левая часть | Правая часть |
| --- | --- |
| id найден | словарь задачи |
| id отсутствует | None |
| список пуст | None |

Один контракт работает для обычного и пустого списка.

> Не смешивайте в одном проекте `None`, `{}`, `False` и строку `"not found"` как четыре разных сигнала одной ситуации.

## 04. Меняем статус только найденной задачи

Нельзя ставить отметку на карточке, которой нет. Сначала `find_task()` возвращает словарь или `None`. Только найденная задача получает новое значение `is_done`.

```python
def mark_task_done(tasks, task_id):
    task = find_task(tasks, task_id)
    if task is None:
        return False
    task["is_done"] = True
    return True
```

**Пошаговый разбор**

1. Поиск возвращает словарь или None. **Состояние:** task = словарь / None
2. Отсутствие проверяется до доступа к полям. **Состояние:** проверка = task is None
3. Если задачи нет, функция сообщает False. **Состояние:** результат = False
4. Для найденного словаря поле меняется на True. **Состояние:** task['is_done'] = True
5. True сообщает интерфейсу об успешном изменении. **Состояние:** результат = True

### Верно или неверно

Словарь, возвращённый `find_task()`, является независимой копией и его изменение не влияет на список.

**Ответ:** неверно. Функция возвращает тот же словарь, который находится внутри списка.

### Сравнение решений

**Вопрос:** Кто должен выбирать текст сообщения?

### Бизнес-функция печатает

```python
def mark_task_done(...):
    print("Готово")
```

Функцию сложнее тестировать и переиспользовать.

### Функция возвращает результат

```text
changed = mark_task_done(tasks, task_id)
if changed:
    print("Готово")
```

Интерфейс решает, что показать пользователю.

**Вывод:** предпочтителен правый вариант. Изменение данных и отображение сообщения имеют разные ответственности.

## 05. Считаем результат без участия меню

Статистика похожа на табло: она читает текущие карточки и вычисляет количество, остаток и прогресс. Табло не должно само спрашивать пользователя, какую кнопку он нажал, поэтому функции статистики получают список аргументом и возвращают данные.

**считаем выполненные задачи**

```python
def count_done(tasks):
    completed = 0

    for task in tasks:
        if task.get("is_done", False):
            completed += 1

    return completed
```

**строим сводку**

```python
def build_statistics(tasks):
    total = len(tasks)
    completed = count_done(tasks)
    left = total - completed

    return {
        "total": total,
        "completed": completed,
        "left": left,
    }
```

### Предскажите результат

```python
tasks = [
    {"is_done": True},
    {"is_done": False},
    {},
]
completed = 0
for task in tasks:
    if task.get("is_done", False):
        completed += 1
print(completed)
```

*Подсказка: Отсутствующий is_done считается False.*

**Ответ**

```text
1
```

> Использование `get("is_done", False)` — явное правило совместимости со старыми или неполными тестовыми данными.

## 06. Что делать, когда задач пока нет

Новое приложение может пока не содержать задач. Это нормальное состояние, как пустая папка в первый день. До формулы процента нужно отдельно договориться, какой результат показывать при общем количестве `0`.

### Разбор ошибки

```python
def completion_percent(tasks):
    completed = count_done(tasks)
    return round(completed / len(tasks) * 100, 1)
```

**Вопрос:** На каком входе функция падает?

A. На пустом списке
B. Когда все задачи выполнены
C. Когда одна задача имеет priority=0

**Ответ:** A. На пустом списке

len([]) равен нулю, поэтому деление невозможно.

**Исправленный вариант**

```python
def completion_percent(tasks):
    if not tasks:
        return 0.0
    completed = count_done(tasks)
    return round(completed / len(tasks) * 100, 1)
```

### Сопоставление

Соедините состояние с процентом.

| Левая часть | Правая часть |
| --- | --- |
| пустой список | 0.0 |
| 0 из 3 выполнено | 0.0 |
| 1 из 2 выполнено | 50.0 |
| 2 из 3 выполнено | 66.7 |
| 3 из 3 выполнено | 100.0 |

Пустой список обрабатывается до формулы.

### Вспомните модель

**Вопрос:** Почему 0.0 для пустого списка — правило проекта, а не единственно возможная математика?

**Ответ:** У ситуации нет естественного деления completed / total, потому что total равен нулю. Проект выбирает удобный и документированный результат для интерфейса.

## 07. Поиск по названию и связь с меню

Поиск по id возвращает не более одной карточки. Поиск по фрагменту названия похож на фильтр каталога: совпадений может быть несколько, поэтому результатом становится новый список задач.

**поиск всех совпадений**

```python
def search_tasks(tasks, query):
    cleaned_query = query.strip().lower()
    result = []

    if cleaned_query == "":
        return result

    for task in tasks:
        if cleaned_query in task["title"].lower():
            result.append(task)

    return result
```

### Обработчик завершения задачи

Расположите шаги так, чтобы неверный id не изменил данные.

**Фрагменты**

- `task_id_text = input("ID: ").strip()`
- `if not task_id_text.isdigit(): return`
- `task_id = int(task_id_text)`
- `changed = mark_task_done(tasks, task_id)`
- `print("Готово" if changed else "Не найдено")`

**Правильный порядок**

```python
task_id_text = input("ID: ").strip()
if not task_id_text.isdigit():
    return
task_id = int(task_id_text)
changed = mark_task_done(tasks, task_id)
print("Готово" if changed else "Не найдено")
```

Строка проверяется до int(), затем бизнес-функция возвращает результат для интерфейса.

> Пустой запрос содержится в любой строке, поэтому без отдельной проверки `"" in title` вернёт `True` для всех задач.

## 08. Проверьте основную модель

### Вопрос 1

Почему поиск по id надёжнее поиска по названию?

A. id должен быть уникальным
B. строки нельзя сравнивать
C. название не хранится

**Ответ:** A. id должен быть уникальным

Одинаковые названия допустимы, одинаковые id — нет.

### Вопрос 2

Что возвращает find_task при отсутствии совпадения?

A. None
B. случайную задачу
C. обязательно ошибку

**Ответ:** A. None

None является явным сигналом отсутствия.

### Вопрос 3

Где должен находиться return None?

A. после цикла
B. внутри первой итерации
C. до цикла

**Ответ:** A. после цикла

Сначала нужно проверить все элементы.

### Вопрос 4

Что проверить перед вычислением процента?

A. список не пуст
B. все id чётные
C. title не содержит пробелы

**Ответ:** A. список не пуст

Иначе произойдёт деление на ноль.

### Главное после занятия

- Поиск возвращает словарь или `None`.
- `return None` находится после полного обхода.
- Статус меняется только после успешного поиска.
- Статистика возвращает данные, а интерфейс их печатает.
- Пустой список обрабатывается до деления.
- Текстовый поиск нормализует обе стороны.
