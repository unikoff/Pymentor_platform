# Теория месяца 1: подробная база Python для новичка

Этот файл связывает все занятия месяца в одну картину. Его не нужно заучивать за один вечер. Читайте нужную главу перед уроком, запускайте каждый пример, меняйте значения и проговаривайте, почему получился именно такой результат.

Главное правило месяца: **код нельзя понять только чтением**. Пример нужно запустить, изменить и хотя бы один раз сломать безопасным способом. Ошибка после эксперимента часто учит лучше, чем ещё один абзац определения.

## 1. Что такое программа

Программа — это точная последовательность инструкций для компьютера. В бытовой жизни похожим объектом является рецепт:

1. взять продукты;
2. смешать их в определённом порядке;
3. нагреть;
4. получить результат.

Если в рецепте забыть температуру или перепутать порядок, блюдо изменится. С программой так же: компьютер не догадывается о намерении разработчика и не исправляет неоднозначное правило самостоятельно.

Python обычно выполняет инструкции сверху вниз:

```python
print("Шаг 1: открыть план")
print("Шаг 2: выбрать задачу")
print("Шаг 3: начать работу")
```

Вывод:

```text
Шаг 1: открыть план
Шаг 2: выбрать задачу
Шаг 3: начать работу
```

Если вторая строка вызовет ошибку, третья уже не выполнится. Значит, результат зависит не только от написанных инструкций, но и от порядка их выполнения.

### Исходный код, интерпретатор и результат

- **Исходный код** — текст в файле `.py`.
- **Интерпретатор Python** — программа, которая читает код и выполняет его.
- **Вывод** — текст, который программа показала через `print`.
- **Возвращаемое значение** — результат функции, переданный через `return` другому коду.

`print` и `return` нельзя считать одинаковыми действиями:

```python
def show_sum(a, b):
    print(a + b)

def calculate_sum(a, b):
    return a + b
```

Первая функция показывает число человеку. Вторая отдаёт число программе, поэтому результат можно сохранить и использовать:

```python
result = calculate_sum(2, 3)
print(result * 10)  # 50
```

На платформе функция `solve(...)` должна именно **вернуть** результат. Платформа сама вызывает функцию и сравнивает возвращённое значение с ожидаемым.

## 2. Файлы, папки, терминал и окружение

Файл можно представить как документ, папку — как ящик для документов, а путь — как точный адрес. Два файла могут называться `main.py`, если они лежат в разных папках. Поэтому команда запуска зависит от текущего каталога.

Пример структуры:

```text
studyhub-planner/
├── main.py
├── README.md
└── .venv/
```

Если терминал открыт в `studyhub-planner`, команда запуска выглядит так:

```powershell
python main.py
```

Если терминал открыт уровнем выше, Python не найдёт `main.py` без дополнительного пути. На Windows полезны команды:

```powershell
Get-Location       # показать текущую папку
Get-ChildItem      # показать содержимое
Set-Location путь  # перейти в другую папку
```

### Виртуальное окружение

Виртуальное окружение похоже на отдельный ящик с инструментами для конкретного проекта. Если одному проекту нужна одна версия библиотеки, а другому — другая, их зависимости не должны конфликтовать.

```powershell
python -m venv .venv
```

Папку `.venv` не отправляют в Git: она большая, зависит от компьютера и воспроизводится командой. В первом месяце сторонние библиотеки почти не нужны, но привычка создавать окружение формируется сразу.

### Git простыми словами

Git похож на систему контрольных сохранений в игре. Коммит фиксирует рабочее состояние и объясняет, что изменилось. Но хороший коммит — не случайная точка сохранения, а законченная мысль.

```powershell
git status
git add main.py README.md
git commit -m "add task creation"
```

`git status` стоит запускать до и после коммита. Он показывает новые, изменённые и подготовленные файлы. GitHub хранит удалённую копию репозитория, но Git способен вести локальную историю и без GitHub.

## 3. Значения, переменные и типы

Переменную удобно представить как подписанную карточку, указывающую на значение. Имя объясняет смысл данных:

```python
task_title = "Изучить циклы"
priority = 3
is_done = False
```

Здесь:

- `task_title` ссылается на строку `str`;
- `priority` — на целое число `int`;
- `is_done` — на логическое значение `bool`.

Оператор `=` выполняет присваивание. Он не утверждает математическое равенство навсегда:

```python
completed = 2
completed = completed + 1
print(completed)  # 3
```

Python сначала вычисляет правую часть `completed + 1`, а затем связывает имя `completed` с новым значением.

### Четыре базовых типа

| Тип | Примеры | Для чего подходит |
| --- | --- | --- |
| `int` | `0`, `5`, `-10` | количество, идентификатор, возраст |
| `float` | `2.5`, `0.0` | измерения и дробные значения |
| `str` | `"SQL"`, `"42"` | текст и текстовое представление данных |
| `bool` | `True`, `False` | да/нет, выполнено/не выполнено |

Строка `"42"` и число `42` — разные значения:

```python
print("42" + "8")  # 428
print(42 + 8)      # 50
```

Тип можно проверить:

```python
value = "42"
print(type(value))  # <class 'str'>
```

### Преобразование типов

`input()` всегда возвращает строку, даже если человек ввёл цифры:

```python
age_text = input("Возраст: ")
age = int(age_text)
print(age + 1)
```

Преобразование может закончиться ошибкой:

```python
int("сорок")  # ValueError
```

Поэтому ввод пользователя — непроверенные данные. В будущем backend будет так же проверять JSON из HTTP-запроса.

## 4. Числа и арифметика

Основные операторы:

```python
print(7 + 2)   # 9
print(7 - 2)   # 5
print(7 * 2)   # 14
print(7 / 2)   # 3.5
print(7 // 2)  # 3
print(7 % 2)   # 1
print(7 ** 2)  # 49
```

`//` отвечает на вопрос «сколько полных частей помещается», а `%` — «какой остаток». Если 125 минут разделить на часы:

```python
minutes = 125
hours = minutes // 60
minutes_left = minutes % 60
print(hours, minutes_left)  # 2 5
```

Остаток также используется для проверки чётности:

```python
number = 12
is_even = number % 2 == 0
print(is_even)  # True
```

### Порядок операций

Python учитывает математический приоритет. Скобки делают намерение явным:

```python
average = (4 + 5 + 3) / 3
```

Не стоит надеяться, что читатель сам восстановит сложное правило. Понятные промежуточные переменные часто лучше одной длинной строки.

## 5. Строки как последовательности

Строка — упорядоченная последовательность символов. У каждого символа есть индекс, начинающийся с нуля:

```text
P  y  t  h  o  n
0  1  2  3  4  5
```

```python
language = "Python"
print(language[0])   # P
print(language[-1])  # n
print(language[1:4]) # yth
```

Правая граница среза не включается. Это правило часто встречается в Python и позволяет вычислить длину части как `end - start`.

Строки неизменяемы:

```python
title = "python"
title.upper()
print(title)  # python

title = title.upper()
print(title)  # PYTHON
```

Полезные методы:

- `strip()` — убрать пробельные символы по краям;
- `lower()` и `upper()` — изменить регистр в новом значении;
- `replace(old, new)` — заменить фрагменты;
- `split(separator)` — разбить строку на части;
- `startswith()` и `endswith()` — проверить начало или конец.

Перед сохранением пользовательское название часто нормализуют:

```python
raw_title = "  ИЗУЧИТЬ sql  "
title = raw_title.strip().lower().capitalize()
print(title)  # Изучить sql
```

Важно не переусердствовать: автоматическое изменение регистра может испортить аббревиатуры. Нормализация должна соответствовать требованиям проекта.

## 6. Boolean, сравнения и логические операторы

Логическое значение отвечает на вопрос, на который можно ответить «да» или «нет»:

- выполнена ли задача;
- существует ли пользователь;
- имеет ли он право на действие;
- подходит ли значение под правило.

Сравнения возвращают `True` или `False`:

```python
priority = 4
print(priority >= 3)  # True
print(priority == 5)  # False
print(priority != 1)  # True
```

Не путайте:

- `=` — присвоить значение;
- `==` — сравнить два значения;
- `!=` — проверить неравенство;
- `>=` и `<=` — включают границу.

### Таблица логики

| A | B | `A and B` | `A or B` |
| --- | --- | --- | --- |
| False | False | False | False |
| False | True | False | True |
| True | False | False | True |
| True | True | True | True |

`not` меняет значение на противоположное:

```python
is_done = False
print(not is_done)  # True
```

Правило «задача срочная, если приоритет не меньше четырёх и она не завершена» переводится почти дословно:

```python
is_urgent = priority >= 4 and not is_done
```

Сначала сформулируйте условие словами, потом замените каждую часть сравнением.

### Truthy и falsy

В условии пустая строка, пустая коллекция, `0` и `None` считаются ложными:

```python
tasks = []

if tasks:
    print("Есть задачи")
else:
    print("Список пуст")
```

Для новичка явное сравнение иногда читается лучше. Не пытайтесь сразу сокращать каждое условие.

## 7. Ветвление `if`, `elif`, `else`

Ветвление похоже на развилку дороги. Программа проверяет условие и выбирает маршрут.

```python
priority = 5

if priority == 5:
    label = "critical"
elif priority >= 3:
    label = "normal"
else:
    label = "low"

print(label)
```

Проверка идёт сверху вниз и прекращается после первой подходящей ветки. Поэтому порядок условий меняет результат.

Неверный порядок:

```python
score = 95

if score >= 60:
    grade = "зачёт"
elif score >= 90:
    grade = "отлично"
```

Ветка `score >= 90` недостижима: число 95 уже подходит под `score >= 60`. Правильнее сначала проверять более строгую границу.

### Отступы — часть синтаксиса

```python
if is_done:
    print("Готово")
    print("Можно брать следующую задачу")

print("Эта строка выполнится в любом случае")
```

Две строки с отступом относятся к `if`. Последняя строка находится вне ветки.

### Вложенные условия

Условия можно вкладывать, но глубокая лестница отступов усложняет чтение:

```python
if user_exists:
    if password_is_correct:
        access = True
```

Иногда яснее объединить условия:

```python
access = user_exists and password_is_correct
```

Выбирайте вариант, который точнее объясняет правило, а не тот, где меньше строк.

## 8. Цикл `for`

Цикл нужен, когда одно действие повторяется для нескольких элементов. Представьте преподавателя, который берёт работы учеников по одной и проверяет каждую. Переменная цикла — это текущая работа, а коллекция — вся стопка.

```python
tasks = ["Python", "Git", "SQL"]

for task in tasks:
    print(task)
```

На каждой итерации `task` получает следующее значение. Цикл завершится после последнего элемента.

### Накопление результата

```python
priorities = [2, 5, 3]
total = 0

for priority in priorities:
    total = total + priority

print(total)  # 10
```

Разбор по шагам:

| Итерация | `priority` | `total` до | `total` после |
| ---: | ---: | ---: | ---: |
| 1 | 2 | 0 | 2 |
| 2 | 5 | 2 | 7 |
| 3 | 3 | 7 | 10 |

Переменная `total` создаётся до цикла. Если обнулять её внутри, предыдущий результат будет теряться.

### `range` и `enumerate`

```python
for number in range(1, 4):
    print(number)
```

Вывод: `1`, `2`, `3`. Правая граница 4 не включается.

Для нумерации:

```python
for position, task in enumerate(tasks, start=1):
    print(f"{position}. {task}")
```

### Вложенные циклы

Вложенный цикл выполняется полностью для каждого шага внешнего. Если внешний делает 3 итерации, а внутренний 4, тело внутреннего выполнится 12 раз. Поэтому вложенные циклы требуют внимательного отношения к объёму работы.

## 9. Цикл `while`

`while` используют, когда повтор зависит не от готового списка, а от состояния. Бытовая аналогия: «пока бак не заполнен, продолжай наливать воду». Нужно понимать, что именно меняется и когда условие станет ложным.

```python
attempt = 1

while attempt <= 3:
    print(f"Попытка {attempt}")
    attempt += 1
```

Если забыть `attempt += 1`, условие навсегда останется истинным и цикл станет бесконечным.

### `break`

`break` завершает ближайший цикл:

```python
commands = ["add", "list", "exit", "delete"]

for command in commands:
    if command == "exit":
        break
    print(command)
```

`delete` не будет обработан.

### `continue`

`continue` пропускает остаток текущей итерации:

```python
titles = ["Python", "", "SQL"]

for title in titles:
    if not title:
        continue
    print(title)
```

Пустое название пропускается, цикл продолжается с `SQL`.

3. Почему цикл обязательно завершится?

## 10. Списки, кортежи и множества

### Список

Список похож на пронумерованную очередь. Порядок сохраняется, элементы можно добавлять и менять:

```python
tasks = ["Python", "Git"]
tasks.append("SQL")
tasks[0] = "Основы Python"
print(tasks)
```

Основные операции:

```python
tasks.append("Docker")    # добавить в конец
last = tasks.pop()        # удалить и вернуть последний
tasks.remove("Git")      # удалить первое совпадение
count = len(tasks)        # количество элементов
```

`remove` вызовет `ValueError`, если значения нет. Индекс вне диапазона вызывает `IndexError`. Перед обращением к первому элементу пустого списка нужна проверка.

### Кортеж

Кортеж похож на запечатанную карточку фиксированного формата:

```python
point = (10, 20)
```

Его нельзя изменить по индексу. Кортеж полезен, когда само отсутствие изменения выражает смысл данных.

### Множество

Множество похоже на список гостей без повторов:

```python
tags = {"python", "sql", "python"}
print(tags)  # два уникальных значения
```

Множество удобно для проверки принадлежности и удаления дублей, но порядок выдачи не стоит использовать как пользовательский порядок.

## 11. Словари

Словарь похож на анкету: значение ищут не по номеру строки, а по имени поля.

```python
task = {
    "id": 1,
    "title": "Изучить словари",
    "priority": 3,
    "is_done": False,
}
```

Чтение и изменение:

```python
print(task["title"])
task["is_done"] = True
task["category"] = "python"
```

Квадратные скобки подходят для обязательного ключа. Если отсутствие поля допустимо, используют `get`:

```python
priority = task.get("priority", 1)
```

Это не означает, что `get` нужно применять всегда. Если обязательный ключ неожиданно пропал, ошибка может быть полезнее тихого значения по умолчанию.

### Список словарей

```python
tasks = [
    {"id": 1, "title": "Python", "is_done": True},
    {"id": 2, "title": "SQL", "is_done": False},
]

for task in tasks:
    if task["is_done"]:
        print(task["title"])
```

Такая структура станет основой проекта первого месяца.

## 12. Функции

Функцию можно сравнить с кухонным прибором: у неё есть вход, понятная операция и выход. Тостер получает хлеб, выполняет одну задачу и возвращает тост. Он не должен одновременно составлять список покупок и включать музыку.

```python
def format_task(title, priority):
    return f"[{priority}] {title}"

result = format_task("SQL", 3)
print(result)  # [3] SQL
```

- `def` начинает определение;
- `format_task` — имя;
- `title`, `priority` — параметры;
- значения `"SQL"`, `3` при вызове — аргументы;
- `return` завершает функцию и возвращает значение.

### Локальные переменные

```python
def calculate_left(total, completed):
    left = total - completed
    return left
```

`left` существует внутри вызова функции. Это помогает не засорять программу случайно связанными переменными.

### Ранний `return`

```python
def normalize_title(title):
    cleaned = title.strip()
    if not cleaned:
        return "Без названия"
    return cleaned
```

После `return` функция прекращает выполнение. Ранний выход помогает отдельно обработать граничный случай.

### Чистая функция

Чистая функция зависит от аргументов и не меняет внешнее состояние:

```python
def is_urgent(priority, is_done):
    return priority >= 4 and not is_done
```

Для одинаковых аргументов результат всегда одинаков. Такую функцию легко тестировать.

## 13. Ошибки и отладка

Ошибка — техническое сообщение о нарушенном правиле. Важно различать:

- `SyntaxError` — код невозможно разобрать;
- `NameError` — используется неизвестное имя;
- `TypeError` — операция не подходит типам;
- `ValueError` — тип допустим, но конкретное значение неверно;
- `IndexError` — индекс отсутствует;
- `KeyError` — ключ словаря отсутствует.

Пример:

```python
tasks = []
print(tasks[0])
```

Последняя строка traceback сообщит `IndexError: list index out of range`. Это не просьба обернуть весь код в `try/except`. Сначала нужно понять правило: список может быть пуст, поэтому до `[0]` требуется проверка.

Алгоритм отладки:

1. Воспроизвести ошибку на том же входе.
2. Прочитать последнюю строку traceback.
3. Найти первую строку своего кода в стеке.
4. Проверить значения и типы перед ней.
5. Сформулировать нарушенное правило.
6. Исправить причину.

## 15. Итоговая карта мышления

Когда получаете задачу, двигайтесь в таком порядке:

1. Что приходит на вход?
2. Какой у каждого значения тип?
3. Что нужно вернуть?
4. Нужен выбор (`if`) или повтор (`for`/`while`)?
5. Какая структура хранит данные?
6. Можно ли выделить отдельное правило в функцию?
7. Какие пустые и граничные случаи возможны?
8. Как проверить решение на маленьком примере?

Эта схема важнее запоминания десятков методов. Синтаксис можно уточнить, но умение разложить задачу формируется только регулярной практикой.
