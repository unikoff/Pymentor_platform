# Числа, операции и преобразование типов

Используем Python как понятный калькулятор: разберём основные операции, полные группы и остаток, а затем превратим текст с цифрами в настоящие числа.

**Темы:** + · - · * · / · // · % · int() · float() · str()

## 01. От чисел к вычислению

Обычный калькулятор получает числа и знак операции. Python делает то же самое, но позволяет сохранить результат под понятным именем и использовать его дальше.

1. **Выполнить арифметику:** сложить, вычесть, умножить или разделить числа.
2. **Разобрать деление:** получить дробный результат, полные группы и остаток.
3. **Подготовить данные:** превратить текст с цифрами в число перед вычислением.

В конце StudyHub рассчитает общее время занятий и разделит минуты на полные часы и остаток.

> Сначала проговаривайте вычисление словами. Код должен повторять понятное правило, а не быть набором случайно выбранных знаков.

## 02. Основные арифметические операции

Представьте кассовый чек: количество товаров умножается на цену, скидка вычитается, а итоговые суммы складываются. Операторы Python записывают те же знакомые действия.

### Сложение — +

Объединяет количества и даёт `8`.

```python
total = 5 + 3
```

### Вычитание — -

Показывает, сколько осталось: `6`.

```python
left = 10 - 4
```

### Умножение — *

Пять занятий по 45 минут дают `225` минут.

```python
minutes = 5 * 45
```

### Обычное деление — /

Делит число на равные части и может вернуть дробный результат `2.5`.

```python
average = 10 / 4
```

```python
lesson_count = 5
minutes_per_lesson = 45
total_minutes = lesson_count * minutes_per_lesson
```

**Пошаговый разбор**

1. Сохраняем количество занятий. **Состояние:** lesson_count = 5
2. Сохраняем продолжительность одного занятия. **Состояние:** minutes_per_lesson = 45
3. Справа вычисляется 5 * 45, затем результат 225 сохраняется в total_minutes. **Состояние:** total_minutes = 225

Имена `lesson_count` и `minutes_per_lesson` объясняют формулу лучше, чем запись `5 * 45` без контекста.

Отрицательное число не всегда является ошибкой Python. `3 - 5` честно даёт `-2`. Позже условия помогут запретить неподходящий результат для конкретной задачи.

## 03. Порядок действий и скобки

Python использует знакомый школьный порядок: сначала скобки, затем умножение и деление, потом сложение и вычитание. Скобки похожи на пометку маркером: «эту часть посчитай первой».

### Предскажите результат

```python
print(2 + 3 * 4)
print((2 + 3) * 4)
```

*Подсказка: Без скобок сначала выполняется умножение. Во второй строке сначала вычисляется 2 + 3.*

**Ответ**

```text
14
20
```

### Сравнение решений

**Вопрос:** Как правильно вычислить среднее трёх оценок?

### Делится только последнее число

```python
average = a + b + c / 3
```

Python сначала разделит c на 3.

### Делится вся сумма

```python
average = (a + b + c) / 3
```

Скобки сначала собирают общую сумму.

**Вывод:** предпочтителен правый вариант. Среднее требует разделить на три сумму всех значений.

Если формула становится длинной, лучше создать промежуточное имя: сначала `total_score`, затем `average`.

Обычное деление `/` возвращает дробный тип `float`, даже если результат выглядит целым: `8 / 2` даёт `4.0`.

### Заполните пропуск

Выберите выражение, которое сначала вычислит сумму.

```python
average = ___ / 3
```

**Варианты:** `(a + b + c)`, `a + b + c`, `a + (b + c / 3)`

**Ответ:** `(a + b + c)`

Скобки объединяют три числа до деления.

## 04. Деление, полные коробки и остаток

Представьте 125 минут. Можно узнать дробное количество часов, количество полных часов или оставшиеся минуты. Для каждого вопроса нужен свой оператор.

### Разделить полностью — /

Получаем примерно `2.0833` часа.

```python
125 / 60
```

### Сколько полных групп — //

В 125 минутах помещаются `2` полных часа.

```python
125 // 60
```

### Что осталось — %

После двух полных часов остаётся `5` минут.

```python
125 % 60
```

```python
minutes = 125
full_hours = minutes // 60
minutes_left = minutes % 60
```

**Пошаговый разбор**

1. Исходное время — 125 минут. **Состояние:** minutes = 125
2. Оператор // находит два полных часа. **Состояние:** full_hours = 2
3. Оператор % находит пять оставшихся минут. **Состояние:** minutes_left = 5

### Аналогия с коробками

Если есть 8 яблок и коробки по 3, то `8 // 3` даёт две полные коробки, а `8 % 3` — два яблока, которые остались снаружи.

### Знак % не вычисляет процент автоматически

В Python это остаток от деления. Процент вычисляется отдельной формулой, например `completed / total * 100`.

## 05. Почему цифры из input() остаются текстом

Поле ввода похоже на бумажную анкету: всё, что человек написал, сначала приходит как текст. Даже запись `18` после `input()` имеет тип `str`.

**неподготовленный ввод**

```python
age_text = input("Возраст: ")
print(age_text + "1")
```

**строки соединяются**

```text
> python main.py
Возраст: 18
181
```

### Сделать целое число — int()

Получается число `18`, с которым можно выполнять арифметику.

```python
age = int("18")
```

### Сделать дробное число — float()

Получается дробное значение `2.5`. В записи используется точка.

```python
price = float("2.5")
```

### Сделать текст — str()

Число превращается в строку `"18"`.

```python
text = str(18)
```

### Подготовьте возраст до вычисления

Соберите понятный маршрут.

**Фрагменты**

- `age_text = input("Возраст: ")`
- `age = int(age_text)`
- `next_age = age + 1`
- `print(next_age)`

**Правильный порядок**

```python
age_text = input("Возраст: ")
age = int(age_text)
next_age = age + 1
print(next_age)
```

Сначала приходит текст, затем создаётся число, после чего выполняется сложение.

> Телефон, индекс или номер документа не всегда нужно превращать в число. Если данные не участвуют в арифметике, строка может быть правильным выбором.

## 06. Три понятных причины ошибки

Название ошибки не нужно зубрить отдельно. Сначала посмотрите, что именно Python пытался сделать: соединить несовместимые типы, преобразовать неподходящий текст или разделить на ноль.

### Не подходят типы — TypeError

Строка и целое число не складываются одним оператором.

```python
"18" + 1
```

### Не подходит содержимое — ValueError

int умеет принимать строку, но эта строка не является записью целого числа.

```python
int("восемнадцать")
```

### Деление на ноль — ZeroDivisionError

Нельзя разделить количество на нулевое число частей.

```python
10 / 0
```

### Разбор ошибки

```python
progress_text = "72,5"
progress = float(progress_text)
```

**Вопрос:** Почему дробное значение не преобразуется?

A. В Python внутри числовой строки нужна точка
B. float не принимает строки
C. Имя progress запрещено

**Ответ:** A. В Python внутри числовой строки нужна точка

Для float используется запись 72.5.

**Исправленный вариант**

```python
progress_text = "72.5"
progress = float(progress_text)
```

После ошибки прочитайте последнюю строку сообщения и найдите операцию, которая не смогла завершиться.

Не исправляйте весь файл одновременно. Измените один тип, одну строку или один делитель и запустите код снова.

## 07. Практикум StudyHub: учебное время

Соединим переменные прошлого занятия с арифметикой. Формула остаётся простой: количество занятий умножить на продолжительность одного занятия.

**studyhub/main.py**

```python
project_name = "StudyHub"
lesson_count = 5
minutes_per_lesson = 45

total_minutes = lesson_count * minutes_per_lesson
full_hours = total_minutes // 60
minutes_left = total_minutes % 60

print(project_name)
print(total_minutes)
print(full_hours)
print(minutes_left)
```

**ожидаемый результат**

```text
> python main.py
StudyHub
225
3
45
```

### Шаг 1. Проговорите формулу

Пять занятий по 45 минут дают 225 минут.

### Шаг 2. Разделите время

225 минут содержат три полных часа и ещё 45 минут.

### Шаг 3. Проверьте простые границы

Временно подставьте `0`, `59`, `60` и `61`. До запуска предскажите количество полных часов и остаток минут.

### Шаг 4. Измените исходные данные

Поставьте `lesson_count = 4` и `minutes_per_lesson = 30`. Ожидается 120 минут, два полных часа и ноль минут остатка.

### Заполните пропуск

Выберите оператор для оставшихся минут.

```python
minutes_left = total_minutes ___ 60
```

**Варианты:** `%`, `//`, `/`

**Ответ:** `%`

Оператор % возвращает остаток после выделения полных часов.

## 08. Проверьте числовую модель

Итоговые вопросы проверяют выбор операции и момент преобразования текста в число.

### Вопрос 1

Что вернёт 8 // 3?

A. 2
B. 2.666...
C. 3

**Ответ:** A. 2

В восьми помещаются две полные группы по три.

### Вопрос 2

Что вернёт 8 % 3?

A. 2
B. 3
C. 2.666...

**Ответ:** A. 2

После двух полных групп остаётся два.

### Вопрос 3

Какой тип возвращает input()?

A. str
B. int
C. float

**Ответ:** A. str

Ввод пользователя сначала приходит как текст.

### Вопрос 4

Что произойдёт при int("abc")?

A. ValueError
B. Получится 0
C. Получится abc

**Ответ:** A. ValueError

Строка abc не является записью целого числа.

### Главное после занятия

- Python может использоваться как калькулятор и сохранять результаты в переменных.
- Умножение и деление выполняются раньше сложения, если нет скобок.
- `/` даёт обычное деление, `//` — полные группы, `%` — остаток.
- `input()` возвращает строку, даже если человек ввёл цифры.
- `int()` и `float()` подготавливают текстовые числа к вычислению.
- TypeError, ValueError и ZeroDivisionError описывают разные причины проблемы.
