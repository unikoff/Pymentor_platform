# Как Python выполняет программу

Начнём с самого начала: разберём, кто запускает Python-код, что означает слово «интерпретатор», как читать print() и почему одна пропущенная кавычка может остановить программу.

**Темы:** первый Python-файл · код → запуск → результат

## 01. Что произойдёт на первом занятии

Здесь не нужно заранее знать программирование. Мы не будем писать большую программу или использовать сложные термины без объяснения. Сначала разберём один короткий путь: от текста в файле до результата на экране.

1. **Поймём, кто выполняет код:** зачем нужен Python-интерпретатор и что значит «интерпретируемый язык».
2. **Запустим первую команду:** соберём традиционную программу `Hello, World!`.
3. **Научимся читать одну строку:** разберём `print`, скобки, кавычки и комментарии.

Все задания будут короткими: восстановить порядок строк, выбрать правильные кавычки, добавить закрывающую скобку или исправить одну опечатку.

> Рабочий ритм занятия: **посмотреть → предположить → запустить → изменить одну деталь → проверить**.

## 02. Кто выполняет Python-код

Файл с кодом похож на рецепт. В рецепте записаны действия, но сам лист бумаги ничего не приготовит. Нужен тот, кто прочитает инструкции и выполнит их. Для Python эту роль выполняет Python-интерпретатор.

### Файл с кодом — 1

Это текстовая инструкция, сохранённая в файле с расширением `.py`. Пока файл просто лежит на диске, ничего не происходит.

```python
print("Привет")
```

### Интерпретатор — 2

Это установленная на компьютере программа Python. Она открывает файл, разбирает инструкции и начинает их выполнять.

```powershell
python main.py
```

### Результат — 3

После выполнения в терминале появляется обычный вывод или сообщение о проблеме.

```text
Привет
```

### Простая аналогия

**Код** — рецепт. **Интерпретатор** — повар, который умеет читать этот рецепт. **Терминал** — место, где мы видим результат работы.

### Что значит «интерпретируемый язык»

Когда мы запускаем `python main.py`, Python читает сохранённый файл и выполняет его инструкции. Для первого месяца достаточно такой модели: каждый запуск снова передаёт файл интерпретатору.

### Почему компьютер не понимает файл сам

Компьютер не знает, что означает слово `print`. Это знает установленный Python. Поэтому без интерпретатора файл `main.py` остаётся обычным текстовым файлом с командами.

**код внутри main.py**

```python
print("Привет из Python")
```

**интерпретатор выполняет файл**

```text
> python main.py
Привет из Python
```

> Внутри Python есть дополнительные технические шаги, но сейчас они не нужны. Главная модель: **файл хранит инструкции, интерпретатор их выполняет, терминал показывает результат**.

## 03. Первая программа: Hello, World!

У программистов есть традиция начинать знакомство с языком с фразы `Hello, World!`. Такая программа ничего не вычисляет, зато сразу подтверждает: файл сохранён, Python установлен и запуск работает.

**main.py**

```python
print("Hello, World!")
```

**первый запуск**

```text
> python main.py
Hello, World!
```

```python
print("Hello, World!")
```

**Пошаговый разбор**

1. Python встречает команду print — показать данные на экране. **Состояние:** команда = print
2. Внутри скобок находится текст "Hello, World!". Кавычки показывают его границы. **Состояние:** текст = "Hello, World!"
3. После выполнения текст появляется в терминале без кавычек. **Состояние:** вывод = Hello, World!

### Заполните пропуск

Дополните первую программу.

```python
print("Hello, ___!")
```

**Варианты:** `World`, `print`, `main.py`

**Ответ:** `World`

Мы изменяем только текст внутри кавычек. Команда print и скобки уже готовы.

> Кавычки нужны в коде, чтобы обозначить текст. В самом выводе они не появляются.

## 04. Python идёт сверху вниз

Представьте список дел на бумаге: сначала первый пункт, затем второй, затем третий. В простом Python-файле инструкции выполняются так же — сверху вниз.

```python
print("Открываем StudyHub")
print("Показываем название")
print("Завершаем запуск")
```

**Пошаговый разбор**

1. Сначала выполняется верхняя строка. **Состояние:** вывод = Открываем StudyHub
2. Затем Python переходит к следующей строке. **Состояние:** вывод = Открываем StudyHub ⏎ Показываем название
3. После последней строки файл заканчивается, и программа завершает работу. **Состояние:** вывод = Открываем StudyHub ⏎ Показываем название ⏎ Завершаем запуск

### Восстановите порядок запуска

Соберите программу: сначала начало, затем название проекта, затем завершение.

**Фрагменты**

- `print("Начинаем запуск")`
- `print("StudyHub")`
- `print("Запуск завершён")`

**Правильный порядок**

```python
print("Начинаем запуск")
print("StudyHub")
print("Запуск завершён")
```

Python выполнит строки в том порядке, в котором они стоят в файле.

В этом задании нет строки с `#`, потому что комментарий не выполняется и не является этапом запуска.

Если поменять местами вторую и третью строки, Python не исправит порядок. Он честно покажет результат сверху вниз именно так, как записал автор.

## 05. Как читать print(), скобки и кавычки

Одну строку можно разобрать как простое поручение: **«выполни действие print и передай ему текст внутри скобок»**.

**одна команда по частям**

```python
print("Привет")
```

### `print` — действие

Слово `print` сообщает Python: «покажи переданные данные в терминале». Это готовая команда, которую уже знает интерпретатор.

### `()` — место для данных

Скобки показывают, что именно мы передаём команде. Их можно представить как конверт: команда получает содержимое, лежащее внутри.

### `"Привет"` — текст

Кавычки отмечают начало и конец текста. Без них Python решит, что `Привет` — это имя, значение которого нужно найти.

### Двойные кавычки — ✓

Прямые двойные кавычки подходят для текста.

```python
print("Привет")
```

### Одинарные кавычки — ✓

Прямые одинарные кавычки тоже подходят.

```python
print('Привет')
```

### Без кавычек — ✗

Python ищет имя `Привет` и не находит его.

```python
print(Привет)
```

### Разбор ошибки

```python
print(Привет)
```

**Вопрос:** Почему Python не показывает слово Привет?

A. Текст записан без кавычек
B. Команду print можно использовать только на английском
C. В конце строки нет точки

**Ответ:** A. Текст записан без кавычек

Без кавычек Python воспринимает Привет как имя, а не как текст.

**Исправленный вариант**

```python
print("Привет")
```

## 06. Первые правила Python без перегрузки

У каждого языка есть свои правила записи. Они похожи на правила оформления адреса: если убрать важный элемент или заменить знак похожим, получатель может не понять сообщение.

### 1. Используйте прямые кавычки

Подходят `" "` и `' '`. Русские типографские кавычки `« »` выглядят похоже, но Python не считает их кавычками языка.

```python
print("Привет")   # правильно
print('Привет')   # правильно
print(«Привет»)   # ошибка
```

### 2. Закрывайте скобку

Закрывающая скобка `)` показывает, где команда закончилась. Python не будет угадывать её место.

```python
print("Готово")   # правильно
print("Готово"    # скобка не закрыта
```

### 3. Комментарий начинается с #

Комментарий — заметка для человека. Python пропускает текст после `#`. Так можно оставить пояснение или временно отключить строку.

```python
print("Видно")
# print("Временно скрыто")
print("Тоже видно")
```

### 4. Точка с запятой в конце не нужна

В некоторых языках обычную инструкцию завершают символом `;`. В Python каждая простая команда обычно записывается на своей строке, поэтому точку с запятой в конце ставить не нужно.

```python
print("Первая строка")
print("Вторая строка")
```

### 5. Пустая строка ничего не печатает

Пустая строка внутри файла только разделяет код визуально. Чтобы вывести пустую строку в терминале, нужна команда `print()`.

> Исправляйте по одному правилу за раз. Если в одном примере сразу четыре ошибки, первая ошибка может помешать увидеть остальные.

## 07. Два момента возникновения ошибки

Ошибку можно сравнить с проверкой инструкции перед работой. Иногда Python видит, что запись составлена неправильно, ещё до выполнения. Иногда запись понятна, но проблема появляется уже во время работы.

### Python не понял запись — до выполнения

Не закрыта скобка. Интерпретатор не может нормально разобрать файл, поэтому программа может вообще не начать выполнение.

```python
print("Шаг 2"
```

### Проблема на конкретной строке — во время

Запись устроена понятно, но Python ищет имя `Готово` и не находит его. Предыдущие строки уже могли выполниться.

```python
print(Готово)
```

**ошибка до выполнения**

```python
print("Первая строка")
print("Вторая строка"
```

**Python не начал выполнять команды**

```text
> python main.py
SyntaxError: '(' was never closed
```

**ошибка во время выполнения**

```python
print("Первая строка")
print(Вторая)
print("Третья строка")
```

**первая строка успела появиться**

```text
> python main.py
Первая строка
NameError: name 'Вторая' is not defined
```

`SyntaxError` часто означает: Python не смог понять структуру записи.

`NameError` часто означает: Python встретил имя, но до этого не получил для него значение.

Пока достаточно одного вопроса: программа не начала работу или остановилась на конкретной строке?

## 08. Первый файл StudyHub

Теперь соберём небольшой файл проекта. Сначала оставим традиционный Hello, World!, затем добавим название StudyHub. Это уже настоящий запускаемый Python-файл, хотя в нём всего несколько строк.

**studyhub/main.py**

```python
# Первый файл проекта StudyHub
print("Hello, World!")
print("StudyHub Planner")
print("Первый запуск выполнен")
```

**результат**

```text
> python main.py
Hello, World!
StudyHub Planner
Первый запуск выполнен
```

### Шаг 1. Измените текст

Замените `Первый запуск выполнен` на `Я запустил свой первый Python-файл`.

### Шаг 2. Временно скройте Hello, World!

Добавьте `#` перед первой командой print. Строка должна остаться в файле, но исчезнуть из терминала.

### Шаг 3. Верните строку обратно

Уберите `#` и снова запустите файл. Это показывает, что комментарий временно отключает инструкцию, а не удаляет её.

### Заполните пропуск

Платформа уже подготовила оболочку. Измените только текст внутри кавычек.

```python
def solve():
    return "___"
```

**Варианты:** `Hello, World!`, `python main.py`, `print`

**Ответ:** `Hello, World!`

def и return пока не нужно разбирать. В этом задании меняется только готовый текстовый результат.

### Вопрос 1

Кто выполняет инструкции из main.py?

A. Python-интерпретатор
B. сам текстовый файл
C. браузер

**Ответ:** A. Python-интерпретатор

Интерпретатор открывает сохранённый файл и выполняет Python-код.

### Вопрос 2

Что означает текст в кавычках внутри print("Привет")?

A. данные, которые нужно показать
B. имя файла
C. команда терминала

**Ответ:** A. данные, которые нужно показать

Кавычки обозначают текстовые данные.

### Вопрос 3

Что произойдёт со строкой, перед которой стоит #?

A. Python пропустит её
B. она выполнится два раза
C. Python удалит файл

**Ответ:** A. Python пропустит её

Комментарий остаётся в файле, но не выполняется.

### Главное после занятия

- Файл `.py` хранит инструкции и сам себя не выполняет.
- Python-интерпретатор читает сохранённый файл и выполняет его команды.
- В простом файле инструкции выполняются сверху вниз.
- `print` — действие, `()` — место для данных, кавычки обозначают текст.
- Комментарий после `#` не выполняется.
- В конце обычной строки Python точка с запятой не нужна.
- Первую ошибку лучше исправлять отдельно, а затем запускать файл снова.
