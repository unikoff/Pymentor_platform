# Месяц 1. Основы Python и мышление программиста

## Результат месяца

За 80 часов ученик перестаёт бояться терминала и кода, понимает, как Python выполняет инструкции, уверенно использует базовые типы, ветвления, циклы, коллекции и простые функции. Итог месяца — консольный `StudyHub Planner`: меню задач, добавление, вывод, поиск, смена статуса и аккуратный Git-репозиторий.

Данные пока живут только в памяти программы. Сохранение в JSON появится во втором месяце после изучения файлов, функций и обработки ошибок.

## Как проходить месяц

- Прочитать [Теория месяца.md](Теория%20месяца.md) перед первыми занятиями и возвращаться к нужной главе при ошибках.
- Пройти 20 занятий из [План обучения.md](План%20обучения.md) по порядку.
- В каждой задаче для терминала изменить только тело `solve(...)`. Платформа сама вызывает функцию и проверяет результат на скрытых тестах.
- Каждый пример вида `solve("Анна") -> "Привет, Анна!"` — это пример вызова и возвращаемого значения, а не текст, который надо печатать через `input()`.
- В конце каждой недели сделать один осмысленный Git-коммит и записать непонятные места для наставника.

## Правило перехода во второй месяц

Ученик не должен «закрыть» месяц только зелёными галочками. До перехода он самостоятельно запускает проект из терминала, объясняет цикл, список, словарь и функцию, читает traceback и делает коммит с README.
