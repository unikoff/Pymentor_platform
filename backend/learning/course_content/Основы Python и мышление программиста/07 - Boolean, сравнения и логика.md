# Boolean, сравнения и логика

Научимся задавать Python простые вопросы с ответом «да» или «нет»: сравнивать значения, учитывать границы и соединять несколько проверок.

**Темы:** сравнения и границы · and · or · not

## 01. Маршрут занятия

Представьте обычный вопрос: «задача выполнена?» Ответ может быть только «да» или «нет». Python хранит такие ответы как `True` и `False`. Сначала мы формулируем вопрос словами, а затем переводим его в код.

1. **Получить ответ:** сравнить значения через `>`, `>=`, `==` и другие операторы.
2. **Соединить проверки:** прочитать `and`, `or` и `not` как части правила.
3. **Проверить границы:** отдельно протестировать значение на пороге, ниже и выше него.

В конце вы соберёте правило срочной задачи StudyHub и научитесь отличать настоящее `False` от строки `"False"`.

> Не нужно запоминать таблицу логики заранее. Сначала проговаривайте вопрос обычными словами, затем подставляйте конкретные значения и только после этого записывайте операторы.

## 02. bool отвечает на вопрос

Тип `bool` можно сравнить с выключателем: он находится только в одном из двух положений. `True` означает «да», `False` — «нет». Хорошее имя переменной сразу показывает, какой вопрос мы задаём.

### Состояние — is_

Читается как «выполнена ли задача?».

```python
is_done = False
```

### Наличие — has_

Читается как «есть ли доступ?».

```python
has_access = True
```

### Возможность — can_

Читается как «можно ли начать?».

```python
can_start = True
```

### Разбор ошибки

```python
is_done = "False"
print(type(is_done).__name__)
```

**Вопрос:** Почему переменная не является boolean?

A. False можно использовать только в if
B. Кавычки превратили значение в строку
C. Имя должно начинаться с has_

**Ответ:** B. Кавычки превратили значение в строку

Значение "False" — обычный текст типа str. Boolean записывается без кавычек.

**Исправленный вариант**

```python
is_done = False
print(type(is_done).__name__)  # bool
```

Имя `status` слишком широкое: статус может быть строкой `"new"`, `"done"` или другим значением. Имя `is_done` явно обещает два состояния.

## 03. Сравнения создают логический результат

Сравнение похоже на проверку на входе: охранник не меняет возраст человека, а только отвечает, подходит ли он под правило. Python тоже не меняет значения, а возвращает `True` или `False`.

### Сопоставление

Соедините оператор с его смыслом.

| Оператор | Проверка |
| --- | --- |
| > | больше |
| >= | больше или равно |
| == | равно |
| != | не равно |

Операторы сравнения всегда дают логический ответ.

### Предскажите результат

```python
priority = 4
print(priority > 3)
print(priority >= 4)
print(priority == 5)
print(priority != 1)
```

*Подсказка: Проверяйте каждую строку независимо.*

**Ответ**

```text
True
True
False
True
```

`=` выполняет присваивание, а `==` сравнивает. Это разные операции.

Строки сравниваются вместе с регистром и пробелами: `"Done" == "done"` и `"done " == "done"` возвращают `False`.

## 04. Граница входит или не входит

Границу удобно представить как ростовую отметку у аттракциона. Фраза «от 120 сантиметров» включает ровно 120. Так же правило «приоритет от четырёх» включает значение `4`, поэтому нужен оператор `>=`.

**проверяем точку перехода**

```python
print(3 >= 4)  # ниже границы: False
print(4 >= 4)  # на границе: True
print(5 >= 4)  # выше границы: True
```

### Сравнение решений

**Вопрос:** Какое выражение соответствует правилу «приоритет от 4 включительно»?

### Граница потеряна

```python
is_high = priority > 4
```

Значение 4 ошибочно исключено.

### Граница включена

```python
is_high = priority >= 4
```

Подходят 4, 5 и более высокие значения.

**Вывод:** предпочтителен правый вариант. Оператор >= включает само граничное значение.

### Верно или неверно

Для диапазона от 1 до 5 включительно подходит выражение `1 <= priority <= 5`.

**Ответ:** верно. Обе границы включены, а Python поддерживает цепочки сравнений.

> Минимальный набор проверки правила: значение ниже границы, значение на границе и значение выше границы.

## 05. and, or и not

Логические операторы соединяют несколько ответов «да» и «нет». Представьте пропуск на мероприятие: иногда нужны и билет, и документ; иногда подойдёт телефон или email; иногда нужно проверить, что задача ещё не выполнена.

### Оба условия — and

Истинно только тогда, когда истинны обе части.

```python
True and False  # False
```

### Хотя бы одно — or

Истинно, когда истинна хотя бы одна часть.

```python
False or True  # True
```

### Отрицание — not

Меняет логический ответ на противоположный.

```python
not False  # True
```

### Сопоставление

Соедините правило с оператором.

| Левая часть | Правая часть |
| --- | --- |
| нужны пропуск и подтверждение | and |
| подойдёт email или телефон | or |
| задача ещё не выполнена | not is_done |

Оператор выбирается по смыслу человеческой фразы.

### Вспомните модель

**Вопрос:** Когда выражение a or b возвращает False?

*Подсказка: Достаточно ли одной истинной части?*

**Ответ:** Только когда обе части равны False.

## 06. Переводим правило в выражение

Длинное правило лучше не читать одним куском. Разделим его на два простых вопроса: «приоритет высокий?» и «задача ещё открыта?». Затем соединим ответы словом `and`.

```python
priority = 5
is_done = True
has_high_priority = priority >= 4
is_open = not is_done
is_urgent = has_high_priority and is_open
```

**Пошаговый разбор**

1. Приоритет равен 5. **Состояние:** priority = 5
2. Задача уже завершена. **Состояние:** is_done = True
3. 5 >= 4 даёт True. **Состояние:** has_high_priority = True
4. not True даёт False. **Состояние:** is_open = False
5. True and False даёт False: завершённая задача не срочная. **Состояние:** is_urgent = False

### Сравнение решений

**Вопрос:** Какой вариант легче объяснить при первом знакомстве со сложным правилом?

### Одна строка

```python
is_urgent = priority >= 4 and not is_done
```

Коротко, но новичку нужно разобрать несколько операций сразу.

### Промежуточные вопросы

```python
has_high_priority = priority >= 4
is_open = not is_done
is_urgent = has_high_priority and is_open
```

Каждая часть правила получила понятное имя.

**Вывод:** предпочтителен правый вариант. После понимания промежуточный вариант можно безопасно сократить до одной строки.

> Избегайте двойных отрицаний вроде `not is_not_done`. Положительные имена читаются быстрее.

## 07. Truthy, falsy и практика StudyHub

Python умеет воспринимать некоторые значения как «пусто» или «не пусто». Пустая корзина, нулевое количество и пустая строка обычно дают `False`; непустые значения — `True`. Это удобное сокращение, но сначала важно понимать явные сравнения.

### Предскажите результат

```python
print(bool(0))
print(bool(1))
print(bool(""))
print(bool("SQL"))
```

*Подсказка: Пустая строка ложна, непустая — истинна.*

**Ответ**

```text
False
True
False
True
```

**правило срочной задачи**

```python
priority = 4
is_done = False

is_urgent = priority >= 4 and not is_done
print(is_urgent)
```

Сначала замените `priority` на `3`, затем верните `4` и установите `is_done = True`. Каждый раз сделайте прогноз до запуска.

На платформе функция `solve(priority, is_done)` должна вернуть логическое значение, а не строки `"True"` или `"False"`.

## 08. Проверь себя

### Вопрос 1

Что возвращает выражение 3 < 5?

A. 3
B. 5
C. True

**Ответ:** C. True

Сравнение возвращает boolean.

### Вопрос 2

Как включить значение 4 в правило «от четырёх»?

A. priority > 4
B. priority >= 4
C. priority == 5

**Ответ:** B. priority >= 4

>= включает границу.

### Вопрос 3

Когда a and b истинно?

A. когда истинна хотя бы одна часть
B. когда истинны обе части
C. когда обе части ложны

**Ответ:** B. когда истинны обе части

and требует истинности обеих частей.

### Вопрос 4

Что вернёт priority >= 4 and not is_done при priority=5 и is_done=True?

A. True
B. False
C. 5

**Ответ:** B. False

not True даёт False, поэтому всё выражение ложно.

### Главное после занятия

- `bool` хранит `True` или `False`.
- Сравнения возвращают логический результат.
- `=` присваивает, `==` сравнивает.
- `and` означает «оба», `or` — «хотя бы одно».
- Границы нужно проверять отдельными тестовыми значениями.
