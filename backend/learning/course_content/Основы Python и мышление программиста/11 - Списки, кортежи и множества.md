# Списки, кортежи и множества

Перейдём от отдельных переменных к коллекциям: научимся безопасно менять список, понимать ссылки и копии, выбирать кортеж для фиксированных данных и множество для уникальности.

**Темы:** list и изменение · tuple · set

## 01. Маршрут занятия

Коллекции решают разные задачи. Важно не просто запомнить три названия, а выбрать структуру по требованиям: нужен ли порядок, допустимы ли изменения и должны ли повторы сохраняться.

1. **Хранить последовательность:** использовать `list`, когда элементы имеют порядок и набор будет меняться.
2. **Зафиксировать запись:** выбрать `tuple`, когда позиции имеют постоянный смысл.
3. **Контролировать уникальность:** применить `set` для проверки повторов и принадлежности.

Дополнительно вы разберёте индексы, пустые коллекции, `append`, `pop`, `remove`, второе имя одного списка и алгоритм уникальности с сохранением порядка.

> Основной вопрос урока: **какое поведение данных требуется проекту**, а не какая запись короче.

## 02. Список — упорядоченная изменяемая последовательность

Список хранит несколько значений под одним именем. Позиции начинаются с нуля, а индекс `-1` обозначает последний элемент.

**индексы списка**

```python
tasks = ["Python", "Git", "SQL"]

print(tasks[0])   # Python
print(tasks[1])   # Git
print(tasks[-1])  # SQL
print(len(tasks)) # 3
```

### Предскажите результат

```python
tasks = ["Python", "Git", "SQL"]
print(tasks[0:2])
print("Docker" in tasks)
```

*Подсказка: Правая граница среза не включается, а in проверяет наличие значения.*

**Ответ**

```text
['Python', 'Git']
False
```

### Разбор ошибки

```python
tasks = []
print(tasks[0])
```

**Вопрос:** Почему программа завершается ошибкой?

A. Списки нельзя передавать в print
B. У пустого списка нет индекса 0
C. Первый индекс списка равен 1

**Ответ:** B. У пустого списка нет индекса 0

Пустой список не содержит ни одного элемента, поэтому обращение по индексу невозможно.

**Исправленный вариант**

```python
tasks = []
if tasks:
    print(tasks[0])
else:
    print("Список пуст")
```

`len(tasks)` возвращает количество элементов, а `"Git" in tasks` проверяет принадлежность.

Срез `tasks[1:3]` создаёт новый верхний список и не включает правую границу.

## 03. Добавление, замена и удаление

Список можно менять на месте. Перед операцией сформулируйте, что известно: новое значение, позиция или существующее значение.

| Код | Назначение |
| --- | --- |
| tasks.append("Docker") | добавить один объект в конец |
| tasks[1] = "GitHub" | заменить элемент по позиции |
| tasks.pop() | удалить и вернуть последний элемент |
| tasks.pop(0) | удалить и вернуть элемент по индексу |
| tasks.remove("Git") | удалить первое совпадение по значению |

### Предскажите результат

```python
numbers = [1]
numbers.append([2, 3])
print(numbers)
print(len(numbers))
```

*Подсказка: append добавляет переданный список как один вложенный элемент.*

**Ответ**

```text
[1, [2, 3]]
2
```

### Сравнение решений

**Вопрос:** Как безопасно удалить Git, если значения может не быть?

### Без проверки

```python
tasks.remove("Git")
```

При отсутствии Git возникнет ValueError.

### С проверкой

```python
if "Git" in tasks:
    tasks.remove("Git")
```

Удаление выполняется только для существующего значения.

**Вывод:** предпочтителен правый вариант. remove работает по значению и требует, чтобы совпадение существовало.

> `pop()` пустого списка вызывает `IndexError`, а `remove(value)` без совпадения — `ValueError`.

## 04. Второе имя не создаёт копию

Список — изменяемый объект. Присваивание сохраняет ещё одну ссылку, поэтому изменение через одно имя видно через другое.

```python
first = ["Python"]
second = first
second.append("SQL")
print(first)
```

**Пошаговый разбор**

1. Создан один список ["Python"]. **Состояние:** first = → ["Python"]
2. second получает ссылку на тот же объект, содержимое не копируется. **Состояние:** first = → один список; second = → тот же список
3. append меняет общий объект. **Состояние:** first = → ["Python", "SQL"]; second = → ["Python", "SQL"]
4. Через first видно изменение, сделанное через second. **Состояние:** вывод = ['Python', 'SQL']

### Сравнение решений

**Вопрос:** Как получить отдельный верхний список?

### Второе имя

```python
second = first
```

Оба имени указывают на один объект.

### Поверхностная копия

```python
second = first.copy()
```

Верхние списки становятся независимыми.

**Вывод:** предпочтителен правый вариант. copy() создаёт новый список верхнего уровня. Вложенные изменяемые объекты могут остаться общими.

### Верно или неверно

После `copy = original.copy()` вызов `copy.append(value)` изменит сам `original`.

**Ответ:** неверно. append меняет отдельную верхнюю копию, поэтому original остаётся прежним.

## 05. Кортеж и множество решают другие задачи

`tuple` сохраняет порядок, но не позволяет заменить позицию. `set` хранит уникальные значения и удобен для проверки принадлежности.

### Последовательность проекта — list

Порядок важен, элементы добавляются и изменяются.

```python
tasks = ["Python", "Git"]
tasks.append("SQL")
```

### Фиксированная запись — tuple

Позиции имеют постоянный смысл. Кортеж из одного элемента требует запятую.

```python
screen_size = (1920, 1080)
single = (5,)
```

### Уникальность — set

Повторы схлопываются. Пустое множество создаётся через `set()`, а не `{}`.

```python
tags = {"python", "sql", "python"}
tags.add("git")
```

### Сопоставление

Соедините требование проекта с подходящей структурой.

| Требование | Структура |
| --- | --- |
| задачи в порядке создания | list |
| фиксированная пара ширина/высота | tuple |
| набор уникальных допустимых команд | set |

Выбор структуры определяется поведением данных.

### Разбор ошибки

```python
tags = {}
tags.add("python")
```

**Вопрос:** Почему вызов add не работает?

A. {} создаёт пустой словарь
B. Множества не поддерживают строки
C. Метод add требует два аргумента

**Ответ:** A. {} создаёт пустой словарь

Пустое множество создаётся вызовом set().

**Исправленный вариант**

```python
tags = set()
tags.add("python")
```

## 06. Уникальность с сохранением порядка

Запись `list(set(tags))` удаляет повторы, но не выражает требование сохранить первое появление. Для двух требований нужны две структуры.

```python
tags = ["python", "sql", "python", "git"]
result = []
seen = set()

for tag in tags:
    if tag not in seen:
        result.append(tag)
        seen.add(tag)
```

**Пошаговый разбор**

1. Исходный порядок задан списком tags. **Состояние:** tags = ["python", "sql", "python", "git"]
2. result будет хранить порядок первого появления. **Состояние:** result = []
3. seen отвечает только за проверку повторов. **Состояние:** seen = set()
4. Первый python отсутствует в seen, поэтому добавляется в обе структуры. **Состояние:** result = ["python"]; seen = {"python"}
5. sql тоже встречен впервые. **Состояние:** result = ["python", "sql"]; seen = {"python", "sql"}
6. Второй python уже есть в seen, тело if пропускается. **Состояние:** result = ["python", "sql"]
7. git добавляется как новое значение. **Состояние:** result = ["python", "sql", "git"]

### Соберите алгоритм уникальных тегов

Расположите строки так, чтобы порядок первого появления сохранился.

**Фрагменты**

- `result = []`
- `seen = set()`
- `for tag in tags:`
- `if tag not in seen:`
- `result.append(tag)`
- `seen.add(tag)`
- `result = list(set(tags))` — не выражает требование порядка

**Правильный порядок**

```python
result = []
seen = set()
for tag in tags:
    if tag not in seen:
        result.append(tag)
        seen.add(tag)
```

Список сохраняет порядок результата, множество сообщает, встречался ли тег раньше.

## 07. Практика и связь со StudyHub

В StudyHub список хранит задачи в порядке создания, кортеж может описывать фиксированную пару, а множество помогает отслеживать уже добавленные теги.

**локальный пример**

```python
tasks = ["Строки", "Boolean", "Циклы"]
allowed_statuses = ("new", "in_progress", "done")
seen_tags = set()

tasks.append("Коллекции")
for tag in ["python", "backend", "python"]:
    if tag not in seen_tags:
        seen_tags.add(tag)

print(tasks)
print(allowed_statuses)
print(len(seen_tags))
```

### Вспомните модель

**Вопрос:** Почему для удаления повторов с сохранением порядка используются и list, и set?

*Подсказка: У каждой структуры своя ответственность.*

**Ответ:** `list` сохраняет порядок первого появления, а `set` быстро сообщает, встречалось ли значение. Одна структура не выражает оба требования так ясно.

> В задаче платформы исходный список нельзя менять. Функция `solve(tags)` возвращает новый список, а для пустого входа возвращает `[]`.

## 08. Проверь себя

### Вопрос 1

Что делает append([1, 2])?

A. добавляет два числа
B. добавляет один вложенный список
C. заменяет весь список

**Ответ:** B. добавляет один вложенный список

append всегда добавляет переданный объект как один элемент.

### Вопрос 2

Чем pop отличается от remove?

A. pop работает по позиции и возвращает значение
B. remove всегда возвращает значение
C. разницы нет

**Ответ:** A. pop работает по позиции и возвращает значение

pop ориентируется на индекс, remove ищет первое совпадение по значению.

### Вопрос 3

Что создаёт запись second = first?

A. глубокую копию
B. поверхностную копию
C. второе имя одного объекта

**Ответ:** C. второе имя одного объекта

Содержимое не копируется.

### Вопрос 4

Как записать кортеж из одного элемента?

A. (5)
B. (5,)
C. [5]

**Ответ:** B. (5,)

Тип создаёт запятая: (5,) — tuple, (5) — int.

### Вопрос 5

Почему нельзя полагаться на list(set(tags)) при требовании порядка?

A. set не принимает строки
B. set отвечает за уникальность, а не пользовательский порядок
C. результат будет словарём

**Ответ:** B. set отвечает за уникальность, а не пользовательский порядок

Требование порядка нужно явно сохранять отдельным списком.

### Главное после занятия

- Список упорядочен и изменяем.
- `append` добавляет один объект.
- `pop` работает по позиции, `remove` — по значению.
- Присваивание не копирует изменяемый объект.
- Кортеж подходит для фиксированной записи.
- Множество выражает уникальность и принадлежность.
- Порядок и уникальность удобно разделить между списком и множеством.
