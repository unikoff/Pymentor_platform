# Превью месяца 1: основы Python и мышление программиста

## Зачем нужен этот месяц

Первый месяц не пытается сразу сделать из ученика backend-разработчика. Его задача важнее: убрать страх перед кодом и создать базу, без которой FastAPI, базы данных и авторизация будут выглядеть как набор чужих слов.

К концу месяца ученик не просто знает, что существуют переменные, циклы и функции. Он умеет разложить простую задачу на шаги, написать программу, запустить её из терминала, найти ошибку по traceback и сохранить результат в Git.

## Что будет создано

Сквозной результат месяца — **Console StudyHub Planner**, консольный планировщик учебных задач.

В нём можно:

- добавить задачу с названием и приоритетом;
- посмотреть список задач;
- найти задачу по идентификатору;
- отметить задачу выполненной;
- увидеть количество выполненных и оставшихся задач;
- закрыть программу через понятное меню.

Данные живут только во время работы программы. Это осознанное ограничение: JSON и файловое хранение появятся во втором месяце, когда ученик будет готов понять, что и почему сохраняется.

## Что ученик узнает

### Python как исполнитель инструкций

Ученик поймёт, как интерпретатор выполняет код сверху вниз, чем код отличается от данных, почему `print` не равен `return` и как платформа проверяет функцию `solve(...)`.

### Данные и правила

Ученик освоит числа, строки, логические значения, сравнения, условия и граничные случаи. Он научится переводить правило из человеческого языка в код: например, «задача срочная, если её приоритет не меньше 4 и она не завершена».

### Повторение и структуры данных

Ученик научится обходить список циклом, выбирать между `for` и `while`, работать со списками, множествами, кортежами и словарями. Одна задача будет представлена словарём, а все задачи — списком словарей.

### Функции и отладка

Ученик поймёт параметры, аргументы и `return`, начнёт делить программу на небольшие функции и не будет скрывать ошибки. Он разберёт `SyntaxError`, `NameError`, `TypeError`, `IndexError` и `KeyError` на понятных примерах.

### Инструменты разработчика

Ученик создаст папку проекта, запустит Python-скрипт из терминала, познакомится с виртуальным окружением, Git, `.gitignore`, README и GitHub.

## Что ученик сможет сделать сам

После месяца ученик должен уметь:

- написать функцию, которая получает данные и возвращает результат;
- объяснить, почему для задачи выбран список или словарь;
- проверить пустой, обычный и граничный сценарий;
- использовать `if/elif/else` для взаимоисключающих правил;
- использовать `for` для обработки набора данных и `while` для повторяющегося меню;
- создать простой CLI-проект без копирования кода из разных мест;
- прочитать последнюю строку traceback и найти место ошибки;
- сделать осмысленный Git-коммит и написать честную инструкцию запуска.

## Почему уроки идут именно в таком порядке

Сначала ученик понимает, как запустить код и что происходит с данными. Затем появляются условия и циклы: они дают программе возможность принимать решения и работать с несколькими значениями. После этого коллекции позволяют хранить задачи, а функции — собрать логику в читаемую структуру. Только после такой базы имеет смысл собирать меню и цельный проект.

Если начать с API, классов или базы данных раньше, ученик будет копировать шаблоны, не понимая, что происходит внутри `if`, цикла и функции.

## Как проходить занятия

Каждый урок рассчитан на 40–60 минут и состоит из теории, запускаемых примеров, автопроверяемой практики и квиза.

Рекомендуемый порядок:

1. Прочитать теорию и не пропускать примеры.
2. Запустить каждый пример и изменить хотя бы одно значение.
3. Сначала решить задачу платформы самостоятельно, затем использовать подсказки.
4. Пройти квиз и вернуться к объяснению, если ответ оказался неверным.
5. После блока тем применить навык в локальном StudyHub Planner.
6. Сделать Git-коммит, когда появилась законченная функция проекта.

Не нужно стремиться пройти все двадцать уроков быстрее. Если задача решена только после подсказки, это не провал: нужно повторить похожую задачу на следующий день без подсказки.

## Как проверяется результат

Платформа проверяет функции `solve(...)` на публичных и скрытых тестах. Публичные примеры показывают формат задачи; скрытые тесты проверяют пустые значения, границы и другие случаи, о которых ученик должен подумать сам.

Наставник проверяет другое:

- ученик объясняет ключевую функцию своими словами;
- проект запускается по README;
- невалидный ввод не ломает меню;
- пустой список задач обработан;
- комментарии и имена функций помогают читать код;
- Git-история отражает этапы работы, а не один финальный коммит.

## Что не нужно знать после первого месяца

Ученик пока не обязан знать классы, JSON, базы данных, HTTP, FastAPI, декораторы, асинхронность или Docker. Отсутствие этих тем не является пробелом: они будут изучаться после того, как ученик уверенно работает с данными, ветвлениями, циклами и функциями.

## Переход ко второму месяцу

Во втором месяце Console StudyHub Planner перестанет быть одним файлом и временным списком в памяти. Ученик научится:

- разбивать проект на модули;
- сохранять задачи в JSON и загружать их при старте;
- обрабатывать ожидаемые ошибки при работе с файлами;
- применять классы и `dataclass` там, где словаря уже недостаточно;
- писать первые тесты через `pytest`;
- изучать `*args`, `**kwargs`, comprehensions, генераторы и декораторы.

Таким образом, знания первого месяца не «закрываются». Они становятся материалом для реального кода: список словарей превратится в сохраняемые данные, функции будут покрываться тестами, а Git-репозиторий продолжит жить как история проекта.
