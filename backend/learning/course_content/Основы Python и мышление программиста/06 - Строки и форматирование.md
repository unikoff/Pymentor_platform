# Строки и форматирование

Научимся работать с текстом как с последовательностью символов: находить отдельные буквы, брать фрагменты, очищать пользовательский ввод и собирать понятные сообщения.

**Темы:** индексы и срезы · методы и f-строки

## 01. Что изменится после занятия

В первом блоке строка была просто текстовым значением. Теперь научимся разбирать её на части и аккуратно подготавливать для проекта. Представьте надпись на длинной ленте: каждый символ занимает своё место, а Python умеет взять одну позицию или вырезать целый фрагмент.

1. **Найти символ:** получить одну букву по её позиции через `text[index]`.
2. **Взять фрагмент:** получить часть текста через `text[start:end]`.
3. **Подготовить ввод:** убрать лишние пробелы и сохранить новый результат.
4. **Собрать сообщение:** вставить значения в готовый шаблон через f-строку.

В конце вы подготовите название задачи StudyHub и соберёте из него готовую текстовую карточку.

> На этом уроке достаточно менять одну вещь за раз: один индекс, одну границу среза или один строковый метод.

## 02. Строка похожа на ряд пронумерованных ячеек

Представьте ряд ящиков, в каждом из которых лежит один символ. Номер ящика называется индексом. В Python нумерация начинается с нуля.

**позиции символов**

```python
P  y  t  h  o  n
0  1  2  3  4  5
-6 -5 -4 -3 -2 -1
```

### Почему первый индекс равен 0

Индекс можно читать как количество шагов от начала строки. До первого символа нужно сделать **ноль шагов**, поэтому его индекс — `0`.

### Отрицательные индексы

Если считать удобнее с конца, последний символ имеет индекс `-1`, предпоследний — `-2`.

### Предскажите результат

```python
language = "Python"
print(language[0])
print(language[2])
print(language[-1])
```

*Подсказка: 0 — первый символ, 2 — третий, -1 — последний.*

**Ответ**

```text
P
t
n
```

### Разбор ошибки

```python
title = "Git"
print(title[3])
```

**Вопрос:** Почему Python не может получить символ с индексом 3?

A. У строки Git существуют только позиции 0, 1 и 2
B. Индекс нужно записать в кавычках
C. Строки нельзя передавать в print

**Ответ:** A. У строки Git существуют только позиции 0, 1 и 2

Длина строки равна трём, но последний существующий индекс равен двум.

**Исправленный вариант**

```python
title = "Git"
print(title[-1])  # t
```

> Если запрашиваемой позиции нет, Python сообщает `IndexError`. У пустой строки `""` нет даже индекса `0`.

## 03. Срез — фрагмент между двумя границами

Одиночный индекс похож на выбор одного ящика. Срез похож на отрезок ленты: мы указываем, где начать и перед какой позицией остановиться.

### Исследование среза

Строка: `backend`

Срез: `backend[0:4]` → `back`

### Левая граница входит

В `text[0:4]` символ с индексом `0` попадёт в результат.

### Правая граница не входит

Индекс `4` показывает место остановки. Поэтому `"backend"[0:4]` возвращает `"back"`.

### Одну границу можно пропустить

`text[:3]` означает «от начала до позиции 3», а `text[4:]` — «от позиции 4 до конца».

### Начало строки — [0:4]

Берём символы с позиций 0, 1, 2 и 3.

```python
"backend"[0:4]  # "back"
```

### Начало можно не писать — [:3]

Пустое место слева означает начало строки.

```python
"Python"[:3]  # "Pyt"
```

### Конец можно не писать — [3:]

Пустое место справа означает конец строки.

```python
"Python"[3:]  # "hon"
```

### Верно или неверно

Срез `"Python"[2:2]` вызывает `IndexError`.

**Ответ:** неверно. Начало и конец совпадают, поэтому получается пустая строка.

> Срезы мягче одиночного индекса: `"Git"[0:20]` просто остановится на конце строки, а `"Git"[20]` вызовет ошибку.

## 04. Строка не меняется по одной букве

Строку удобно представить как уже напечатанную наклейку. Нельзя стереть только одну букву внутри готовой наклейки. Нужно создать новую надпись и при необходимости сохранить её под тем же именем.

```python
title = "python"
title.upper()
print(title)
title = title.upper()
print(title)
```

**Пошаговый разбор**

1. title получает строку "python". **Состояние:** title = "python"
2. upper() создаёт новую строку "PYTHON", но мы её пока не сохранили. **Состояние:** title = "python"; новая строка = "PYTHON"
3. Поэтому печатается прежнее значение. **Состояние:** title = "python"; вывод = python
4. Теперь новое значение присваивается обратно переменной title. **Состояние:** title = "PYTHON"
5. Печатается уже сохранённая строка. **Состояние:** title = "PYTHON"; вывод = PYTHON

### Сравнение решений

**Вопрос:** Какой вариант сохраняет результат upper()?

### Новая строка потеряна

```python
title = "python"
title.upper()
```

Метод создал результат, но мы не дали ему имя.

### Новая строка сохранена

```python
title = "python"
title = title.upper()
```

Переменная title теперь хранит новое значение.

**Вывод:** предпочтителен правый вариант. Строковые методы возвращают новую строку. Результат нужно присвоить переменной или сразу использовать.

### Разбор ошибки

```python
title = "python"
title[0] = "P"
```

**Вопрос:** Почему нельзя заменить первый символ таким способом?

A. Строки неизменяемы
B. Индекс 0 не существует
C. Буквы можно менять только внутри print

**Ответ:** A. Строки неизменяемы

Python не меняет отдельный символ готовой строки.

**Исправленный вариант**

```python
title = "python"
title = "P" + title[1:]
print(title)  # Python
```

## 05. Строковые методы — инструменты для нового текста

Методы можно представить как инструменты на рабочем столе. Каждый выполняет одно действие и возвращает новую строку. Перед использованием сначала проговорите, какую проблему нужно решить.

### Очистить края — strip

Убирает пробельные символы только в начале и в конце.

```python
"  SQL  ".strip()  # "SQL"
```

### Нижний регистр — lower

Удобно для сравнений, когда регистр не должен влиять на результат.

```python
"Python".lower()  # "python"
```

### Верхний регистр — upper

Создаёт новую строку из заглавных букв.

```python
"sql".upper()  # "SQL"
```

### Заменить фрагмент — replace

Находит указанный фрагмент и создаёт строку с заменой.

```python
"Python course".replace("course", "backend")
```

**понятная последовательность**

```python
raw_title = "  Изучить Python  "
clean_title = raw_title.strip()
normalized_title = clean_title.lower()

print(raw_title)
print(clean_title)
print(normalized_title)
```

### Почему полезны отдельные переменные

Новичку легче увидеть три этапа: исходный текст, очищенный текст и нормализованный текст. Позже действия можно объединять в цепочку.

### Что strip() не делает

Метод не удаляет пробелы между словами. Строка `" Python backend "` сохранит внутренние пробелы после очистки краёв.

### Не меняйте текст без правила

Автоматическое `capitalize()` может превратить `"SQL"` в `"Sql"`. Сначала определите, допустимо ли это для данных проекта.

## 06. F-строка — шаблон с местами для значений

Представьте готовый бланк: «Задача: ___; приоритет: ___». F-строка позволяет оставить в тексте места и подставить туда значения переменных.

**карточка задачи**

```python
title = "Изучить строки"
priority = 4

message = f"Задача: {title}; приоритет: {priority}"
print(message)
```

### Буква f включает подстановку

Без буквы `f` фигурные скобки останутся обычным текстом.

### В фигурных скобках находится выражение

Python берёт значение `title`, затем значение `priority` и вставляет их в готовую строку.

### Число не нужно преобразовывать вручную

F-строка умеет вставлять и строки, и числа. Поэтому запись читается проще, чем сложение нескольких частей через `+`.

### Сравнение решений

**Вопрос:** Какой вариант проще прочитать и изменить?

### Склеивание частей

```python
"Задача: " + title + "; приоритет: " + str(priority)
```

Нужно вручную преобразовать число и следить за пробелами.

### Шаблон f-строки

```python
f"Задача: {title}; приоритет: {priority}"
```

Видно итоговую форму сообщения.

**Вывод:** предпочтителен правый вариант. F-строка показывает шаблон целиком и сама преобразует вставляемые значения в текст.

### Заполните пропуск

Вставьте значение переменной title в f-строку.

```python
message = f"Задача: {___}"
```

**Варианты:** `title`, `"title"`, `print`

**Ответ:** `title`

В фигурных скобках указывается имя переменной без кавычек.

## 07. split() и join(): разделить и собрать

Эти методы удобно представить как ножницы и соединительную ленту. `split()` разделяет одну строку на части, а `join()` соединяет несколько строк выбранным разделителем.

**теги StudyHub**

```python
tags_text = "python,sql,git"
tags = tags_text.split(",")
formatted_tags = " | ".join(tags)

print(tags)
print(formatted_tags)
```

### Что возвращает split()

Python получает несколько частей внутри квадратных скобок: `["python", "sql", "git"]`. Это список. Подробно списки будут изучаться в следующем блоке; сейчас достаточно увидеть результат.

### Кто вызывает join()

Метод вызывается у строки-разделителя: `" | ".join(tags)`. Python помещает эту строку между частями.

**практика StudyHub**

```python
raw_title = "  Изучить строки  "
priority = 4

title = raw_title.strip()
card = f"Задача: {title}; приоритет: {priority}"
print(card)
```

### Вспомните модель

**Вопрос:** Чем индекс отличается от среза?

*Подсказка: Подумайте, сколько символов получается и что происходит при слишком большой границе.*

**Ответ:** Индекс получает один символ и требует существующую позицию. Срез получает фрагмент, не включает правую границу и может спокойно остановиться на конце строки.

> В задаче платформы используйте аргументы функции и `return`. Не добавляйте `input()`, если значение уже передано в `solve(...)`.

## 08. Проверь основную модель

Вопросы проверяют только главные действия: получить символ, взять фрагмент, сохранить результат метода и собрать сообщение.

### Вопрос 1

Что вернёт "Python"[0]?

A. P
B. y
C. Python

**Ответ:** A. P

Индекс 0 обозначает первый символ.

### Вопрос 2

Что вернёт "backend"[0:4]?

A. back
B. backe
C. end

**Ответ:** A. back

Правая граница 4 не включается.

### Вопрос 3

Что произойдёт после title.upper(), если результат не сохранить?

A. title останется прежним
B. title изменится навсегда
C. возникнет IndexError

**Ответ:** A. title останется прежним

Метод возвращает новую строку, но исходная переменная не меняется.

### Вопрос 4

Что удаляет strip()?

A. Пробельные символы по краям
B. Все пробелы внутри строки
C. Все буквы нижнего регистра

**Ответ:** A. Пробельные символы по краям

Внутренние пробелы сохраняются.

### Вопрос 5

Зачем нужна буква f перед строкой?

A. Разрешить подстановку значений в фигурные скобки
B. Сделать все буквы заглавными
C. Разделить строку на части

**Ответ:** A. Разрешить подстановку значений в фигурные скобки

F-строка вычисляет выражения в фигурных скобках и вставляет результат в текст.

### Главное после занятия

- Строка состоит из символов с позициями, а первый индекс равен `0`.
- Отрицательный индекс `-1` обозначает последний символ.
- Срез получает фрагмент и не включает правую границу.
- Строка неизменяема: методы возвращают новое значение.
- `strip()` очищает только края строки.
- F-строка вставляет значения в понятный текстовый шаблон.
- `split()` разделяет строку, а `join()` соединяет части.
