# Ветвления if / elif / else

Научимся давать программе выбор: выполнить один путь, другой путь или запасной вариант. Разберём if, elif, else и роль отступов.

**Темы:** if · elif · else · порядок и границы

## 01. Маршрут занятия

Ветвление похоже на развилку дороги. Условие решает, по какой дороге пойдёт программа. Сначала мы задаём простой вопрос, затем описываем действия для ответа «да» и для остальных случаев.

1. **Проверить условие:** получить `True` или `False`.
2. **Выбрать структуру:** использовать отдельный `if`, пару `if/else` или цепочку `if/elif/else`.
3. **Проверить порядок:** поставить узкие условия раньше широких и протестировать границы.

В конце вы классифицируете приоритет StudyHub, обработаете неизвестный статус и найдёте недостижимую ветку.

> На первом проходе достаточно помнить: отдельные `if` задают независимые вопросы, а `if/elif/else` выбирает один путь из нескольких.

## 02. if выполняет блок по условию

`if` можно читать как слово «если». После него стоит вопрос с ответом True или False. Двоеточие открывает список действий, а отступ показывает, какие строки принадлежат этому условию. Отступ похож на вложенный пункт обычного списка.

```python
priority = 5

if priority == 5:
    print("Высокий приоритет")

print("Обработка завершена")
```

```python
priority = 5
if priority == 5:
    print("Высокий приоритет")
print("Обработка завершена")
```

**Пошаговый разбор**

1. priority получает значение 5. **Состояние:** priority = 5
2. Сравнение 5 == 5 даёт True. **Состояние:** условие = True
3. Строка с отступом выполняется. **Состояние:** вывод = Высокий приоритет
4. Строка без отступа находится вне if и выполняется всегда. **Состояние:** вывод = Высокий приоритет ⏎ Обработка завершена

### Разбор ошибки

```python
if priority >= 4
    print("Высокий")
```

**Вопрос:** Почему Python не может прочитать условие?

A. После условия отсутствует двоеточие
B. print нельзя использовать внутри if
C. Сравнение >= запрещено

**Ответ:** A. После условия отсутствует двоеточие

Двоеточие отделяет условие от начинающегося блока.

**Исправленный вариант**

```python
if priority >= 4:
    print("Высокий")
```

## 03. else обрабатывает остаточный случай

`else` означает «иначе». Это запасная дверь: если условие `if` не подошло, программа выполняет блок `else`. Нового вопроса после else нет.

```python
is_done = False

if is_done:
    message = "Задача выполнена"
else:
    message = "Задача открыта"

print(message)
```

### Предскажите результат

```python
age = 17
if age >= 18:
    access = "разрешён"
else:
    access = "запрещён"
print(access)
```

*Подсказка: 17 не проходит проверку age >= 18.*

**Ответ**

```text
запрещён
```

### Заполните пропуск

Добавьте ветку для всех случаев, не прошедших условие

```python
if is_done:
    message = "Готово"
___:
    message = "Открыто"
```

**Варианты:** `else`, `elif`, `and`

**Ответ:** `else`

else не проверяет новое условие и ловит остаточный случай.

## 04. elif создаёт цепочку вариантов

`elif` добавляет ещё один вариант. Python читает цепочку сверху вниз и выбирает первую подходящую дверь. После этого остальные двери уже не проверяются.

### Разбор ветвей

```python
if priority == 5:
    label = "critical"
elif priority >= 3:
    label = "normal"
else:
    label = "low"
```

| Сценарий | Выполняется | Результат |
| --- | --- | --- |
| priority = 5 | строка 1 | "critical" |
| priority = 4 | строка 3 | "normal" |
| priority = 1 | строка 5 | "low" |

При `priority = 4` первая проверка ложна, вторая истинна, поэтому присваивается `"normal"`.

После выбора ветки Python не возвращается к остальным условиям этой цепочки.

### Вспомните модель

**Вопрос:** Сколько веток может выполниться в одной цепочке if/elif/else?

**Ответ:** Не более одной: первой, чьё условие оказалось истинным, либо else.

## 05. Строгие условия ставятся раньше

Представьте сортировку посылок: сначала нужно проверить особую категорию «хрупкое», а уже затем общую категорию «обычная посылка». Слишком широкое условие сверху может забрать значения, предназначенные следующей ветке.

### Сравнение решений

**Вопрос:** Какой порядок корректно классифицирует 95 баллов?

### Широкая граница раньше

```python
if score >= 60:
    grade = "зачёт"
elif score >= 90:
    grade = "отлично"
```

95 сразу попадает в первую ветку.

### Строгая граница раньше

```python
if score >= 90:
    grade = "отлично"
elif score >= 60:
    grade = "зачёт"
```

Сначала проверяется более узкая категория.

**Вывод:** предпочтителен правый вариант. Условие >= 60 включает и 95, поэтому его нельзя ставить раньше >= 90.

Для границ `60` и `90` проверьте значения `59, 60, 61, 89, 90, 91`.

Если предыдущая ветка включает все значения следующей, следующая ветка недостижима.

### Верно или неверно

В цепочке категорий условие `score >= 60` можно поставить раньше `score >= 90`, потому что Python всё равно проверит обе ветки.

**Ответ:** неверно. После первой истинной ветки остальные условия цепочки не проверяются.

## 06. Независимые if и вложенные решения

Два отдельных `if` похожи на два независимых вопроса: оба ответа могут быть «да». Цепочка `if/elif/else` похожа на выбор одной категории: программа выбирает только один вариант.

### Независимые признаки — if + if

Обе надписи могут появиться одновременно.

```python
if priority >= 4:
    print("Высокий приоритет")

if not is_done:
    print("Задача открыта")
```

### Одна категория — if / elif

Выбирается ровно один текстовый уровень.

```python
if priority >= 4:
    label = "high"
elif priority >= 2:
    label = "medium"
else:
    label = "low"
```

**вложенное решение**

```python
if user_exists:
    if password_correct:
        print("Доступ разрешён")
    else:
        print("Неверный пароль")
else:
    print("Пользователь не найден")
```

> Глубокая лестница отступов быстро становится тяжёлой. Иногда правило лучше разложить на boolean-переменные или объединить через `and`.

Сначала реализуйте классификацию приоритета, затем отдельно обработайте текстовый статус. Не объединяйте две разные задачи в одну длинную цепочку.

**категория приоритета**

```python
priority = 4

if priority == 5:
    label = "critical"
elif priority >= 3:
    label = "normal"
elif priority >= 1:
    label = "low"
else:
    label = "invalid"

```

### Разбор ошибки

```python
status = "done"
if status == "new":
    message = "Новая задача"
if status == "done":
    message = "Задача выполнена"
else:
    message = "Неизвестный статус"
```

**Вопрос:** Почему структура плохо выражает взаимоисключающие статусы?

A. Строки нельзя сравнивать
B. else относится только ко второму if, а не ко всей классификации
C. Переменная message запрещена

**Ответ:** B. else относится только ко второму if, а не ко всей классификации

Для одной категории статуса нужна общая цепочка if/elif/else.

**Исправленный вариант**

```python
if status == "new":
    message = "Новая задача"
elif status == "done":
    message = "Задача выполнена"
else:
    message = "Неизвестный статус"
```

> В задачах платформы верните выбранную строку через `return`. Значения вне диапазона должны иметь явный результат, например `"invalid"`.

## 08. Проверь себя

### Вопрос 1

Когда выполняется else?

A. всегда
B. если все условия выше ложны
C. после первого if

**Ответ:** B. если все условия выше ложны

else обрабатывает остаточный случай.

### Вопрос 2

Что определяет блок в Python?

A. фигурные скобки
B. отступ
C. точка с запятой

**Ответ:** B. отступ

Отступ является частью синтаксиса.

### Вопрос 3

Почему >= 90 ставят раньше >= 60?

A. так быстрее
B. >= 60 иначе перехватит 90 и выше
C. Python требует убывающий порядок

**Ответ:** B. >= 60 иначе перехватит 90 и выше

Широкое условие включает значения строгого.

### Вопрос 4

Когда нужны два отдельных if?

A. для независимых правил
B. для одной категории
C. вместо любого else

**Ответ:** A. для независимых правил

Независимые проверки могут выполниться одновременно.

### Главное после занятия

- `if` выполняет блок при истинном условии.
- `else` обрабатывает остаточный случай.
- `elif` добавляет проверку в одну цепочку.
- Отступ определяет принадлежность строки блоку.
- Более строгие условия ставятся раньше широких.
