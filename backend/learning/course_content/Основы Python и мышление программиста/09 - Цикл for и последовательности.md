# Цикл for и последовательности

Научимся повторять одно действие для нескольких значений: брать элементы по одному, считать сумму и нумеровать задачи.

**Темы:** for · range · накопление

## 01. Маршрут занятия

Представьте стопку карточек с задачами. Вместо трёх одинаковых команд можно брать по одной карточке и выполнять одно и то же действие. Цикл `for` делает именно это.

1. **Получить текущий элемент:** проследить, как переменная цикла меняется на каждом шаге.
2. **Повторить общий блок:** читать тело `for` по отступу и понимать поведение пустой последовательности.
3. **Накопить результат:** создать число или новую последовательность до цикла и обновлять её на каждом шаге.

В конце вы посчитаете сумму приоритетов StudyHub, пронумеруете задачи через `enumerate` и объясните, почему нельзя обнулять накопитель внутри цикла.

> Перед запуском короткого цикла выпишите три строки вручную: какой элемент взяли, что сделали и каким стал результат. Это проще, чем пытаться удержать весь цикл в голове.

## 02. Один элемент на каждой итерации

Цикл похож на конвейер: элементы приезжают по одному. Переменная цикла — рабочее место, на котором сейчас лежит только один текущий элемент.

```python
tasks = ["Python", "Git", "SQL"]
for task in tasks:
    print(task)
```

**Пошаговый разбор**

1. Создаётся последовательность из трёх строк. **Состояние:** tasks = ["Python", "Git", "SQL"]
2. Первая итерация: task = "Python". **Состояние:** task = "Python"
3. Печатается первый элемент. **Состояние:** вывод = Python
4. Вторая итерация: task = "Git". **Состояние:** task = "Git"; вывод = Python
5. Третья итерация: task = "SQL". После неё цикл завершается. **Состояние:** task = "SQL"; вывод = Python ⏎ Git ⏎ SQL

Для пустой последовательности тело цикла не выполнится ни разу. Это нормальный сценарий.

Имя переменной должно описывать один элемент: `task` для `tasks`, `char` для символов, `priority` для приоритетов.

### Вспомните модель

**Вопрос:** Что хранит переменная task внутри for task in tasks?

**Ответ:** Один текущий элемент последовательности на конкретной итерации.

## 03. Строки и range тоже можно обходить

Цикл может обходить разные последовательности. Строка отдаёт буквы по одной, а `range` создаёт ряд чисел. Его можно представить как талончики с номерами.

### Предскажите результат

```python
for char in "SQL":
    print(char)
```

*Подсказка: Строка отдаёт символы слева направо.*

**Ответ**

```text
S
Q
L
```

### Сопоставление

Соедините вызов range с получаемыми значениями.

| Левая часть | Правая часть |
| --- | --- |
| range(5) | 0, 1, 2, 3, 4 |
| range(2, 6) | 2, 3, 4, 5 |
| range(2, 10, 2) | 2, 4, 6, 8 |

Правая граница range не включается.

> Шаг `0` недопустим. Python не сможет перейти к следующему значению и сообщит об ошибке.

## 04. Накопление числа

Накопитель похож на копилку. До начала она пустая, поэтому сумма равна `0`. На каждом шаге мы добавляем очередное значение. Если поставить пустую копилку внутрь цикла, она будет обнуляться снова и снова.

```python
priorities = [1, 3, 2]
total = 0
for priority in priorities:
    total = total + priority
print(total)
```

**Пошаговый разбор**

1. До цикла total равен 0. **Состояние:** total = 0
2. Первый элемент priority = 1. **Состояние:** priority = 1; total = 0
3. total становится 1. **Состояние:** priority = 1; total = 1
4. После второго элемента total равен 4. **Состояние:** priority = 3; total = 4
5. После третьего элемента total равен 6. **Состояние:** priority = 2; total = 6
6. Цикл завершён, печатается 6. **Состояние:** total = 6; вывод = 6

### Разбор ошибки

```python
priorities = [1, 3, 2]
for priority in priorities:
    total = 0
    total = total + priority
print(total)
```

**Вопрос:** Почему код не накапливает общую сумму?

A. В цикле нельзя складывать числа
B. total обнуляется на каждой итерации
C. print должен находиться внутри цикла

**Ответ:** B. total обнуляется на каждой итерации

Начальное значение накопителя должно создаваться один раз до цикла.

**Исправленный вариант**

```python
priorities = [1, 3, 2]
total = 0
for priority in priorities:
    total = total + priority
print(total)
```

### Верно или неверно

Для пустого списка сумма останется равна начальному значению `0`.

**Ответ:** верно. Тело цикла не выполнится, поэтому накопитель не изменится.

## 05. Накопление нового результата

Иногда результат — не одно число, а новый список. Это похоже на перекладывание обработанных карточек в отдельную коробку: исходная стопка остаётся на месте, а новые значения складываются отдельно.

**форматируем названия**

```python
titles = ["Python", "SQL"]
formatted = []

for title in titles:
    formatted.append(title.upper())

print(formatted)
```

### Сравнение решений

**Вопрос:** Какой вариант сохраняет исходные данные и явно строит результат?

### Менять во время обхода

```python
for title in titles:
    titles.remove(title)
```

Позиции сдвигаются, элементы могут быть пропущены.

### Собирать отдельно

```python
result = []
for title in titles:
    result.append(title.upper())
```

Исходная последовательность остаётся прежней.

**Вывод:** предпочтителен правый вариант. Изменение коллекции во время обхода делает поведение трудно предсказуемым.

> Подробные методы списков будут изучаться отдельно. Сейчас запомните шаблон: пустой результат до цикла, преобразование элемента и `append()`.

## 06. enumerate: позиция и элемент

Для нумерованного списка нужны сразу две вещи: номер и сама задача. `enumerate` похож на сотрудника, который выдаёт каждой карточке порядковый номер и передаёт карточку дальше.

```python
titles = ["Python", "Git"]

for position, title in enumerate(titles, start=1):
    print(f"{position}. {title}")
```

### Предскажите результат

```python
for i, title in enumerate(["SQL", "Git"], start=1):
    print(f"{i}. {title}")
```

*Подсказка: start=1 задаёт первое отображаемое число.*

**Ответ**

```text
1. SQL
2. Git
```

### Сравнение решений

**Вопрос:** Какой вариант проще, если нужен и номер, и название?

### Ручной счётчик

```python
position = 1
for title in titles:
    print(f"{position}. {title}")
    position += 1
```

Работает, но требует отдельного обновления.

### enumerate

```python
for position, title in enumerate(titles, start=1):
    print(f"{position}. {title}")
```

Позиция и элемент получаются одной конструкцией.

**Вывод:** предпочтителен правый вариант. enumerate уменьшает количество состояния, которым нужно управлять вручную.

## 07. Практика StudyHub

Сначала посчитайте суммарный приоритет, затем соберите нумерованные названия. Это два разных накопления: число и новая последовательность.

### Соберите алгоритм суммы

Выберите строки в порядке, который корректно накапливает сумму приоритетов.

**Фрагменты**

- `total = 0`
- `for priority in priorities:`
- `total = total + priority`
- `print(total)`
- `total = 0` — обнуляет накопитель внутри цикла

**Правильный порядок**

```python
total = 0
for priority in priorities:
    total = total + priority
print(total)
```

Накопитель создаётся один раз, затем обновляется на каждой итерации.

**нумерация задач**

```python
tasks = ["Строки", "Boolean", "Ветвления"]

for position, title in enumerate(tasks, start=1):
    print(f"{position}. {title}")
```

> В задаче платформы верните итог через `return`. Для пустого списка сумма должна быть `0`, а новый список — пустым.

## 08. Проверь себя

### Вопрос 1

Что получает переменная цикла?

A. всю последовательность
B. один элемент за шаг
C. только индекс

**Ответ:** B. один элемент за шаг

for перебирает элементы по одному.

### Вопрос 2

Какие значения даёт range(3)?

A. 1, 2, 3
B. 0, 1, 2
C. 0, 1, 2, 3

**Ответ:** B. 0, 1, 2

Правая граница не включается.

### Вопрос 3

Где создаётся total = 0?

A. до цикла
B. в каждой итерации
C. после цикла

**Ответ:** A. до цикла

Иначе накопитель будет обнуляться.

### Вопрос 4

Когда нужен enumerate?

A. когда нужны позиция и элемент
B. только для строк
C. когда нужен только элемент

**Ответ:** A. когда нужны позиция и элемент

enumerate возвращает пару: позицию и текущий элемент.

### Главное после занятия

- `for` получает элементы по одному.
- Тело цикла определяется отступом.
- Правая граница `range` не включается.
- Накопитель создаётся до цикла.
- `enumerate` даёт позицию и элемент.
