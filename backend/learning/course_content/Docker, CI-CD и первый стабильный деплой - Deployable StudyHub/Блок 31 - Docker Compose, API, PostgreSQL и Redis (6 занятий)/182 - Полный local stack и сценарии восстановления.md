## 182. Полный local stack и сценарии восстановления

### Цель

Собрать Compose-проект, который другой разработчик поднимает и диагностирует одной процедурой.

### Почему тема появляется сейчас

Финальная точка блока проверяет не отдельный YAML, а воспроизводимость всей системы.

### Основное содержание

- единая команда запуска;
- build и pull;
- просмотр logs нескольких services;
- health status;
- migrations status;
- reset dev environment;
- backup перед reset;
- README quick start;

### Практика StudyHub

Клонировать проект в чистую папку, поднять stack, применить migrations, выполнить API scenario, перезапустить и восстановить после намеренной ошибки.

### Визуальная и интерактивная подача

Панель состояния services; диагностическое дерево API/db/redis/migrations; финальный маршрут Compose.

### Результат занятия

Создан `compose.yaml`, `.env.example` и проверенный quick start.

### Граница занятия

Production deployment пока не выполняется.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;

## Контрольная точка блока

- Одна Compose-команда поднимает API, PostgreSQL, Redis, migrations и проверки готовности.
- есть основной успешный сценарий;
- есть ожидаемый сбой и понятная диагностика;
- результат воспроизводится другим человеком;
- README, схема, test matrix или runbook обновлены;
- ученик прослеживает путь данных и ответственность компонентов без подсказки;
