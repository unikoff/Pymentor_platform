## 179. Volumes и постоянные данные PostgreSQL

### Цель

Сохранить database data между пересозданиями container и научиться осознанно сбрасывать окружение.

### Почему тема появляется сейчас

Database работает, но без volume её состояние связано с жизнью container.

### Основное содержание

- named volume;
- mount point PostgreSQL;
- container lifecycle против data lifecycle;
- `down` и `down -v`;
- просмотр volumes;
- backup не равен volume;
- development reset;
- опасность случайного удаления;

### Практика StudyHub

Создать запись, пересоздать containers и подтвердить сохранность; затем выполнить контролируемый полный reset.

### Визуальная и интерактивная подача

Две линии жизни container и volume; True/False сценарии удаления.

### Результат занятия

Ученик умеет сохранить или намеренно очистить локальные данные.

### Граница занятия

Не считать volume резервной копией.
