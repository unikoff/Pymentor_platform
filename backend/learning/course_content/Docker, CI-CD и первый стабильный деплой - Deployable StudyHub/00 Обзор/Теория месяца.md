# Этап 8. Docker, CI/CD и первый стабильный деплой

## Общая цель

Этап делает Async StudyHub воспроизводимым вне компьютера автора. Ученик сначала осваивает Linux-среду, процессы, configuration и logs, затем упаковывает API в Docker image, соединяет API, PostgreSQL и Redis через Compose и завершает этап автоматическими проверками, сборкой image, deployment, smoke test и rollback.

## Сквозной проект

`Deployable StudyHub`

## Методика этапа

Каждый инфраструктурный слой появляется после наблюдаемой проблемы: Linux объясняет среду процесса, Docker фиксирует runtime, Compose соединяет services, CI проверяет commit, а CD доставляет только уже проверенный artifact. Ученик всегда сначала запускает и диагностирует минимальный сценарий, затем переносит его в StudyHub.

```text
понять проблему
→ увидеть минимальную модель
→ предсказать поведение
→ запустить отдельный пример
→ встроить в StudyHub
→ проверить сбой
→ автоматизировать проверку
→ объяснить решение
```
