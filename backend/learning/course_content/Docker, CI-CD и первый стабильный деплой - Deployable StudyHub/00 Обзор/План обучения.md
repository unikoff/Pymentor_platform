# План развития StudyHub: этапы 6–9

## Общая рамка

После этапа 5 у ученика уже есть `Personal StudyHub API`: синхронный FastAPI, SQLite/SQLAlchemy, Alembic, пользователи, password hashing, cookie-сессии, JWT access/refresh, права доступа, тесты и воспроизводимый запуск.

Оставшиеся четыре этапа не повторяют регистрацию, JWT и базовый CRUD. Они последовательно решают новые проблемы уже работающего проекта:

```text
Personal StudyHub API
→ PostgreSQL StudyHub
→ Async StudyHub
→ Deployable StudyHub
→ StudyHub LMS Release
→ портфолио и собеседования
```

## Формат

- 4 основных блока на этап;
- 6 занятий в блоке;
- 24 занятия на этап;
- блоки 21–36;
- занятия 117–212;
- отдельный нулевой блок `LearningRoadmap + MonthTheory` для каждого этапа;
- один сквозной проект `StudyHub`;
- после каждого блока — практикум, проверка ошибок и code review;
- переход дальше только после контрольной точки.

# Этап 8. Docker, CI/CD и первый стабильный деплой

## Цель

Сделать приложение воспроизводимым для другого разработчика и для сервера. Этап не превращается в отдельный курс DevOps: изучается инфраструктура, необходимая одному FastAPI-монолиту.

## Блок 29. Linux, процессы, окружения и логи

Занятия 165–170.

- Linux filesystem;
- процессы и сигналы;
- ports;
- environment variables;
- stdout/stderr;
- dev/test/prod;
- logging levels;
- healthcheck;
- secrets и `.env.example`.

**Результат:** StudyHub уверенно запускается в Linux-подобном окружении и пишет диагностируемые логи.

## Блок 30. Dockerfile и контейнер приложения

Занятия 171–176.

- layers;
- build context;
- Dockerfile;
- `WORKDIR`, `COPY`, `RUN`, `CMD`;
- `.dockerignore`;
- ports;
- rebuild;
- logs;

**Результат:** FastAPI запускается из собственного Docker image.
