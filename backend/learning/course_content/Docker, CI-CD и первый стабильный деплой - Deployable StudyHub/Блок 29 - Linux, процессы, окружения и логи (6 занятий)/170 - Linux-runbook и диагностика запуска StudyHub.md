## 170. Linux-runbook и диагностика запуска StudyHub

### Цель

Собрать базовые инструменты в воспроизводимую инструкцию запуска и поиска проблемы.

### Почему тема появляется сейчас

Блок должен закончиться не набором команд, а процедурой, которой воспользуется другой разработчик.

### Основное содержание

- проверка Python и dependencies;
- проверка cwd и config;
- проверка процесса и порта;
- проверка database connection;
- чтение последних логов;
- healthcheck;
- типовые причины failed startup;
- runbook и checklist;

### Практика StudyHub

С чистой директории запустить StudyHub по runbook, затем по очереди исправить неверный cwd, занятый port, отсутствующую variable и недоступную database.

### Визуальная и интерактивная подача

Диагностическое дерево «команда не запускается / процесс падает / process жив, API не отвечает»; TerminalDemo полного расследования.

### Результат занятия

Создан `docs/runbook-linux.md`, по которому другой человек запускает и диагностирует StudyHub.

### Граница занятия

Новая функциональность API не добавляется. Цель — эксплуатационная ясность.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;

## Контрольная точка блока

- StudyHub запускается в Linux-подобном окружении, получает конфигурацию извне, имеет healthcheck и оставляет пригодные для диагностики логи.
- есть основной успешный сценарий;
- есть ожидаемый сбой и понятная диагностика;
- результат воспроизводится другим человеком;
- README, схема, test matrix или runbook обновлены;
- ученик прослеживает путь данных и ответственность компонентов без подсказки;
