## 167. Environment variables и конфигурация окружений

### Цель

Вынести изменяемую конфигурацию из кода и разделить development, test и production.

### Почему тема появляется сейчас

Процесс уже виден. Теперь нужно определить, откуда он получает URL базы, режим логирования и секреты.

### Основное содержание

- environment variable как значение процесса;
- `export` и чтение переменной;
- Settings как граница конфигурации;
- development, test и production;
- `.env` и `.env.example`;
- секреты и безопасные значения по умолчанию;
- валидация обязательной конфигурации при старте;
- почему config не должен зависеть от IDE;

### Практика StudyHub

Настроить `APP_ENV`, `DATABASE_URL`, `LOG_LEVEL` и `SECRET_KEY`; проверить два запуска StudyHub с разными значениями без изменения Python-кода.

### Визуальная и интерактивная подача

Три карточки окружений; схема environment → Settings → application; BugHunt с секретом в репозитории.

### Результат занятия

Один и тот же код запускается в разных окружениях через внешнюю конфигурацию.

### Граница занятия

Не вводить облачные secret managers. Достаточно корректной модели и `.env.example`.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
