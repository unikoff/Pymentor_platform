## 165. Linux-путь проекта и базовая навигация

### Цель

Научиться уверенно находить проект, запускать команды из правильной директории и читать структуру файлов без графического интерфейса.

### Почему тема появляется сейчас

До Docker ученик должен понимать среду контейнера. Иначе команды внутри image будут восприниматься как непрозрачная магия.

### Основное содержание

- домашняя директория и корень файловой системы;
- абсолютный и относительный path;
- `pwd`, `ls`, `cd`;
- `mkdir`, `touch`, `cp`, `mv`, `rm` на безопасных примерах;
- `cat`, `head`, `tail`;
- скрытые файлы и `.env`;
- current working directory;
- почему приложение не находит файл при запуске из другой папки;

### Практика StudyHub

Развернуть копию StudyHub в учебной Linux-среде, найти `app/main.py`, `.env.example`, migrations и tests, затем запустить проект из корректной директории.

### Визуальная и интерактивная подача

Карта файловой системы с отмеченным cwd; TerminalDemo «одна команда из двух разных папок»; BugHunt с неверным относительным path.

### Результат занятия

Ученик может открыть проект из терминала, объяснить cwd и найти причину ошибки пути.

### Граница занятия

Не превращать урок в справочник всех shell-команд. Изучаются только действия, необходимые backend-разработчику.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
