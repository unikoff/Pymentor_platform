## 184. Первый workflow GitHub Actions

### Цель

Создать workflow, который устанавливает Python dependencies и запускает проверки.

### Почему тема появляется сейчас

После модели CI можно перейти к минимальному YAML и наблюдаемому run.

### Основное содержание

- workflow file;
- event `push` и `pull_request`;
- runner;
- checkout;
- setup Python;
- install dependencies;
- run formatter/linter/tests;
- чтение logs и annotations;

### Практика StudyHub

Добавить workflow `ci.yml`, открыть pull request, получить красный run, исправить проблему и добиться зелёного результата.

### Визуальная и интерактивная подача

YAML как иерархическая карта workflow/job/steps; CodeSequence минимального workflow.

### Результат занятия

Каждый pull request StudyHub автоматически проверяется.

### Граница занятия

Не создавать сложную matrix и reusable workflows до появления необходимости.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
