## 186. Автоматическая сборка и tagging Docker image

### Цель

Собирать один и тот же image после успешных проверок и присваивать ему прослеживаемую версию.

### Почему тема появляется сейчас

CI зелёный; теперь проверенный commit можно превратить в deployable artifact.

### Основное содержание

- Docker build в workflow;
- registry на концептуальном уровне;
- authentication token;
- tag по commit SHA;
- release tag;
- неизменяемость artifact;
- build cache;
- проверка image до публикации;

### Практика StudyHub

Собрать image по SHA, запустить smoke test container и опубликовать учебный tag в registry.

### Визуальная и интерактивная подача

Маршрут commit SHA → image tag → registry; CompareSolutions `latest` против прослеживаемой версии.

### Результат занятия

Каждой опубликованной версии соответствует конкретный commit.

### Граница занятия

Supply-chain signing и SBOM остаются дополнительным уровнем.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
