## 171. Image, container и изоляция процесса

### Цель

Построить точную модель Docker до написания Dockerfile.

### Почему тема появляется сейчас

Linux-процесс уже знаком. Container становится понятным как изолированный процесс с подготовленной файловой системой и конфигурацией.

### Основное содержание

- Docker daemon и client на прикладном уровне;
- image как неизменяемый шаблон;
- container как запущенный экземпляр;
- filesystem container;
- process внутри container;
- port container и host port;
- ephemeral state;
- почему container не является виртуальной машиной в рамках учебной модели;

### Практика StudyHub

Запустить готовый минимальный image, посмотреть process и filesystem, остановить и создать новый container.

### Визуальная и интерактивная подача

Схема Dockerfile → image → container → process; карточки «что сохраняется» и «что исчезает».

### Результат занятия

Ученик не путает image, container и Python process.

### Граница занятия

Namespaces, cgroups и internals ядра не входят в основной урок.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
