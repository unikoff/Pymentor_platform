## 174. Ports, environment и файловое состояние container

### Цель

Передавать конфигурацию при запуске и не путать внутренний port с опубликованным host port.

### Почему тема появляется сейчас

Container запускается, но пока ученик не управляет его окружением и не понимает границу файлового состояния.

### Основное содержание

- container port;
- host port mapping;
- `-p`;
- environment variables при `docker run`;
- `--env-file`;
- container filesystem;
- bind mount для development как отдельный режим;
- почему database-файл внутри удаляемого container ненадёжен;

### Практика StudyHub

Запустить один image на двух host ports с разными `APP_ENV`; создать файл внутри container и проверить его исчезновение после пересоздания.

### Визуальная и интерактивная подача

Двойная схема host/container ports; эксперимент «container removed → file gone».

### Результат занятия

Ученик управляет config и объясняет, какие данные не должны жить только внутри container layer.

### Граница занятия

Production bind mounts и хранение PostgreSQL пока не настраиваются — это следующий блок.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
