## 175. .dockerignore, непривилегированный пользователь и чистый image

### Цель

Сократить build context и убрать очевидные риски image.

### Почему тема появляется сейчас

После понимания context можно осмысленно исключить лишние и секретные файлы.

### Основное содержание

- `.dockerignore`;
- исключение `.git`, caches, local env и test artifacts;
- почему `.env` нельзя копировать;
- пользователь root внутри container;
- создание непривилегированного user;
- file ownership;
- минимальный набор runtime-файлов;
- проверка содержимого image;

### Практика StudyHub

Добавить `.dockerignore`, удалить secrets из context, запустить API не от root и проверить доступ к нужным файлам.

### Визуальная и интерактивная подача

BugHunt с утечкой `.env`; карточки «нужно в runtime / не нужно»; CompareSolutions root/non-root.

### Результат занятия

Image содержит только необходимые файлы и запускает приложение от отдельного пользователя.

### Граница занятия

Глубокий container hardening и vulnerability scanners остаются следующим уровнем.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
