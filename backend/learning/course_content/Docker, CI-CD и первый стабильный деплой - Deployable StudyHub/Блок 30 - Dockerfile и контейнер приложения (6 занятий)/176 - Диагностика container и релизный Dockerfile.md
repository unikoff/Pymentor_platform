## 176. Диагностика container и релизный Dockerfile

### Цель

Собрать рабочий Dockerfile блока и научиться находить ошибки build и runtime.

### Почему тема появляется сейчас

Блок завершается процедурой, а не только успешным запуском на машине автора.

### Основное содержание

- `docker ps` и stopped containers;
- `docker logs`;
- `docker inspect` на прикладном уровне;
- `docker exec` для диагностики;
- build error против runtime error;
- exit code;
- healthcheck image/application;
- tag локальной версии;

### Практика StudyHub

Исправить четыре сценария: неверный CMD, отсутствующий module, неправильный port и missing environment variable; собрать финальный image.

### Визуальная и интерактивная подача

Диагностическая матрица build/run/health; TerminalDemo расследования; контрольный маршрут image.

### Результат занятия

Создан релизный Dockerfile и инструкция `docker build`/`docker run`.

### Граница занятия

Публикация в registry и deployment появятся в блоке CI/CD.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;

## Контрольная точка блока

- StudyHub собирается в собственный Docker image, запускается непривилегированным пользователем и проходит healthcheck.
- есть основной успешный сценарий;
- есть ожидаемый сбой и понятная диагностика;
- результат воспроизводится другим человеком;
- README, схема, test matrix или runbook обновлены;
- ученик прослеживает путь данных и ответственность компонентов без подсказки;
