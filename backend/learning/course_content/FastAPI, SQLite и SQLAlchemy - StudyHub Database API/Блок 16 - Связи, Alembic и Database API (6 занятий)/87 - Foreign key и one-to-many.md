## 3. ORM-модели и ForeignKey

Сначала описываем только структуру хранения. CategoryModel получает primary key, TaskModel — nullable category_id с ForeignKey. Навигация по объектам появится в следующем уроке.

### Точная модель

- **Контекст:** StudyHub уже находится в состоянии, описанном в паспорте занятия.
- **Механизм:** раздел «ORM-модели и ForeignKey» добавляет одну новую техническую идею.
- **Контракт:** до выполнения называются вход, ожидаемый результат и допустимые ошибки.
- **Граница:** механизм не подменяет ответственность соседнего слоя.

### Пример 1

```python
from sqlalchemy import ForeignKey, String
from sqlalchemy.orm import Mapped, mapped_column

from app.database import Base


class CategoryModel(Base):
    __tablename__ = "categories"

    id: Mapped[int] = mapped_column(primary_key=True)
    name: Mapped[str] = mapped_column(
        String(80),
        unique=True,
        nullable=False,
    )


class TaskModel(Base):
    __tablename__ = "tasks"

    id: Mapped[int] = mapped_column(primary_key=True)
    title: Mapped[str] = mapped_column(String(200))
    category_id: Mapped[int | None] = mapped_column(
        ForeignKey("categories.id"),
        nullable=True,
    )
```

    session: Session,
