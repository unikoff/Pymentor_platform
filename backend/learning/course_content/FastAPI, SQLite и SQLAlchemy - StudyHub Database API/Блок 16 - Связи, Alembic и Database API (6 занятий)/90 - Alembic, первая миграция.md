## 4. revision --autogenerate и проверка

Команда создаёт кандидата migration. До upgrade проверяются имена таблиц, типы, nullable, unique, ForeignKey и порядок downgrade.

### Точная модель

- **Контекст:** StudyHub уже находится в состоянии, описанном в паспорте занятия.
- **Механизм:** раздел «revision --autogenerate и проверка» добавляет одну новую техническую идею.
- **Контракт:** до выполнения называются вход, ожидаемый результат и допустимые ошибки.
- **Граница:** механизм не подменяет ответственность соседнего слоя.

### Пример 1

```text
def upgrade() -> None:
    op.create_table(
        "categories",
        sa.Column("id", sa.Integer(), nullable=False),
        sa.Column("name", sa.String(80), nullable=False),
        sa.PrimaryKeyConstraint("id"),
        sa.UniqueConstraint("name"),
    )
    op.create_table(
        "tasks",
        sa.Column("id", sa.Integer(), nullable=False),
        sa.Column("title", sa.String(200), nullable=False),
        sa.Column("category_id", sa.Integer(), nullable=True),
        sa.ForeignKeyConstraint(
            ["category_id"],
            ["categories.id"],
        ),
        sa.PrimaryKeyConstraint("id"),
    )


def downgrade() -> None:
    op.drop_table("tasks")
    op.drop_table("categories")
```
