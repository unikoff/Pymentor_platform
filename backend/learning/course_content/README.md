# Контент платформы: треки и занятия

Полная инструкция с готовыми шаблонами практики, автопроверки и чеклистов: [README-практика.md](README-практика.md).

## Структура

```
course_content/
  README.md               <- эта инструкция (студентам не отдаётся)
  python-basics/          <- ТРЕК = папка
    track.json            <- название и порядок трека
    План обучения.md      <- файлы в корне трека = модуль «00 Обзор/Старт»
    Блок 1 - .../урок.md  <- ПОДПАПКА = модуль, .md-ФАЙЛ = занятие
  sql-basics/
    track.json
    01 Основы баз данных/Знакомство с SQL.md
```

## Как создать новый трек

1. Создай папку в `course_content`, например `js-basics`.
2. Положи в неё `track.json`:

```json
{
  "title": "JavaScript с нуля",
  "tagline": "Короткое описание под заголовком",
  "order": 3
}
```

`order` задаёт позицию в переключателе треков.

3. Добавь подпапки-модули и .md-файлы занятий (см. ниже).
4. Перезапусти бэкенд — трек появится в переключателе автоматически.

## Как добавить занятие

- Создай `.md`-файл в подпапке трека. Название модуля в дереве = имя подпапки
  (для python-basics подписи модулей переопределены в `learning/content.py` → `MODULE_LABELS`).
- Первая строка `# Урок: Название` станет заголовком занятия.
- Порядок занятий: по алфавиту имён файлов — используй префиксы `01 `, `02 `...
  (для python-basics порядок задан вручную в `LESSON_SEQUENCE` в `content.py`).
- Файлы `README.md`, `Задания.md` и `Ответы*.md` в контент не попадают.

## Доступ к занятиям

Карта обучения (`План обучения.md`) бесплатна в каждом треке. В первом курсе
дополнительно бесплатны весь блок 00 и блок 1, блок 2 доступен после регистрации,
а все остальные занятия требуют активную Pro-подписку
(логика в `content.py` → `_lesson_access`).

## Видео к занятию

В конце каждого `.md` есть скрытый маркер `<!-- youtube: -->`. Вставь ссылку на ролик внутрь него и перезапусти backend:

```md
<!-- youtube: https://youtu.be/VIDEO_ID -->
```

Поддерживаются ссылки `youtu.be`, `youtube.com/watch`, `embed`, `shorts` и `live`. Пока ссылка пуста, видеоблок на странице занятия не показывается.

## Практика: автопроверяемые задания

Внутри секции с заголовком, содержащим «Домашнее задание» или «Контрольная точка»,
добавь два сервисных блока (студент их не увидит — они вырезаются из теории):

1. `starter` — шаблон кода в редакторе студента:

    ```starter
    def solve(a, b):
        # верните сумму
        pass
    ```

2. `tests` — JSON-список тестов (`name`, `args` — аргументы solve, `expected`):

    ```tests
    [
      {"name": "пример", "args": [1, 2], "expected": 3},
      {"name": "проверка вывода", "args": [5], "expected": "5", "assert": "stdout"}
    ]
    ```

- `"assert": "stdout"` — сравнивать печать (`print`), а не возвращаемое значение.
- Кнопка «Запустить» выполняет код студента как есть; «Проверить» гоняет тесты.
- Успешная проверка автоматически засчитывает занятие в прогресс.
- Рабочий пример: `python-basics/Блок 2 .../Списки (Часть 1).md`.

## Занятия без автопроверки (кнопка «Выполнено»)

Если задание нельзя проверить кодом (например, SQL или установка софта) — добавь путь
файла в `SELF_CHECK_LESSONS` в `learning/content.py`:

```python
SELF_CHECK_LESSONS = {
    "sql-basics/01 Основы баз данных/Знакомство с SQL.md",
}
```

Вместо практики студент увидит кнопку «Отметить выполненным».

## Красивые страницы уроков (вместо markdown)

Теория по умолчанию рендерится из markdown. Чтобы сделать занятию дизайнерскую
страницу (как у первых двух уроков Python) — добавь React-компонент в
`frontend/src/lessonPages.tsx` и зарегистрируй его по `source_file` урока:

```tsx
export const richLessonPages: Record<string, React.ComponentType> = {
  "python-basics/Блок 1 - .../Базовые типы данных 1.md": NumbersAndStringsLesson,
};
```
