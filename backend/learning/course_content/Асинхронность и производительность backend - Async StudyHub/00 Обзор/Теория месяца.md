# Этап 7. Асинхронность и производительность backend

## Общая цель

Этап строит точную модель конкурентного I/O. Сначала asyncio изучается отдельно от FastAPI, затем применяется к внешнему HTTP-сервису и только после этого к PostgreSQL через Async SQLAlchemy. Финальная проверка опирается на измерения, тесты и наблюдаемость.

## Сквозной проект

`Async StudyHub`

## Карта блоков

| Блок | Занятия | Название | Назначение |
| ---: | ---: | --- | --- |
| 25 | 141–146 | Coroutine, event loop и async/await | Сформировать модель асинхронного выполнения без FastAPI. |
| 26 | 147–152 | Конкурентные задачи, timeout и cancellation | Научиться управлять несколькими coroutine и ограничивать их жизненный цикл. |
| 27 | 153–158 | Асинхронный FastAPI и внешние HTTP-сервисы | Применить async к реальному сетевому I/O и тестировать интеграцию без сети. |
| 28 | 159–164 | Async SQLAlchemy, нагрузка и наблюдаемость | Перевести database I/O на AsyncSession и измерить результат. |

## Методика этапа

```text
проблема → минимальная модель → ручной эксперимент → применение в StudyHub
→ ошибочный сценарий → тест/измерение → объяснение результата
```
