# План развития StudyHub: этапы 6–9

## Общая рамка

После этапа 5 у ученика уже есть `Personal StudyHub API`: синхронный FastAPI, SQLite/SQLAlchemy, Alembic, пользователи, password hashing, cookie-сессии, JWT access/refresh, права доступа, тесты и воспроизводимый запуск.

Оставшиеся четыре этапа не повторяют регистрацию, JWT и базовый CRUD. Они последовательно решают новые проблемы уже работающего проекта:

```text
Personal StudyHub API
→ PostgreSQL StudyHub
→ Async StudyHub
→ Deployable StudyHub
→ StudyHub LMS Release
→ портфолио и собеседования
```

## Формат

- 4 основных блока на этап;
- 6 занятий в блоке;
- 24 занятия на этап;
- блоки 21–36;
- занятия 117–212;
- отдельный нулевой блок `LearningRoadmap + MonthTheory` для каждого этапа;
- один сквозной проект `StudyHub`;
- после каждого блока — практикум, проверка ошибок и code review;
- переход дальше только после контрольной точки.

# Этап 7. Асинхронность и производительность backend

## Цель

Не заменить механически `def` на `async def`, а построить точную модель:

```text
coroutine
→ event loop
→ ожидание I/O
→ выполнение другой задачи
→ продолжение первой задачи
```

Сначала механизм изучается отдельно от FastAPI. Затем подключается асинхронный HTTP-клиент, после этого — Async SQLAlchemy.

## Блок 25. Coroutine, event loop и async/await

Занятия 141–146.

- blocking execution;
- CPU-bound и I/O-bound;
- coroutine object;
- `asyncio.run`;
- `await`;
- последовательные await;
- типичные async-ошибки;
- лабораторный загрузчик.

**Результат:** ученик трассирует coroutine и понимает, когда event loop может переключиться.

## Блок 26. Конкурентные задачи, timeout и cancellation

Занятия 147–152.

- `create_task`;
- `gather`;
- timeout;
- cancellation;
- обязательная очистка;
- semaphore;
- частичные результаты;
- итоговый async-агрегатор.

**Результат:** несколько I/O-операций выполняются конкурентно и имеют контролируемый жизненный цикл.

## Блок 27. Асинхронный FastAPI и внешние HTTP-сервисы

Занятия 153–158.

- `def` против `async def`;
- `httpx.AsyncClient`;
- network errors и status mapping;
- dependency для клиента;
- lifespan;
- агрегирующий endpoint;
- fake/mock внешнего API.

**Результат:** StudyHub безопасно обращается к внешнему сервису без реальной сети в тестах.

## Блок 28. Async SQLAlchemy, нагрузка и наблюдаемость

- async driver;
- `AsyncEngine`;

- `AsyncSession`;

**Итог этапа:** `Async StudyHub`.
