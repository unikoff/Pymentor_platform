## 142. async def и coroutine object

### Цель

Различить создание coroutine и её выполнение.

### Основное содержание

- `async def`;
- coroutine function;
- coroutine object;
- вызов без выполнения;
- repr/type coroutine;
- почему `print(fetch())` не даёт данные;
- runtime warning;
- осторожная связь с генераторами;

### Практика StudyHub

Создать несколько coroutine objects, предсказать вывод и намеренно получить `coroutine was never awaited`.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик понимает, что обычный вызов async-функции не запускает её тело.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
