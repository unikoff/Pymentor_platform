## 141. Блокирующее выполнение, CPU и I/O

### Цель

Увидеть, какую проблему решает асинхронность и какую не решает.

### Основное содержание

- последовательное выполнение;
- blocking operation;
- CPU-bound;
- I/O-bound;
- network request;
- database wait;
- latency;
- почему async не ускоряет вычисление автоматически;

### Практика StudyHub

Сравнить `time.sleep`, вычисление и имитацию сетевого ожидания; отметить, где программа работает, а где ждёт.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик объясняет, когда event loop может быть полезен.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
