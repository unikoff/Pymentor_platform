## 161. Асинхронный SELECT и CRUD

### Цель

Перенести знакомые SQLAlchemy statements на AsyncSession.

### Основное содержание

- `await session.execute`;
- `scalars`;
- `scalar_one_or_none`;
- `add` без await;
- commit/refresh с await;
- delete;
- 404;
- Pydantic response;

### Практика StudyHub

Перенести GET list, GET item и POST; сравнить sync/async diff.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Основной CRUD работает через AsyncSession.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
