## 160. async_sessionmaker и get_db

### Цель

Создавать одну AsyncSession на request и гарантированно закрывать её.

### Основное содержание

- `async_sessionmaker`;
- `AsyncSession`;
- expire_on_commit;
- async dependency;
- `async with`;
- `yield`;
- request scope;
- cleanup on error;

### Практика StudyHub

Создать async `get_db`, подключить к одному read endpoint и проверить lifecycle.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

FastAPI выдаёт каждому request отдельную AsyncSession.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
