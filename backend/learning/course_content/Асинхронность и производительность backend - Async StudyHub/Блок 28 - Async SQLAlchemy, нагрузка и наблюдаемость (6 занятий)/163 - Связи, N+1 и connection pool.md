## 163. Связи, N+1 и connection pool

### Цель

Не перенести скрытые проблемы синхронного приложения в async.

### Основное содержание

- N+1;
- eager loading;
- `selectinload`;
- неожиданный lazy I/O;
- pool size;
- pool exhaustion;
- долгая transaction;
- SQL logs;

### Практика StudyHub

Сравнить tasks + owner до/после selectinload и смоделировать ограниченный pool.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Связи загружаются предсказуемо, connections не удерживаются лишней работой.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
