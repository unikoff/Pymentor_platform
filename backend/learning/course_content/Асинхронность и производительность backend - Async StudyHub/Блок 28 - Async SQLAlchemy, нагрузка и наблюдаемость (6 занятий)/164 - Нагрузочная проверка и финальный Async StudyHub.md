## 164. Нагрузочная проверка и финальный Async StudyHub

### Цель

Завершить миграцию измерением, логами и регрессионной проверкой.

### Основное содержание

- latency;
- throughput;
- concurrency level;
- простая load tool;
- baseline;
- request id;
- structured logs;
- p50/p95;
- регрессионные tests;
- README решения;

### Практика StudyHub

Провести повторяемый sync/async I/O-тест, записать ограничения и защитить проект.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

`Async StudyHub` проходит тесты и имеет наблюдаемый request trace.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;

## Контрольная точка блока

- все шесть занятий связаны одним проектным изменением;
- есть воспроизводимый успешный сценарий;
- есть ожидаемая ошибка и её диагностика;
- обновлены README, схема данных или таблица решений;
- ученик прослеживает путь данных без подсказки;

---

# Итоговая контрольная точка этапа 7

- объяснить CPU-bound и I/O-bound;
- объяснить coroutine object и event loop;
- показать suspend/resume;
- отличить sequential await от concurrency;
- использовать create_task и gather;
- обработать timeout/cancellation;
- ограничить concurrency;
- выбрать def или async def;
- вызвать внешний API через AsyncClient;
- заменить интеграцию fake/mock;
- выполнить CRUD через AsyncSession;
- показать rollback;
- обнаружить N+1 и blocking call;
- сравнить измеренный sync/async сценарий;
- защитить Async StudyHub;

## Что намеренно не входит

- async ради синтаксиса;
- переписывание CPU-bound кода в async;
- WebSocket;
- Celery и distributed queues;
- Kafka/RabbitMQ;
- сложные cancellation scopes;
- глубокая настройка event loop;
- микросервисы;
- обещание ускорения каждого endpoint;
