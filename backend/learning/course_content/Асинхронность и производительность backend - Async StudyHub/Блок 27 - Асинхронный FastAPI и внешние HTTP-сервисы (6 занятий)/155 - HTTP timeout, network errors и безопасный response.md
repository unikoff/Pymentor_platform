## 155. HTTP timeout, network errors и безопасный response

### Цель

Разделить сбои внешней сети и внутренние ошибки StudyHub.

### Основное содержание

- connect/read timeout;
- `TimeoutException`;
- `RequestError`;
- неуспешный status;
- `raise_for_status`;
- mapping в 502/503/504;
- не выдавать traceback;
- structured log;

### Практика StudyHub

Проверить success, внешний 404, timeout и connection error.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Внешняя ошибка превращается в контролируемый API-контракт.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
