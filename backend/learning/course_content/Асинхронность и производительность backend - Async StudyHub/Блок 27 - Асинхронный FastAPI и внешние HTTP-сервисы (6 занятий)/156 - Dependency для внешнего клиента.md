## 156. Dependency для внешнего клиента

### Цель

Централизовать конфигурацию клиента и сделать его заменяемым в тестах.

### Основное содержание

- dependency provider;
- тип клиента;
- base URL из Settings;
- timeout из config;
- client в service;
- dependency override;
- fake client;
- endpoint/service boundary;

### Практика StudyHub

Создать `get_recommendation_client`, вынести service и заменить dependency в тесте.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Клиент создаётся по одному контракту и легко заменяется.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
