# План развития StudyHub: этапы 6–9

## Общая рамка

После этапа 5 у ученика уже есть `Personal StudyHub API`: синхронный FastAPI, SQLite/SQLAlchemy, Alembic, пользователи, password hashing, cookie-сессии, JWT access/refresh, права доступа, тесты и воспроизводимый запуск.

Оставшиеся четыре этапа не повторяют регистрацию, JWT и базовый CRUD. Они последовательно решают новые проблемы уже работающего проекта:

```text
Personal StudyHub API
→ PostgreSQL StudyHub
→ Async StudyHub
→ Deployable StudyHub
→ StudyHub LMS Release
→ портфолио и собеседования
```

## Формат

- 4 основных блока на этап;
- 6 занятий в блоке;
- 24 занятия на этап;
- блоки 21–36;
- занятия 117–212;
- отдельный нулевой блок `LearningRoadmap + MonthTheory` для каждого этапа;
- один сквозной проект `StudyHub`;
- после каждого блока — практикум, проверка ошибок и code review;
- переход дальше только после контрольной точки.

# Принципы последовательности

## Каждая технология отвечает на проблему

```text
PostgreSQL
→ SQLite недостаточно похожа на серверную production-СУБД

asyncio
→ приложение ожидает несколько независимых I/O-операций

Docker
→ проект трудно воспроизвести на другой машине

Redis
→ некоторые дорогие чтения повторяются

LMS-домен
→ простой CRUD задач недостаточно показывает бизнес-логику
```

## Что не входит в обязательный маршрут

- Kubernetes;
- Terraform;
- микросервисы;
- Kafka;
- RabbitMQ или Celery в глубину;
- сложный DDD;
- generic repository;
- enterprise RBAC;
- несколько постоянных баз в одном основном проекте;
- глубокая оптимизация PostgreSQL;
- сложная облачная инфраструктура.

## Педагогическая схема каждого блока

```text
понять проблему
→ увидеть минимальную модель
→ предсказать результат
→ запустить отдельный пример
→ встроить в StudyHub
→ проверить ошибочный сценарий
→ написать тест
→ объяснить решение
```

## Финальная контрольная точка

Ученик должен:

- развернуть проект по README;
- показать основной LMS-сценарий;
- объяснить HTTP request от клиента до PostgreSQL и обратно;
- показать права разных ролей;
- объяснить async-участок;
- показать кеш и invalidation;
- запустить тесты и CI;
- найти проблему по логам;
- объяснить Docker Compose;
- разобрать один SQL-запрос;
- пройти mock-интервью;
- защитить архитектурные решения.
