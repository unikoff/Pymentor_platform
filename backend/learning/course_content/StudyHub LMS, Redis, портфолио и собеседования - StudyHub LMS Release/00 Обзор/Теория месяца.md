# Этап 9. StudyHub LMS, Redis, портфолио и собеседования

## Общая цель

Финальный этап превращает технически зрелый сервис в содержательный LMS-проект. Сначала ученик проектирует домен и права, затем реализует вертикальный flow teacher/student, после этого добавляет обоснованные Redis-функции и завершает курс аудитом, тестами, документацией, interview lab и публичной защитой проекта.

## Методика этапа

Сначала фиксируются требования, данные и права. Реализация идёт вертикальными slices, чтобы каждый шаг давал работающий пользовательский сценарий. Redis появляется только после измеримой проблемы повторного чтения. Финальные занятия не добавляют новые модные технологии: они повышают качество, воспроизводимость и способность объяснять решения.

```text
понять проблему
→ увидеть минимальную модель
→ предсказать поведение
→ запустить отдельный пример
→ встроить в StudyHub
→ проверить сбой
→ автоматизировать проверку
→ объяснить решение
```
