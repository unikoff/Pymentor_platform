## 194. API contract, ER-диаграмма и migration plan

### Цель

Собрать проектирование блока в реализуемый план без большого одномоментного изменения.

### Почему тема появляется сейчас

Финальная точка проектирования должна дать порядок безопасной разработки.

### Основное содержание

- таблица method/path/role/input/response/errors;
- Pydantic schemas;
- ER-диаграмма;
- migration sequence;
- seed/demo data;
- обратная совместимость существующих endpoints;
- порядок реализации вертикальными slices;
- definition of done;

### Практика StudyHub

Подготовить contract для 12–16 LMS endpoints и разбить реализацию на четыре вертикальных сценария.

### Визуальная и интерактивная подача

Карта endpoint → schema → service → tables; CodeSequence migration plan; финальная architecture board.

### Результат занятия

Есть согласованный blueprint LMS Core, по которому можно начинать кодирование.

### Граница занятия

Не создавать все таблицы и endpoints в одном giant commit.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;

## Контрольная точка блока

- Утверждены MVP, ER-диаграмма, permissions matrix, API contract и безопасный migration plan.
- есть основной успешный сценарий;
- есть ожидаемый сбой и понятная диагностика;
- результат воспроизводится другим человеком;
- README, схема, test matrix или runbook обновлены;
- ученик прослеживает путь данных и ответственность компонентов без подсказки;
