## 211. Interview lab: Python, SQL, API и code review

### Цель

Потренировать типовые рабочие действия в ограниченное время и объяснять ход решения.

### Почему тема появляется сейчас

Перед финальной защитой нужен отдельный режим, где подсказки проекта недоступны.

### Основное содержание

- чтение чужой Python-функции;
- поиск дефекта по traceback/log;
- небольшой FastAPI endpoint;
- SQL SELECT/JOIN;
- pytest на ошибочный scenario;
- code review diff;
- оценка сложности простого алгоритма;
- уточнение требований до кода;

### Практика StudyHub

Пройти 90-минутный mock-набор: исправить bug, дописать test, написать SQL и провести review небольшого PR.

### Визуальная и интерактивная подача

Таймлайн интервью; карточки «уточнить → предложить → проверить → объяснить»; разбор решений.

### Результат занятия

Ученик демонстрирует процесс решения, а не только финальный код.

### Граница занятия

Сложные олимпиадные алгоритмы и системный дизайн senior-уровня не входят.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример или инфраструктурный сценарий;
- изменяет один параметр и предсказывает результат;
- проверяет успешный и ошибочный путь;
- фиксирует воспроизводимую проверку, тест или runbook;
- делает один осмысленный Git-коммит;
