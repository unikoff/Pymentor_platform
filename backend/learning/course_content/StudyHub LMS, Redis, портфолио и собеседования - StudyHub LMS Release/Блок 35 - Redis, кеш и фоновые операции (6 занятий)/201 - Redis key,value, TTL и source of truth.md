## 201. Redis key/value, TTL и source of truth

### Цель

Понять временную модель Redis и не перепутать кеш с основной базой.

### Почему тема появляется сейчас

Redis уже присутствует в Compose, но пока не хранит бизнес-данные. Теперь появляется реальный каталог, который можно кешировать.

### Основное содержание

- key и value;
- TTL;
- expire;
- cache hit и miss;
- serialization;
- namespace ключей;
- PostgreSQL как source of truth;
- поведение при недоступном Redis;

### Практика StudyHub

Через небольшой Python client записать временное значение, проверить TTL и удалить Redis data без потери курсов PostgreSQL.

### Визуальная и интерактивная подача

Две базы с разными ролями; timeline TTL; True/False «где находится истина».

### Результат занятия

Ученик объясняет, почему удаление кеша не удаляет продуктовые данные.

### Граница занятия

Streams, pub/sub и Redis persistence не входят.
