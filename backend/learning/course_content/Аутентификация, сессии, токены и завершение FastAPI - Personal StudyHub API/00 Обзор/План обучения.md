# План обучения: этап 5

## Цель этапа

Добавить пользователей и безопасно защитить данные. Ученик сравнивает способы аутентификации, реализует cookie-сессии и JWT, получает текущего пользователя через dependency, проверяет владение объектом и тестирует безопасность API.

## Сквозной проект

`Personal StudyHub API`

К концу этапа API поддерживает регистрацию, хеширование паролей, cookie-сессии, bearer/JWT-аутентификацию, refresh-токены, роли, права, загрузку файлов, тесты и документацию.
