# Теория этапа 5

Этап посвящён защите данных в FastAPI. Ученик различает идентификацию, аутентификацию и авторизацию, хранит пароли только в виде хешей, строит зависимости для current user и сравнивает cookie-сессии с bearer/JWT-токенами.

Главный принцип этапа: доступ к ресурсу проверяется на сервере, а пользователь, сессия, токен, роль и владелец объекта являются отдельными понятиями.

## Результат

Ученик умеет собрать Personal StudyHub API с регистрацией, входом, logout, истечением и отзывом сессий, JWT и refresh rotation, ролями, правами, тестами и понятной документацией.
