# План развития StudyHub: этапы 6–9

## Общая рамка

После этапа 5 у ученика уже есть `Personal StudyHub API`: синхронный FastAPI, SQLite/SQLAlchemy, Alembic, пользователи, password hashing, cookie-сессии, JWT access/refresh, права доступа, тесты и воспроизводимый запуск.

Оставшиеся четыре этапа не повторяют регистрацию, JWT и базовый CRUD. Они последовательно решают новые проблемы уже работающего проекта:

```text
Personal StudyHub API
→ PostgreSQL StudyHub
→ Async StudyHub
→ Deployable StudyHub
→ StudyHub LMS Release
→ портфолио и собеседования
```

## Формат

- 4 основных блока на этап;
- 6 занятий в блоке;
- 24 занятия на этап;
- блоки 21–36;
- занятия 117–212;
- отдельный нулевой блок `LearningRoadmap + MonthTheory` для каждого этапа;
- один сквозной проект `StudyHub`;
- после каждого блока — практикум, проверка ошибок и code review;
- переход дальше только после контрольной точки.

# Этап 6. SQL, PostgreSQL и выбор хранилища

## Цель

Убрать восприятие ORM как магии. Ученик должен увидеть SQL под SQLAlchemy, перенести StudyHub с SQLite на PostgreSQL, читать запросы, понимать ограничения, JOIN, транзакции и индексы.

MongoDB и Redis изучаются как другие модели хранения, но основной StudyHub остаётся реляционным приложением на PostgreSQL.

## Блок 21. SQL как язык данных

Занятия 117–122.

- реляционная модель;
- `CREATE TABLE` и ограничения;
- `INSERT` и параметры;
- `SELECT`, `WHERE`, сортировка и пагинация;
- безопасные `UPDATE` и `DELETE`;
- двусторонний перевод SQL ↔ SQLAlchemy.

**Результат:** ученик выполняет базовый CRUD вручную и понимает SQL под ORM.

## Блок 22. PostgreSQL и перенос StudyHub

Занятия 123–128.

```text
рабочая SQLite-версия
→ сервер PostgreSQL
→ database и role
→ connection URL
→ Alembic upgrade
→ перенос тестовых данных
→ запуск API и тестов
```

**Результат:** StudyHub работает на PostgreSQL, а его HTTP-контракт не изменился.

## Блок 23. JOIN, агрегаты и транзакции

Занятия 129–134.

- `INNER JOIN`;
- `LEFT JOIN`;
- many-to-many;
- `COUNT`, `GROUP BY`;
- `HAVING`, `EXISTS`, подзапросы;
- атомарные операции и rollback.

**Результат:** появляются статистические endpoints и безопасные операции из нескольких изменений.

## Блок 24. Индексы, планы запросов и модели хранения

Занятия 135–140.

- измерение запроса;
- одиночные и составные индексы;
- `EXPLAIN` и `EXPLAIN ANALYZE`;
- backup и restore;
- MongoDB как документная модель;
- Redis, TTL и итоговый выбор хранилища.

**Итог этапа:** `PostgreSQL StudyHub`.
