## 122. SQL и SQLAlchemy: две формы одного запроса

### Цель

Закрепить SQL через двусторонний перевод между raw SQL и SQLAlchemy 2.x.

### Основное содержание

- чтение сгенерированного SQL;
- `select`, `where`, `order_by`, `limit`;
- ORM object и Row;
- параметры statement;
- когда raw SQL оправдан;
- когда ORM понятнее;
- проверка одинакового результата;

### Практика StudyHub

Для четырёх операций записать SQL и SQLAlchemy-вариант и сравнить результаты тестом.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Создан `sql-lab/` и таблица соответствий SQL ↔ ORM.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;

## Контрольная точка блока

- все шесть занятий связаны одним проектным изменением;
- есть воспроизводимый успешный сценарий;
- есть ожидаемая ошибка и её диагностика;
- обновлены README, схема данных или таблица решений;
- ученик прослеживает путь данных без подсказки;
