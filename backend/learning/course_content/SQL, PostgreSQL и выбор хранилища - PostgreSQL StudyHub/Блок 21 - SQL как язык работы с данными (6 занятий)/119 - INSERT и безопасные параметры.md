## 119. INSERT и безопасные параметры

### Цель

Добавлять строки через SQL без склеивания пользовательских значений с текстом запроса.

### Основное содержание

- `INSERT INTO`;
- явный список колонок;
- одна и несколько строк;
- `RETURNING`;
- server default;
- параметризованный запрос;
- почему f-строка в SQL опасна;
- SQL injection на безопасном примере;

### Практика StudyHub

Вставить несколько задач параметризованно и сравнить `RETURNING` с `session.add()` + `refresh()`.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик отделяет SQL statement от передаваемых значений.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
