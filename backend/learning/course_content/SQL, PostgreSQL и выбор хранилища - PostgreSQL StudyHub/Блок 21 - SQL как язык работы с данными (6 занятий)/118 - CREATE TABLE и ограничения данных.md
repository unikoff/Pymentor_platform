## 118. CREATE TABLE и ограничения данных

### Цель

Научиться проектировать минимальную таблицу и переносить часть гарантий на уровень базы.

### Основное содержание

- `CREATE TABLE`;
- `INTEGER`, `VARCHAR`, `BOOLEAN`, `TIMESTAMP`;
- `PRIMARY KEY`;
- `NOT NULL`;
- `UNIQUE`;
- `DEFAULT`;
- `CHECK`;
- Pydantic validation против database constraint;

### Практика StudyHub

Создать учебную таблицу задач и намеренно нарушить три ограничения.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик создаёт таблицу с явными ограничениями и читает ошибки базы.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
