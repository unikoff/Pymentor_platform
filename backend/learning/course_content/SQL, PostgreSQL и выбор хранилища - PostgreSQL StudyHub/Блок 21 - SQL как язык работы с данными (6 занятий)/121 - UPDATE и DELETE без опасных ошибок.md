## 121. UPDATE и DELETE без опасных ошибок

### Цель

Изменять и удалять только выбранные строки и проверять число затронутых записей.

### Основное содержание

- `UPDATE ... SET`;
- `DELETE FROM`;
- обязательный `WHERE`;
- `RETURNING`;
- row count;
- 0/1/много строк;
- опасность запроса без условия;
- учебный rollback;

### Практика StudyHub

Обновить одну задачу, удалить тестовую запись и остановить опасный запрос без `WHERE`.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик безопасно выполняет UPDATE/DELETE и проверяет результат.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
