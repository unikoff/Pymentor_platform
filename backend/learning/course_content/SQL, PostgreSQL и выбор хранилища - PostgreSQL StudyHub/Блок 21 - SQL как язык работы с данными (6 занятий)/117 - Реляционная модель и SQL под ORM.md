# Блок 21. SQL как язык работы с данными

**Занятия:** 117–122.

**Назначение:** Связать SQLAlchemy с реляционной моделью и ручным SQL.

## 117. Реляционная модель и SQL под ORM

### Цель

Увидеть таблицу как набор строк, колонок, типов и ограничений, а ORM-модель — как отображение этой структуры.

### Основное содержание

- таблица, строка, колонка и значение;
- схема таблицы и данные;
- тип колонки;
- primary key;
- ORM class → table;
- чтение SQL-лога SQLAlchemy;

### Практика StudyHub

Открыть таблицы StudyHub, сопоставить поля моделей с колонками и подписать SQL для `select(TaskModel)`.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Ученик объясняет путь `TaskModel → SQL → строка таблицы`.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
