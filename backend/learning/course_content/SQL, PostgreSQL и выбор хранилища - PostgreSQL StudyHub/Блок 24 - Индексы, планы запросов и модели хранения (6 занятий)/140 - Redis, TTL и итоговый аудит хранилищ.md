## 140. Redis, TTL и итоговый аудит хранилищ

### Цель

Понять key-value модель и выбрать хранилище по требованиям.

### Основное содержание

- key/value;
- TTL;
- cache hit/miss;
- временность;
- PostgreSQL source of truth;
- опасность важного state только в cache;
- redis-cli lab;
- матрица выбора;

### Практика StudyHub

Сохранить временный verification code или cached stats с TTL и защитить карту данных StudyHub.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Защищён `PostgreSQL StudyHub` и выбор хранилищ.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;

## Контрольная точка блока
