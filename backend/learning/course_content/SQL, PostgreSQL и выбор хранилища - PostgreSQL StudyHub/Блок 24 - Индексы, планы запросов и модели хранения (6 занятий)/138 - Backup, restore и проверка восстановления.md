## 138. Backup, restore и проверка восстановления

### Цель

Отличить migrations от резервной копии данных.

### Основное содержание

- `pg_dump`;
- `pg_restore`;
- plain/custom format;
- schema + data;
- restore в отдельную database;
- проверка rows;
- migrations vs backup;
- README runbook;

### Практика StudyHub

Создать backup, восстановить в `studyhub_restore_test` и запустить smoke tests.

### Визуальная подача

Использовать схему движения данных, карточки ролей, сравнение корректного и ошибочного подхода, пошаговый разбор и одну доминирующую интерактивную механику.

### Результат занятия

Есть проверенная процедура восстановления.

### Критерии готовности

- ученик объясняет главную модель своими словами;
- запускает минимальный пример;
- изменяет один параметр и предсказывает эффект;
- проверяет успешный и ошибочный сценарий;
- фиксирует результат отдельным Git-коммитом;
